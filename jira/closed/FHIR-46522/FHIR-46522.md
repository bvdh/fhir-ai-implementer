# FHIR-46522: Integer64 vs. System.Integer

## Metadata

- **__EMPTY**: Bryn Rhodes

- **Assignee**: Unassigned

- **Created**: 8/1/24
- **Description**: I noticed that the Integer64 type is currently listed with a datatype of http://hl7.org/fhirpath/System.Integer, which is not correct.

The FHIRPath Integer type is defined as a 32-bit integer.

The extended type listed in the extension http://hl7.org/fhir/StructureDefinition/structuredefinition-fhir-type is correct as integer64.
- **Grouping**: [Applied]
- **Issue Type**: Technical Correction
- **Key**: FHIR-46522
- **Related Page(s)**: Datatypes
- **Reporter**: Gino Canessa
- **Resolution**: Persuasive
- **Resolution Description**: The CI build of FHIRPath includes a 64-bit integer type: [Long|https://build.fhir.org/ig/HL7/FHIRPath/#long].

Update the primitive type definition to reference this type, assuming that FHIRPath 2.1.0 is published before R6.
