# FHIR-56000: Represent approximate comparator in quantity

- **Created**: 3/3/26
- **Description**: During the data mapping work between openEHR and FHIR, it was noted that we do not have a way to represent an 'approximate' quantity via the comparator.

For example, representing that a quantity is approximately 10 mg.

Propose either adding a new code to the comparators set or defining an extension that can be used to represent this concept.
- **Issue Type**: Change Request
- **Key**: FHIR-56000
- **Related Page(s)**: Datatypes
- **Related URL**: http://build.fhir.org/valueset-quantity-comparator.html
- **Reporter**: Gino Canessa
- **Resolution**: Persuasive with Modification
- **Resolution Description**: Will add a code for approximate:
* Code: {{~}}
* Display: {{Approximately}}
* Definition: The actual value is an approximation.
* Comment:
{quote}
Note that a recommended value for the approximation is 10% of the stated value (or for a date, 10% of the gap between now and the date), but systems may choose other values where appropriate. Systems SHOULD use the {{Quantity Confidence Interval}} extension ({{http://hl7.org/fhir/StructureDefinition/quantity-confidenceInterval}}) to provide more detail when possible.
{quote}
