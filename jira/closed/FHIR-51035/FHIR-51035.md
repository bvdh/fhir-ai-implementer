# FHIR-51035: Interfaces are not valid base types

## Metadata

- **Assignee**: Unassigned
- **Created**: 6/6/25
- **Description**: The section discussing additional resources has the following statement:

The baseDefinition SHALL be a direct reference to one of the abstract resources published by this specification (Resource, DomainResource, CanonicalResource or MetadataResource)

The interfaces (CanonicalResource and MetadataResource) are not valid entries to use as the base of a resource definition.

Resources that implement those interfaces use the appropriate base (Resource or DomainResource) and indicate support for the interface via the extension http://hl7.org/fhir/StructureDefinition/structuredefinition-implements.
