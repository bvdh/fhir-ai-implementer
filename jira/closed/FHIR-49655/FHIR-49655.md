- **Description**: I believe that the below bullet list point is a left over from an earlier version of the FHIR spec when canonicals were not a real datatype, and the uri type was used.

	This search only works for specific data elements of type of uri that act as canonical URL's (see list below)



Should this just be updated to state that:

"This search only works for specific data elements of type of canonical (see list below of resources that contain them)"

Don't know if that's a generated list - otherwise this is probably stale.

Also might want to mention the definition or MetadataResource interfaces

Also, the bullet list has an extra empty bullet list at the bottom - remove it.

- **Reporter**: Brian Postlethwaite
- **Resolution**: Persuasive with Modification
- **Resolution Description**: The text is referring to the {{url}} element on canonical resources that defines their own canonical URL, which is defined as the {{uri}} type.

That said, the text could use some clarification. Change:
{quote}This search only works for specific data elements of type of uri that act as canonical URL's (see list below)
{quote}
to:
{quote}This search only works when resolving to resources having a 'url' element of type uri that acts as the canonical URL for that resource (see list below).
{quote}
