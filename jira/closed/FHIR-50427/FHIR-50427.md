# FHIR-50427: Wording on absolute URL is too absolute

## Metadata

- **Assignee**: Unassigned
- **Created**: 4/28/25
- **Description**: We currently say: "When populated, the url element SHALL be an absolute url that can be de-referenced to the attachment's content."

We should change this to say "If the data element is not populated and the url element is populated, the url element SHALL be an absolute url that can be de-referenced to the attachment's content.
