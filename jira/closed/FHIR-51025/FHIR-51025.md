# FHIR-51025: TestScript enhancement – the ability to define sub-fixtures

- **Description**: Based on FHIR-37824, this proposed enhancement to the TestScript resource will provide the ability for fixture definitions to support path expressions to extract an internal structure to be used as the fixture contents.

The primary example usage of this feature is the ability to simplify the extraction of Bundle.entry.resource contents.

New TestScript.fixture sub-elements, expression and path, will be defined where one or the other but not both will be allowed. If either is present, the Test Engine SHALL evaluate the defined fixture.resource with the defined expression or path to extract the required structure which then becomes the fixture contents.

An additional update will be applied to set the cardinality of the TestScript.fixture.resource as 1..1.

Here is the proposed TestScript change to TestScript.fixture:


- fixture
  |- autocreate         1..1  boolean
  |- autodelete         1..1  boolean
  |- resource           1..1  Reference(Any)
  |- expression         0..1  string          The FHIRPath expression against the fixture to extract the required structure
  |- path               0..1  string          XPath or JSONPath against the fixture to extract the required structure
