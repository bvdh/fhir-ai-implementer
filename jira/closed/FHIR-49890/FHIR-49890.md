# FHIR-49890: Raise cardinality of statusReason in the request pattern to 0..*

- **Description**: OO rejected a request to change Task.statusReason to 0..* bacause the underlying request pattern has it constrained to 0..1.

If their reasoning is correct, then this request is to change the cardinality in the pattern to 0..*

If their reasoning is incorrect, the request is to convey this to OO, so we can reopen FHIR-31775 and get a decision based on the request's merits.

 

For completeness, here is the original reasoning for the request:
&#91;...&#93; there could be multiple reasons why a status was chosen. For example, a Task with a status of "rejected" could be rejected for multiple recorded reasons.

So, it would make sense for statusReason to have cardinality 0..* for Task and other resources.

This would also match Medication.statusReason, which has a cardinality of 0..*
