- **Description**: Add language to the Package spec that says:

1. NPM FHIR Packages SHOULD contain an ImplementationGuide resource with the url:
&lt;package-canonical&gt;/ImplementationGuide/&lt;packageId&gt;, and most publication pipelines do.

2. NPM FHIR Packages can be referenced as a canonical reference using the canonical reference to the implementation guide (as they are in the dependsOn element of the implementation guide resource).
- **Issue Type**: Change Request
- **Key**: FHIR-50860
- **Related Page(s)**: Packages
- **Related URL**: https://build.fhir.org/packages.html
- **Reporter**: Bryn Rhodes
- **Resolution**: Persuasive with Modification
- **Resolution Description**: Add language to the Package spec that says:

1. NPM FHIR Packages SHOULD contain an ImplementationGuide resource with the url:
<package-canonical>/ImplementationGuide/<packageId> and the version to match the package version.

2. It is reasonable for tooling to resolve a canonical reference to an implementation guide resource as the NPM FHIR Package matching that definition (e.g., per the {{dependsOn}} element of the implementation guide resource).
