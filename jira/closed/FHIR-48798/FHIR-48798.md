# FHIR-48798: Replace binding on Basic.code

## Metadata

- **Assignee**: Unassigned
- **Created**: 11/7/24
- **Description**: Currently, Basic.code has a binding (example) to Valueset-basic-resource-type - FHIR v6.0.0-ballot2. When attempting to use Basic to pre-adopt EnounterHistory attributes into v2-FHIR mapping, which is based on FHIR R4, we learned that we needed to use http://hl7.org/fhir/fhir-types.  We recommend that either the current binding is replaced and reference this one, or that it is added as a valid value set.
