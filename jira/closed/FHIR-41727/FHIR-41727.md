# FHIR-41727: Performance of txt-2 invariant

## Metadata

- **Assignee**: Unassigned
- **Created**: 8/10/23
- **Description**: Invariant `txt-1` calls the FhirPath function `htmlChecks()` to do a full xsd validation validation of the narrative, this is on most systems a quite expensive operation, but necessary to validate what `txt-1` is supposed to do.

 

`txt-2` however, is about having at least some non-whitespace content in the narrative. Unfortunately, this invariant is also enforced by just running `htmlChecks()` again, meaning that a validator operationally will run the expensive xsd check twice. I am not even sure an empty document is considered invalid.

Is there no more efficient way to formulate this invariant?  like `trim().length() &gt; 0`?

Or maybe this is just a copy &amp; paste mistake
