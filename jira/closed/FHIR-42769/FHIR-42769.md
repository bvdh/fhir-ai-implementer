# FHIR-42769: Clarify whether 202 is allowed on creates, and what it means

## Metadata

- **Assignee**: Gino Canessa
- **Created**: 9/18/23
- **Description**: The summary table at https://build.fhir.org/fhir/http#create:~:text=201%2C-,202,-%2C%20400%2C%20404%2C%20405%2C%20422 indicates that 202 is an allowed response on `create` operations.

But the section on `create` does not mention this status code.

Is it allowed, and if so what does it mean?
