# FHIR-41478: Clarify when OID and GUID identifiers, etc. must use RFC 3986 system and format

- **Description**: From https://chat.fhir.org/#narrow/stream/179166-implementers/topic/Distinguishing.20OID.20identifiers/near/366559499 (June 15 through June 16 approximately) it appears that there is some ambiguity about when OIDs and GUIDs are permitted/required to use the urn:ietf:rfc:3986 system and urn:oid: or urn:uuid format.

We should clarify if any system or IG can declare certain uses of OIDs/GUIDs are outside the scope of the urn:ietf:rfc:3986 system or if that capability is reserved to just HL7.

We should also clarify that if a use of OIDs/GUIDs is outside the urn:ietf:rfc:3986 system, whether the values are still permitted/required to have the prefix urn:oid: or urn:uuid: (and any other format requirements, like lowercase).

For convenience, we should point out that the use of OIDs/GUIDs that use the RFC3986 system might be distinguished by identifier.type; while OIDs/GUIDs that use other systems might be distinguished by either system or type.
