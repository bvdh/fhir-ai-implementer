# FHIR-50201: Extension tab in Resources page brings back to R5

## Metadata

- **Assignee**: Unassigned
- **Created**: 4/8/25
- **Description**: When navigating the page of a resource on a given version (issue observed with 3rd ballot version), and clicking on the "extensions" tab, the version of the page switches to R5.
- **Issue Type**: Technical Correction
- **Key**: FHIR-50201
- **Related Page(s)**: ActivityDefinition Extensions
- **Related URL**: https://hl7.org/fhir/extensions/extensions-ActivityDefinition.html
- **Reporter**: IgnacioJauregui
- **Resolution**: Not Persuasive
- **Resolution Description**: The extensions pack has fancy behavior that allows it to appear as though it is 'part' of the current FHIR release, even though it's actually published as a separate IG.  Unfortunately, there's no way for the code that manages the menu to know whether you came there from looking at the official release or some other place (R6 ballot, R6 CI-build, etc.).  That means that the links are always going to take you to the current 'authoritative' version of FHIR, regardless of where you may have started.
- **Status**: Resolved - No Change
- **Summary**: Extension tab in Resources page brings back to R5
- **Work Group**: FHIR Infrastructure

- **Jira**: https://jira.hl7.org/browse/FHIR-50201
