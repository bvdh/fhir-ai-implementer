- **Description**: The async-bundle pattern does not state when the data returned in the result is no longer available. Is the data automatically deleted once the Status Request returns an Error/Complete or can a a client depend on the data to be available after such response? 

I suggest to:
1. update the delete section to state that calling delete on a completed request will remove any status data.
2. state in 3.2.6.2.4 that the client should not assume the data to be present after receiving a Response-Error or Response-Complete result.
