# FHIR-42721: Requirements need be able to reference the IG it created from

## Metadata

- **Assignee**: Unassigned
- **Created**: 9/10/23
- **Description**: Currently, it is not clear how to say a Requirements A is based on / created from / used with an IG B.

 

Requirement.reference is close but the description is not clear.

 

Current Requirement.reference description: "A reference to another artifact that created this set of requirements. This could be a Profile, etc., or external regulation, or business requirements expressed elsewhere."

 
Example:

I created a Requirements instance that US core server shall support US Core Patient profile. How do I link that Requirement instance to US Core IG v3.1.1?
Suggestion:

Clarify if Requirements.reference can be used to link a Requirements instance to an IG.

If not, should we create a new element for that purpose.
