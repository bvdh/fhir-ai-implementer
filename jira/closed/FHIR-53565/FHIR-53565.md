# FHIR-53565: SearchParameters should not start with a hyphen

- **Description**: The hyphen ("-") character is recommended as a prefix for custom search parameters to avoid collisions with future matching parameter names, but consider the following:

	User implements a custom search parameter on Encounter "-end-date". 
	Clients of user's FHIR implementation search for Encounters, sorted by their end date, using


Encounter?_sort=-end-date

to sort Encounters by their end dates in ascending order, and 


Encounter?_sort=--end-date

 to sort Encounters by their end dates is descending order.
	FHIR specification implements "end-date"
	There is now no way of telling if the search "Encounter?_sort=-end-date" is a search for the custom parameter in ascending order or the specification parameter in descending order.



One could make the argument that upgrading FHIR server versions should be done such that clients are migrated to avoid this behaviour. If that is the case, there is no need to stipulate that the server should prefix custom search parameter codes with a hyphen. 

Suggested solution options:

	recommend a different prefix character (e.g. underscore, `_`)
