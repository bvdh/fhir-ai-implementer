# FHIR-49732: Discrepancy between "nolist" and "unlist" for CSS class name when styling XHTML

## Metadata

- **Assignee**: Unassigned
- **Created**: 2/6/25
- **Description**: The page https://www.hl7.org/fhir/R5/narrative.html#css defines CSS classes for use in XHTML narratives.
The text of this page defines a class named "unlist".
However, the stylesheet that this page links to (https://www.hl7.org/fhir/R5/fhir-runtime.css) names this class "nolist" instead.

As a result, there's a discrepancy about which class name to use. This is also why the rendering for "unlist"/"nolist" is incorrect at https://www.hl7.org/fhir/R5/narrative-examples.html#2.1.4.1.2.3

The backwards-compatible fix would be to define both classes in both places.
