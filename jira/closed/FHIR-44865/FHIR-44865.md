# FHIR-44865: Validate should also be able to take a GraphDefinition

## Metadata

- **Assignee**: Unassigned
- **Created**: 3/4/24
- **Description**: Should add a new parameter called graph of 0..1 canonical(GraphDefinition).  Description: "Indicates that the referenced resource should be treated as the 'root' as the specified graph, validating all references for the resource to ensure they follow the rules.
- **Issue Type**: Change Request
- **Key**: FHIR-44865
- **Related Artifact(s)**: BaseResource
- **Related URL**: https://build.fhir.org/resource-operation-validate.html
- **Reporter**: Lloyd McKenzie
- **Resolution**: Persuasive with Modification
- **Resolution Description**: Will add the parameter, but will mark it as STU and add a note that it is not widely supported and implementations of the operation are free to not support it.
- **Status**: Applied
- **Summary**: Validate should also be able to take a GraphDefinition
