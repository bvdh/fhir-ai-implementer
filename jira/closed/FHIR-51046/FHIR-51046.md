- **Description**: In the current operations page, there is the following text:

Operations may be invoked using a GET, with parameters as HTTP URL parameters, if:

	there are only simple input parameters - i.e. no complex datatypes like 'Identifier' or 'Reference', and
	and the operation does not affect the state of the server



I think we should revisit that rule to include the ability to express values as search-typed parameters (e.g., passing a reference is something we know how to do).

There are requirements when things get too complicated (e.g., if you wanted to pass multiple values to a coding) that we would want to draw boundaries around.

Also, we should fix the "may" to be a conformance "MAY" and fix the double-and typo in the list (trailing "and" in 1 and leading "and" in 2).
