- **Description**: &gt; The outcome of the kick-off request SHALL be present as the first entry in the bundle. 

What is meant with this? Does this mean that the first entry in the Bundle contain the http-response and possible OperationOutcome of the kick-off request or the result of the FHIR operation that is requested (I assume the latter).

Please clarify this in the specification.

- **Resolution**: Persuasive
- **Resolution Description**: h3. Original Text
{quote}The outcome of the kick-off request SHALL be present as the first entry in the bundle.
{quote}
h3. Proposed Update
{quote}The outcome of the requested FHIR operation SHALL be present as the first entry in the bundle. This entry contains the final result of processing the operation that was initiated by the kick-off request, including:
 * The operation's success/failure status in {{Bundle.entry[0].response.status}}
 * Any error details in {{Bundle.entry[0].response.outcome}} (as an OperationOutcome resource)
 * The created/modified resource location in {{Bundle.entry[0].response.location}} when applicable
 * The actual resource in {{Bundle.entry[0].resource}} when the server includes a resource body in the response (determined by server policy and/or client-supplied prefer header)

Note that this is different from the initial kick-off request's response (which only returns 202 Accepted). This entry represents the final outcome of the requested operation after its asynchronous processing has completed.
{quote}
