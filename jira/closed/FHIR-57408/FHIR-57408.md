# FHIR-57408: We need to resolve the problem on SampledData.offsets running out

## Metadata

- **__EMPTY**: Bas van den Heuvel
- **__EMPTY_1**: Public
- **__EMPTY_2**: https://github.com/HL7/fhir/pull/4126
- **Assignee**: Unassigned
- **Comments**: 6/4/26
- **Created**: 5/20/26
- **Description**: There's no rule that there has to be enough offsets for the data. Either we should explicitly say that there must be, or remove the interval related invariant, and say that the interval applies if the offsets run out
- **Issue Type**: Change Request
- **Key**: FHIR-57408
- **Related Page(s)**: Datatypes
- **Reporter**: Grahame Grieve
- **Resolution**: Persuasive with Modification
- **Resolution Description**: We will remove the constraint.  Will indicate that if there are both offsets and an interval, the interval is used once the offsets run out.  If there are offsets and no interval, it is an error if the number offsets and the number of data points aren't the same.  If the editors are able to define FHIRPath that enforces this and is reasonably performant, they'll do so
- **Status**: Applied
- **Summary**: We need to resolve the problem on SampledData.offsets running out
