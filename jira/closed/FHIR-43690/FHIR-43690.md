# FHIR-43690: dom-3 invariant is inefficient and also has a duplicate expression inside

- **Description**: (Initially discovered in R4, but is still same in R6)

The current invariant expression is this:

contained.where(
    (
        ('#'+id in (
            %resource.descendants().reference | %resource.descendants().ofType(canonical) | %resource.descendants().ofType(uri) | %resource.descendants().ofType(url)
             )
        ) or descendants().where(reference = '#').exists() or descendants().where(
            ofType(canonical) = '#').exists() or descendants().where(
               ofType(canonical) = '#').exists()
            ).not()
        ).trace('unmatched', id).empty()

 

The fragment: descendants().where(ofType(canonical) = '#').exists() 
is repeated in the section testing if the contained resource references the container.

Also, this part of the expression does not check the alternate types uri/url as is in the earlier section.

Suggest that this expression be updated to a more efficient:

contained.where(
 (
    ('#'+id.trace('id') in 
      %resource.descendants().select(reference | as(canonical) | as(uri) | as(url))
    )
    or descendants().where(reference='#' | as(canonical)='#' | as(uri)='#' | as(url)='#').exists() 
 ).not()
).trace('unmatched', id).empty()
