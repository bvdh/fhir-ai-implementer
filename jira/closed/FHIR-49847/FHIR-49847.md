# FHIR-49847: Clarify there is no expectation of automatically retrieving item.definition

## Metadata

- **Assignee**: Unassigned
- **Created**: 2/20/25
- **Description**: The Questionnaire page talks about using item.definition to point to an ElementDefinition.  It should be clarified that there is not expectation of a Questionnaire renderer automatically pulling in that data.   (It might point to the SDC $assemble operation).
- **Grouping**: [SDC-CORE]
- **Issue Type**: Change Request
- **Key**: FHIR-49847
- **Related Artifact(s)**: Questionnaire
- **Reporter**: Paul Lynch
- **Resolution**: Persuasive
- **Resolution Description**: Will insert a sentence after the period here: "provides details for the item. If a definition is provided" (i.e. after item and before 'If')

"The SDC guide has defined an [assembly] process to support retrieving and injecting information from definitions into a Questionnaire.  In most cases, systems rendering forms can't be expected to do this automatically as part of form display without a separate workflow step."
- **Status**: Applied
- **Summary**: Clarify there is no expectation of automatically retrieving item.definition
- **Work Group**: FHIR Infrastructure

- **Jira**: https://jira.hl7.org/browse/FHIR-49847
