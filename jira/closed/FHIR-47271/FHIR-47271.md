# FHIR-47271: Make the trigger that decides the async pattern used more clear

- **Description**: Both async patterns are triggered by the "Prefer: respond-async" http-header. What pattern is used depends (I think) on whether the _outputFormat field is present (if present and set to application/fhir+ndjson) async-bulk is used.

We should make this trigger more clear in both async-bulk and async-batch pages. We should also explain (or prohibit) what happens is the _outputFormat field is present but holds a different value (e.g. application/fhir+json). Does this trigger the async-bundle method?
