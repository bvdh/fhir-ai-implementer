# FHIR-50981: Clarify delimiters for Address.text

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/27/25
- **Description**: Address.text is the "entire address as it should be displayed".  However, address info is pretty universally multi-line, and Address.text doesn't provide any guidance on what delimiter should be used to join the multiple lines of the address.

 

Based on Address.text -delimiting lines Grahame suggests the delimiter should be \n.  This isn't currently written down anywhere that I can see, so this ticket recommends updating the Address.text definition or comments to indicate that \n should be used.
- **Issue Type**: Change Request
- **Key**: FHIR-50981
- **Related Page(s)**: Datatypes
- **Related URL**: https://build.fhir.org/datatypes.html#Address
- **Reporter**: Cooper Thompson
- **Resolution**: Persuasive
- **Resolution Description**: Will update the definition from:
{quote}
Specifies the entire address as it should be displayed e.g. on a postal label. This may be provided instead of or as well as the specific parts.
{quote}
to
{quote}
Specifies the entire address as it should be displayed e.g. on a postal label, using a newline delimiter for each line (e.g., {{\n}}). This may be provided instead of or as well as the specific parts.
{quote}
- **Status**: Applied
- **Summary**: Clarify delimiters for Address.text
