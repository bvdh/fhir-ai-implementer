- **Description**: Hello, we have some questions regarding the use of the 'SHOULD always' formulation in the _elements description: 'Servers SHOULD always return mandatory and modifier elements whether they are requested or not.'

The word 'always' right after "SHOULD" is never found in the description of other modifying search results elements, so we are unsure how to interpret it.

We discussed two different interpretations regarding the HL7 FHIR recommendation :

	it is not mandatory to return mandatory and modifier elements. However, if a server chooses to do so for a particular resource, it should consistently apply this practice for every resource.
	It is mandatory for the server to return mandatory and modifier elements.



Which of these two interpretation is the one to apply ?

Thanks

- **Resolution**: Persuasive with Modification
- **Resolution Description**: Before the paragraph starting "Servers are not obliged to return just the requested elements." will add:

"This parameter is allowed to be used for interactions other than 'search'.  See the [REST page](http.html#parameters)."

Will change "Servers SHOULD always return mandatory and modifier elements whether they are requested or not." to

"Servers SHOULD return mandatory and modifier elements whether they are requested or not."

 

While consistency is certainly a good thing, the use of 'always' was not intended to suggest that it's more important here than anywhere else.  We're removing the word to avoid confusion.
