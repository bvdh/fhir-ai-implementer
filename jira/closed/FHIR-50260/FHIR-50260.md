- **Description**: It is unclear whether SampledData.lowerLimit and SampledData.upperLimit are to be interpreted as scaled values or as original non-scaled values. The text needs to clarify this.
The assumption I and others made is that these values are not scaled and that the interval &#91;lowerLimit..upperLimit&#93; represents all possible measured values for a sample.

Proposed text:

The lowerLimit, when present, represents the lowest possible measured value and is not scaled. Similarly the upperLimit, when present, represents the highest possible measured value and is not scaled. The range &#91;lowerLimit..upperLimit&#93; can be used  by visualisation tools for scaling when representing sampledData graphically.

 

And although submitted on FHIR R4, this is still applicable in R6 as well.

 

And on a side note: the use of dimensions in R4..R6 is rather limited. All dimensions will have the same unit, origin, scale factor and limits. Ideally these are defined per dimension.
