- **Description**: https://jira.hl7.org/browse/FHIR-47643 raised the issue that JSON parsing might break if someone changes the maximum cardinality of a field.

During discussion, it was noted that this also affects FHIRPath, and resources that use FHIRPath, because suddenly you get either multiple resources when expecting 1 or 1 resource when expecting multiple.  There should be some warning that FHIRPath expressions need to be reviewed when moving to a new version of FHIR.
