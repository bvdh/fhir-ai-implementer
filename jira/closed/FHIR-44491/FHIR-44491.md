### 2. Elliot Silver - 2/12/24
Security: Public

The question is about example responses which are of type Binary, rather than example resources conveyed in a Binary. For example, if the content is a Binary of a JPEG, rather than an example JPEG. Does it become double wrapped (JPEG in Binary in Binary)?

Regarding 2, “must be a resource we can include in an IG package”:  Where is that stated/implied? Why is that necessary? Is that bloating the IG package unnecessarily?

### 3. Lloyd McKenzie - 2/12/24
Security: Public

A binary resource is just a Binary instance.  No double-wrapping.  In your source, you'd just have the raw JPEG file.  In the package, you'd have a Binary instance with the base64-encoded JPEG.
