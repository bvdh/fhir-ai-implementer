- **Description**: Outcome of the FHIR Connectathon Testing Track in Phoenix:

As discussed with Richard Ettema, there is currently no possibility to reference from a testCase in the Testplan Resource to a single statement of a Requirement Resource which would be beneficial for TestPlan.

 

The proposal therefore is to change Testplan.testcase.scope into a backbone element with at least two elements:

	Artifact 0..1 of type Reference(Any)        "The scope or artifact covered by the case"
	Description 0..1 type markdown   "Requirement statement which is covered by this test case (&#91;url#&#93;key)"
