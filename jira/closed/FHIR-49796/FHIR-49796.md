# FHIR-49796: Adding Hebrew (he) to "Common Languages" valueset-languages

## Metadata

- **Assignee**: Unassigned
- **Created**: 2/13/25
- **Description**: In the current value-set, Hebrew (he) language is missing. 

As we are in the middle of the FHIR national regulation and certification process we must have this code for all Israeli health eco-systems that implement FHIR.

Also, we would like to stress that It is crucial that this change will affect R4 version, and all future version.

Thanks and best regards
- **Issue Type**: Change Request
- **Key**: FHIR-49796
- **Related Artifact(s)**: ValueSet
- **Related Section(s)**: http://hl7.org/fhir/ValueSet/languages
- **Related URL**: https://hl7.org/fhir/R4/valueset-languages.html
- **Reporter**: Erez Sahlom
- **Resolution**: Persuasive with Modification
- **Resolution Description**: Will add this to the common list of languages for R6+.  We can't update 4.0.

Note that the list of common languages in R4 is a "starter set".  There's a maximum value set that covers all language codes in IETF, so it's perfectly fine to use "he" in R4, it just won't show up in dropdowns that use that value set.  You're also free to update your binding in your profiles to be a value set that includes the "he" code for use in your own dropdowns and avoid confusion in your implementer community.
