- **Description**: Need to explain how to use search derivation to add 'common' search parameters to existing resources, including additional resources.  E.g. There's a new resources that has a 'subject' element, but isn't one of the resources supported by the standard '_patient' search parameter - how do we let you search by _patient there in a way that will treat that resource as being part of the Patient compartment?
- **Issue Type**: Change Request
- **Key**: FHIR-50846
- **Related Artifact(s)**: SearchParameter
- **Reporter**: Lloyd McKenzie
- **Resolution**: Not Persuasive with Modification
- **Resolution Description**: We don't actually need to do anything here.  So long as the SearchParameter is exposed in a CapabilityStatement on a server with the same 'code', it can be used alongside the standard multi-resource search parameter for system level searches.  Once an additional resource migrates to be a standard resource, we can deprecate the additional resource-specific search parameter and add the new 'official' resource to the standard search parameter.

 

We will add documentation to the Additional Resources page explaining this.
