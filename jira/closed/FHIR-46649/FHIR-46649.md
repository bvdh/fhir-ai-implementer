# FHIR-46649: Clarify sensitive content conformance for links

- **Created**: 8/16/24
- **Description**: In Search &gt; Other Links - Paging, there was trial use conformance language added about sensitive information in links:

&gt; Servers SHOULD NOT include content that is considered sensitive in the URLs.

This was intended to apply to the next and prev links, and clearly cannot apply to the self link.

Recommend clarifying the language to specifically exclude the self link.
- **Issue Type**: Change Request
- **Key**: FHIR-46649
- **Related Page(s)**: Search
- **Reporter**: Gino Canessa
- **Resolution**: Persuasive
- **Resolution Description**: Will update
> Servers SHOULD NOT include content that is considered sensitive in the URLs.

to 

> Servers SHOULD NOT include content that is considered sensitive in URLs, excluding the {{self}} link.
- **Status**: Applied
- **Summary**: Clarify sensitive content conformance for links
