# FHIR-50848: Explain how to extend compartments to handle 'additional' resources

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/12/25
- **Description**: If someone defines a new patient-specific resource, we're going to want it to be handled as part of the Patient Compartment - but only HL7 is allowed to define compartments.  How will this work?
- **Issue Type**: Change Request
- **Key**: FHIR-50848
- **Related Artifact(s)**: CompartmentDefinition
- **Reporter**: Lloyd McKenzie
- **Resolution**: Persuasive with Modification
- **Resolution Description**: We acknowledge that additional resources may desire to be part of an existing compartment (e.g., the Patient compartment).  However, changes to compartment definitions have signficant impact on some categories of implementations and there is a strong desire to ensure the changes are infrequent and throughly vetted.

We will add a complex extension for new resources to allow the definition to indicate the resource is a candidate for inclusion in a compartment:
* Extension: {{additional-resource-compartment}} {{0..*}}
* slice: {{compartment}}: {{1..1}} {{canonical}} - Reference to the compartment this resource would like to be included in
* slice: {{param}} {{1..*}} {{string}} - Search Parameter Name, or chained parameters
* slice: {{documentation}} {{0..1}} {{string}} - Additional documentation about the resource and compartment
* context: {{StructureDefinition}}
Servers will have discretion on supporting the resource in a compartment when adding support for the resource.

In order to become part of the HL7-defined compartments, a resource must be promoted into FHIR Core. Once included, the compartment additions will be considered for inclusion in the next FHIR release.
