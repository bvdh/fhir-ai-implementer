- **Issue Type**: Change Request
- **Key**: FHIR-49899
- **Related Page(s)**: Datatypes
- **Related Section(s)**: https://hl7.org/fhir/datatypes-definitions.html#SampledData.offsets

- **Reporter**: Ardon Toonstra
- **Resolution**: Persuasive
- **Resolution Description**: Change the beginning of the definition of Sampledata.offsets from:

A series of data points which are decimal values separated by a single space (character u20).

to:
A series of increasing decimal values separated by a single space (character u20), which represent the offset from the logical start point. Offset values may be negative.


Also, to the section https://build.fhir.org/datatypes.html#SampledData add the requirement that expressed in the constraint (currently shows on the definition page, but not the narrative):
A SampledData SHALL have either an interval and offsets but not both	interval.exists().not() xor offsets.exists().not()
- **Status**: Applied
- **Summary**: Finetune specification of SampledData.offsets to clarify they are relative to initial value
