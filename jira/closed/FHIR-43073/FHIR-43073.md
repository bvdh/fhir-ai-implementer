# FHIR-43073: Clarify chaining language for references

## Metadata

- **Assignee**: Gino Canessa
- **Created**: 11/7/23
- **Description**: Ticket filed based on this Zulip Thread.

Generally, there is some confusion about the search type reference, the FHIR datatype reference and the concept of referring to another resource (e.g., "a reference").  This is a request to review content and clarify the differences, for example in the sections:

	Special Search Conditions: Canonical Identifiers and
	Chaining.
- **Grouping**: [AwaitingMerge]
- **Issue Type**: Change Request
- **Key**: FHIR-43073
- **Related Page(s)**: Search
- **Reporter**: Gino Canessa
- **Resolution**: Persuasive with Modification
- **Resolution Description**: * Review the search page references (with special attention to the referenced sections) and clarify instances of "reference" to be "search type reference", "FHIR datatype Reference", or "a reference to another resource".
 * Add a base section for Basic Search Types with a table showing the allowed search type -> FHIR type mappings (after "Escaping Search Parameters", before "date").
