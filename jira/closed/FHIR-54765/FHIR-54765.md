- **Description**: Inclusion of images is a valuable way to enrich the content of medical documents. The current specification allows this by referring from the Narrative to contained Binary/DocumentReference resources.

The specification should also explain in what way such images can include references to the the content that provided (part) of the content in the image. As documentation of the source of the info and/or to allow inclusion of hyperlinks to this content when rendering the content.
- **Issue Type**: Change Request

- **Resolution Description**: To the narrative page [Image Reference Section|https://build.fhir.org/narrative.html#id]

Image source data (the {{src}} attribute) may refer to an image found in the resource (as a contained [DocumentReference|https://build.fhir.org/documentreference.html] or [Binary|https://build.fhir.org/binary.html] resource) by its {_}id{_}:

 

Add "or by its fullUrl if contained within a document bundle"

 

And we will add an example of what this looks like and an implementer note saying that is ahs to be the same bundle and point out the need to track the resources to be able to rebuild the narrative.
