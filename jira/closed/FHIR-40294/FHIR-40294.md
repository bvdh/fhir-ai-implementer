- **Description**: MessageHeader.event (this can be defined by a GraphDefinition resource) should be 

MessageHeader.event (this can be defined by a MessageDefinition resource) or EventDefinition resource?

Is this still applicable when the event point to an EventDefinition?

	The message header has an event.system of urn:ietf:rfc:3986
	The event.code is the URL from the operation definition OperationDefinition.url



example:
       &lt;eventCoding&gt;
          &lt;system value="urn:ietf:rfc:3986"/&gt;
          &lt;!-- value set expansion --&gt;
          &lt;code value="http://hl7.org/fhir/OperationDefinition/ValueSet-expand"/&gt;
        &lt;/eventCoding&gt;
{{}}

GraphDefinition appears inside the MessageDefinition, so wouldn't be referenced here.  EventDefinition doesn't provide a (computable) description of the resources to be transmitted.
