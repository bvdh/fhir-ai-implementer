# FHIR-31583: Document how jurisdiction is constrained in packages

## Metadata

- **Assignee**: Grahame Grieve
- **Created**: 3/22/21
- **Description**: If an ImplementationGuide that has no value for jurisdiction is derived from another that has a jurisdiction, the validator raises a warning e.g.
The resource should declare its jurisdiction to match the package id (hl7.fhir.au.dh.medicare-records, jurisdiction = urn:iso:std:iso:3166#AU) (for FSH: 'jurisdiction: urn:iso:std:iso:3166#AU "null"')
This behaviour not described in the ImplementationGuide docco.

It should be documented.

See https://chat.fhir.org/#narrow/stream/179252-IG-creation/topic/Jurisdiction.20of.20packages for a Zulip thread. 

See https://github.com/HL7/fhir-ig-publisher/issues/246 for a related overenthusiasm in the validator
