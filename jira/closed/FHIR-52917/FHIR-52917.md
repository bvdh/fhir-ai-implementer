# FHIR-52917: Prevent derived profiles of complex extensions from adding non-fullURL slices

## Metadata

- **Assignee**: Unassigned
- **Created**: 9/18/25
- **Description**: In a complex extension, the allowed children are defined using slicing.

We can't have closed slicing because there's a need to allow full URL extensions (e.g. data absent reason) in the complex extension.  However, we don't want child profiles to add new 'local' extensions, because the meaning of the complex extension has to come from the same profile where the URL is established.  Without that, we could have multiple child profiles each defining their own (possibly colliding/conflicting) local children.
