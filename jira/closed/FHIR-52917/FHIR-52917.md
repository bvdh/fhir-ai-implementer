# FHIR-52917: Prevent derived profiles of complex extensions from adding non-fullURL slices

## Metadata

- **Assignee**: Unassigned
- **Created**: 9/18/25
- **Description**: In a complex extension, the allowed children are defined using slicing.

We can't have closed slicing because there's a need to allow full URL extensions (e.g. data absent reason) in the complex extension.  However, we don't want child profiles to add new 'local' extensions, because the meaning of the complex extension has to come from the same profile where the URL is established.  Without that, we could have multiple child profiles each defining their own (possibly colliding/conflicting) local children.
- **Issue Type**: Change Request
- **Key**: FHIR-52917
- **Related Page(s)**: Extensibility
- **Reporter**: Lloyd McKenzie
- **Resolution**: Persuasive
- **Resolution Description**: On the extensibility page under the bullet that ends with "Except for child extensions defined within complex extensions, the URL SHALL be an absolute URL.", will add a new bullet:

"Complex extensions SHOULD have 'open' slicing on the extension element to allow use of other full URL extensions within the complex extension.  However, if creating a derived profile on a complex extension, that adds additional extension slices, those slices SHALL have full URLs.  This is because derived profiles are not defining the extension - i.e. they are not establishing the 'url' value for the extension."
- **Status**: Applied
- **Summary**: Prevent derived profiles of complex extensions from adding non-fullURL slices
- **Work Group**: FHIR Infrastructure

- **Jira**: https://jira.hl7.org/browse/FHIR-52917
