- **Description**: Ensure that the definitions of the standards levels on this page are consistent with the definitions of the http://hl7.org/fhir/standards-status code system.

Add clarification, to both this page and the code system, that for in-progress content (such as the CI Build or a ballot release), the standards status should be the status in the published version (e.g., STU1-ballot1 will show the status we want of the resource in STU1 publication).

Other changes to this section:

	Move "Note that this version of the specification has NOT been submitted to ANSI for consideration as a normative standard." from the description of the standard level. This is not part of the definition of the level, but a description of the spec. Put it elsewhere in the section.
