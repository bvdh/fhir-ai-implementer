- **Description**: The output field of the result should have an optional filesize attribute that provides alternative feedback (of count) to clients on the file size and the amount of resources required to parse the result.
- **Issue Type**: Change Request
- **Key**: FHIR-47469
- **Related Page(s)**: Asynchronous API (async)
- **Related URL**: https://build.fhir.org/async-bulk.html
- **Reporter**: Bas van den Heuvel
- **Resolution**: Persuasive
- **Resolution Description**: Original properties on output:
- count - the number of resources in the file, represented as a JSON number.

New:
- count - the number of resources in the file, represented as a JSON number.
- filesize - the size of the file in bytes, represented as a JSON number. This provides clients with information about the storage and processing requirements for downloading and parsing the file.

(These are both optional, per the intro of "Each file item MAY optionally contain the following field:")
