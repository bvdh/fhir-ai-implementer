- **Description**: In the Replacing hyperlinks and full-urls section of transaction processing, there are some rules that appear to fall somewhere between quite challenging to implement correctly and not possible to programmatically implement.

Specifically, I was implementing this feature in order to process some Synthea bundles and noticed that every entry.fullUrl in the bundle use the same guid literal as an identifier within the resource for that entry.  For simplicity, consider a bundle with a Patient:

	entry.fullUrl = a
	entry.resource.identifier = http://synthea|a



Further, many of the references between resources use reference.identifier instead of reference.reference.

This means that updating references in the narrative requires an understanding on whether a reference to a is a literal reference to the resource or a reference to an identifier of the resource.

Looking through several OSS implementations of this feature, I did not find any that had implemented this correctly.  Either the implementation did not replace the narrative (currently required with a SHALL) or broke identifiers (e.g., doing string replacement on a serialized representation).

I realize these are normative rules, so I am not sure the best path forward.
