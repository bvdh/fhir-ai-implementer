# FHIR-40228: ConformanceExpectation missing code in code system

- **Description**: The codeSystem for ConformanceExpectation (and therefore the value set http://build.fhir.org/valueset-conformance-expectation.html as well that uses it) is missing the code for "SHALL-NOT", although is is part of the conformance language, see http://build.fhir.org/conformance-rules.html#conflang .
- **Grouping**: [R6]
- **Issue Type**: Change Request
- **Key**: FHIR-40228
- **Related Page(s)**: Conformance Rules
- **Related URL**: http://build.fhir.org/codesystem-conformance-expectation.html
- **Reporter**: Reinhard Egelkraut
- **Resolution**: Persuasive with Modification
- **Resolution Description**: First, we have duplicates of the code system and value set in both the FHIR spec and THO with different canonical URLs
[http://terminology.hl7.org/CodeSystem/conformance-expectation]
[http://hl7.org/fhir/conformance-expectation]

[http://terminology.hl7.org/ValueSet/conformance-expectation]
[http://hl7.org/fhir/ValueSet/conformance-expectation]

So will work with THO to clean that up. 

Agreed there are use cases for SHALL-NOT in the Requirements resource, and possibly some use cases for a new modifier extension that could be used in places such as rest.resource.interaction, rest.resource.search-param, rest.resource.operation, rest.interaction, rest.search-param, rest.operation

We will add SHALL-NOT to the code system, and the current value set will inherit it. Will then create a new value set called capability-expectation for the current extension that does not contain the SHALL-NOT code so that it will not need to become a modifier extension. 

Lastly, will create a new modifier extension called "prohibited" that can be used in the more limited contexts listed above.
