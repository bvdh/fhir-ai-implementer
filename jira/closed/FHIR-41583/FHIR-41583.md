- **Description**: There have been extensive discussions coordinated across HL7 Work Groups (BRR, CDS, CQI, LHS) and related activities (Vulcan RWD, Vulcan SOA, HEvKA/EBMonFHIR, Cohort Definition Connectathon Tracks) with consistent recommendations to provide a single method across many use cases in FHIR for defining the set of characteristics which represent the inclusion and exclusion criteria to be considered a member of a group, including groups of many types of entities (not necessarily specific to entities commonly represented with FHIR Resources).  Having worked through element-level discussions in other spaces (including [Changes to Group for CohortDefinition and CohortDefinition Profile of Group Proposal ) the proposed changes to Group are:

	Add most of the elements from the MetadataResource Interface to Group. (up to 23 elements to add)

	Add 'conceptual' as a third choice to Group.membership 1..1 code
