# FHIR-50120: Two questions regarding the Reference data type

## Metadata

- **Assignee**: Unassigned
- **Created**: 3/26/25
- **Description**: Two questions regarding the Reference data type.

1. Should Reference.reference(a literal reference) and Reference.identifier(a logical reference) be mutually exclusive to each other in each instance, or can they coexist within a single Reference datatype instance.
If they are required to be mutually exclusive to each other, could we enforce this constraint by introducing an invariant in the Reference datatype, such as:
 
 not(reference.exists() and identifier.exists())
 
2. Is it appropriate to use a  Canonical identifier within Reference.identifier?
  If it is allowed, may we specify it explicitly in the description of Reference.identifier?

reference:
https://www.hl7.org/fhir/references.html


Some resource types have a defined element url which is the 'canonical URL' that always identifies the resource. This is a special kind of Business Identifier.

The canonical URL serves as a stable logical identifier for the resource
https://www.hl7.org/fhir/resource.html#identifiers
