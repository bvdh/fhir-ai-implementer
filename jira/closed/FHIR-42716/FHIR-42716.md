3.5.6.1 Invoking Search via Messages 

To perform a search with messaging that mirrors REST functionality, the message should leverage a Parameters instance with a standard parameter name of "url" and a valueString that corresponds to the search string that would typically be to the right of the base URL. E.g. if in REST you would say http://someserver.org/fhir/Patient?name=joecode&gt;, then the parameter would have a value of Patient?name=joe.
