# FHIR-7710: 2015May core #1001 - Clarify relationship agents and entities used in activity

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/10/15
- **Description**: Existing Wording: Provenance.agent.role - Definiton: The function of the agent with respect to the activity. Binding: ProvenanceAgentRole: Extensible: See http://hl7.org/fhir/vs/provenance-agent-role (The role that a provenance agent played with respect to the activity) such as author (resource), performer (activity), enterer (entering data into system), attester (resource), verifier (activity), informant (resource)

         Comment:

         The 1.23.2.1.269  Value Set http://hl7.org/fhir/vs/provenance-agent-role used in the Provenance.agent.role (Extensible) is clearly a mixed set of agents that are either involved with an activity or involved with a Resource. The enterer agent is a role interacting with a system.  How are these roles that are participating in different things (activities as in processes, Resource as in the documentation of the agent's contribution to the Resource content, and data entry as a particular type of activity) supposed to be participating in an unnamed activity - other than an activity that is a specialization of a ControlAct?



         Relationship among the Provenance Resource as an Activity with agents and the relationship between agents and entities used in the activity that resulted in the target Resource need to be clarified.
