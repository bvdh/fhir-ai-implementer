- **Description**: The composition and message search parameters on Bundle are not valid.

They are each defined as reference type, but the FHIRPath expressions return resources and not references.

The ability to target a Resource via a reference search type is not defined on the search page.  Interactions with the various input formats, modifiers, etc., would need to be defined.  Additionally, the definitions around chaining in search would need to be updated to allow for traversal through resource types.

It might also be possible to update the FHIRPath expressions, but most paths in that direction I believe also require additional definitions (e.g., creating a new reference object based on the resource or pointing the expression at Bundle.entry.fullUrl).

We could also change the search parameters to special search type (as they require custom code) or add an extension to indicate this 'special' behavior.
