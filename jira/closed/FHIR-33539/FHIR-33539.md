- **Description**: The SDC IG has defined and pre-adopted a number of FHIRPath variables and functions that SHOULD be defined as part of the core FHIR spec FHIRPath extensions.  They can be found here: https://build.fhir.org/ig/HL7/sdc/expressions.html#fhirpath

Specifically, we should define the %questionnaire, %qitem, answers(), and ordinal() variables and functions

 # Add %questionnaire and %qItem and answers() to the Core FHIR Spec's QuestionnaireResponse page at [https://build.fhir.org/questionnaireresponse]
 # Add a note referring to (2) and to [https://build.fhir.org/subscriptiontopic.html#resource-trigger-update] in [https://build.fhir.org/fhirpath.html]

### 3. Brian Postlethwaite - 8/13/24
Security: Public

My 2c worth - The SDC spec is the right place for the SDC variables to be defined - it provides the context that is appropriate for these variables, and how they fit with everything else, and when you might be expected to use these.

We could however indicate that systems that implement FHIRPath engines should have mechanisms to be able to provide variables to the engine for evaluation, and possibly include a reference to the SDC spec for examples of this.

Also could highlight that custom functions may also need to be able to be injected to permit users of an engine to customize functionalities.
