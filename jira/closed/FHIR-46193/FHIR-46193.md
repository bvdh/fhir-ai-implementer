# FHIR-46193: Why is patch format all mime types

## Metadata

- **Assignee**: Unassigned
- **Created**: 6/20/24
- **Description**: The spec only recognizes 3 patch syntaxes - JSON, XML.  We should be limiting to the 5 codes allowed as a required binding (application/fhir+xml, application/fhir+json, application/fhir+turtle, application/json-patch+json, application/xml-patch+xml).  I think this is a technical correction because the allowed set is already defined in the spec, so it's not possible to use others and be conformant.

If new formats are introduced in a future version of FHIR, it's possible to update the value set.
