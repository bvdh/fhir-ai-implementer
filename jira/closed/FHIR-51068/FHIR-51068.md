Summary:

Should the period [2015-04-13T20:27:01-04:00, 2015-04-13T20:42:01-04:00) be included for the includsive search query le2015-04-13T20:27:01-04:00?

There are two different approach here.

1st approach follows the defintion of LE prefix in the &#91;serach prefix section&#93;(https://build.fhir.org/search.html#prefix) which states

"the range below the parameter value intersects (i.e. overlaps) with the range of the resource value or the range of the parameter value fully contains the range of the resource value"

	the range below the search value, which is (-infinite, 2015-04-13T20:27:01-04:00), does not intersect with the target period [2015-04-13T20:27:01-04:00, 2015-04-13T20:42:01-04:00);
	the search value [2015-04-13T20:27:01-04:00, 2015-04-13T20:27:02-04:00) does not fully contain the target period [2015-04-13T20:27:01-04:00, 2015-04-13T20:42:01-04:00);
	Since both LT and EQ check failed, the resource shall not be included in the LE search.



The 2nd approach, as described in the thread,

	Combines the search ranges, which create the combined search range as (-infinite, 2015-04-13T20:27:01-04:00],
	Check if the combine search range intersects with the target range
	Since there is one second intersection at the search range's higher boundary, the resource shall be included in the LE search



I would like this difference to be clarified and documented. If we could not agree on the approaches, we should deprecate LE and GE comparator for range searches and looking for other comparator, such as "start before" or "end before"
