# FHIR-55151: CapabilityStatement.fhirVersion has no escape valve

## Metadata

- **__EMPTY**: Lloyd McKenzie
- **__EMPTY_1**: Public
- **__EMPTY_2**: Grahame Grieve will look at if we can use a binding mechanism similar to language/mime/UCUM to allow for future versions.
- **Assignee**: Unassigned
- **Comments**: 2/9/26
- **Created**: 1/20/26
- **Description**: The element CapabilityStatement.fhirVersion is a mandatory element with a required binding, but appears to have neither an 'unknown' or 'other' concept, which can make the containing element impossible to populate in some cases. Please add 'unknown', 'other', or some equivalent escape valve concept to the value set.

(Comment 117 - imported by: Lloyd McKenzie)
- **Issue Type**: Change Request
- **Key**: FHIR-55151
- **Related Artifact(s)**: CapabilityStatement (Conformance)
- **Related URL**: http://build.fhir.org/capabilitystatement.html
- **Reporter**: Lloyd McKenzie
- **Resolution**: Not Persuasive with Modification
- **Resolution Description**: Will will adjust this binding to be version-independent, which will make the problem go away.  We will advise tool-smiths to behave as though prior releases of FHIR also bound in a version-independent manner.
