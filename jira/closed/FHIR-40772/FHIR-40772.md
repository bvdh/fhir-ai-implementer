# FHIR-40772: Allow If-Match Usage on Delete Interactions

## Metadata

- **Assignee**: Gino Canessa
- **Created**: 4/6/23
- **Description**: Currently, the REST interactions/HTTP section allows servers and clients to use ETag values + If-Match headers to manage resource contention/prevent race conditions. However, the summary table of interactions (3.2.0.17) allows If-Match headers on update and patch requests, but not delete requests.

The same issue exists for deletes - conflicting updates/deletes may arrive at the same time, the the client may act differently if it knew about the conflicting update. Consider adding If-Match headers as an option for Delete requests.
