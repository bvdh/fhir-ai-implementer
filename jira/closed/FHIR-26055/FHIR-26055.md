# FHIR-26055: STU3 Questionnaire StructureDefinition URL returns R4

## Metadata

- **__EMPTY**: Lloyd McKenzie
- **__EMPTY_1**: Public
- **__EMPTY_2**: Have confirmed that if you have an Accept header asking for XML and you ask for http://hl7.org/fhir/3.0.2/StructureDefinition/Questionnaire you end up with the R4 (version=4.0.1) Questionnaire StructureDefinition.  Looks like something is redirecting that shouldn't?  (Or at least is redirecting in the wrong way.)
- **Assignee**: Unassigned
- **Comments**: 3/5/20
- **Created**: 2/12/20
- **Description**: It was suggested to me that one might determine the version of a Questionnaire by resolving the meta.profile StructureDefinition URL and reading the FHIR version from that.  I tested that, and got the wrong FHIR version for an STU3 URL.  Note:  The browser seems to request the HTML page, and that works.

I used "curl" to resolve the URLs from the command line.  For example:

curl http://hl7.org/fhir/3.0.2/StructureDefinition/Questionnaire
  returns a redirect to http://hl7.org/fhir/STU3/StructureDefinition/Questionnaire, 
  which redirects to http://hl7.org/fhir/StructureDefinition-Questionnaire.xml,
  which returns the XML StructureDefinition, but in R4.
