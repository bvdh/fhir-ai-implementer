# FHIR-48885: Clarify applicability of RFC 8174 (updates 2119)

## Metadata

- **Assignee**: Unassigned
- **Created**: 11/22/24
- **Description**: RFC 8174 updates RFC 2119. The FHIR conformance page cites RFC 2119 but makes no mention of 8174. Recommend adding a citation to clarify the intent that conformance verbs only have meaning in uppercase.

There have been several trackers about this against individual IGs, but I did not find any against FHIR core
- **Issue Type**: Change Request
- **Key**: FHIR-48885
- **Related Page(s)**: Conformance Rules
- **Related URL**: https://build.fhir.org/conformance-rules.html#conflang
- **Reporter**: Alex Kontur
- **Resolution**: Persuasive
- **Resolution Description**: Will change "This specification uses the conformance verbs SHALL, SHOULD, and MAY as defined in RFC 2119"

to "This specification uses the conformance verbs SHALL, SHOULD, and MAY as defined in the RFC 8174 clarification of RFC 2119"
- **Status**: Applied
- **Summary**: Clarify applicability of RFC 8174 (updates 2119)
- **Work Group**: FHIR Infrastructure

- **Jira**: https://jira.hl7.org/browse/FHIR-48885
