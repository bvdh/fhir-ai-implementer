# FHIR-46404: Documentation error in R5 Search docs page.

## Metadata

- **Assignee**: Unassigned
- **Created**: 7/17/24
- **Description**: Hello there, I found an error on the current build of the FHIR spec documentation.

In R4, the `_text` Search parameter is of type `string`, as evidenced here: https://www.hl7.org/fhir/R4/domainresource.html#search

In R5 the `_text` Search Parameter is of type `special` as evidenced here: https://www.hl7.org/fhir/domainresource.html#search

However the search.html page for text incorrectly states that it is of type `string`, even on the main build site (which is currently R5), as evidenced here:  https://www.hl7.org/fhir/search.html#_text

It is also incorrect in the summary table, shown here: https://www.hl7.org/fhir/search.html#table

 

Just wanted to report this as it threw HAPI-FHIR for a loop while attempting to process `_text` in our fulltext engine.

 

Please let me know the right spot to report this if this is not the right project.

Thanks for your time,

 

--Gary
