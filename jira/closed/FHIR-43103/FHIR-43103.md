# FHIR-43103: Propose to revise the Description of the ele-1

## Metadata

- **Assignee**: Unassigned
- **Created**: 11/13/23
- **Description**: https://www.hl7.org/fhir/types.html#Element

Defined on this element
ele-1    Rule     All FHIR elements must have a @value or children    hasValue() or (children().count() &gt; id.count())

The ele-1 description might need to be revised to carry the meaning of the expression acurately.
Current:
All FHIR elements must have a @value or children

Proposed:
All FHIR elements must have a @value or children or both.

The reason of the proposed change
Any element could have extension; therefore, an element could have both value and children.
