# FHIR-43501: Group.characteristic.instances[x] needs constraints

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/4/24
- **Description**: The intent of this element is to convey the number of instances.  It uses Quantity for the use of the comparator and/or Range to convey a range of numbers.  Thus a constraint on the datatypes is needed to only allow non-negative whole numbers for Quantity and Range.low/high.

There should be constraint on the Quantity.value and Range.low.value and Range.high.value that can only be unsignedInt and the x.unit should be missing and/or the x.code should be '1'.
- **Issue Type**: Change Request
- **Key**: FHIR-43501
- **Related Artifact(s)**: Group
- **Reporter**: Jean Duteau
- **Resolution**: Persuasive with Modification
- **Resolution Description**: We will change instances to be unsignedInt|Range.  We will add an invariant on instances that says that minimum and maximum must be >= 0 and unit, code and system must be omitted.
- **Status**: Applied
- **Summary**: Group.characteristic.instances[x] needs constraints
- **Work Group**: FHIR Infrastructure
