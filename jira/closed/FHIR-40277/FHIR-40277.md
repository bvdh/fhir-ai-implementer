# FHIR-40277: Add usage element to MetadataResource pattern

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/18/23
- **Description**: The usage element is defined on all KnowledgeArtifact resources, and was identified as part of artifact metadata in the clinicalreasoning metadata topic:

https://hl7.org/fhir/clinicalreasoning-knowledge-artifact-representation.html#metadata

However, this element does not appear in the current MetadataResource pattern. Add it to the pattern:

usage 0..1 markdown Describes the clinical usage of the artifact

A detailed description of how the artifact is used from a clinical perspective
- **Issue Type**: Change Request
- **Key**: FHIR-40277
- **Related Artifact(s)**: MetadataResource
- **Related URL**: https://build.fhir.org/metadataresource.html
- **Reporter**: Bryn Rhodes
- **Resolution**: Persuasive
- **Resolution Description**: .usage isn't very clearly differentiated from .usageContext, but we can add it to the Metadata pattern and then start improving the definitions from there.

In a previous discussion we also asked how usage is distinct from purpose. It seems that purpose answers "why does it exist" and usage answers "how do I apply it clinically".

Add .usage as 0..1 markdown, as requested.
- **Status**: Applied
- **Summary**: Add usage element to MetadataResource pattern
