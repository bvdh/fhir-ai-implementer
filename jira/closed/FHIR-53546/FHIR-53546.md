# FHIR-53546: Conditional Create's "If-None-Exist" isn't transactionally enforceable

## Metadata

- **Assignee**: Unassigned
- **Created**: 11/26/25
- **Description**: The fundamental problem is that you can't solve this for arbitrary search criteria in an efficient way.

An example for Conditional Create, where there are 2 competing request, the first uses:
If-None-Exist: identifier=http://lab|123&amp;patient=Patient/456
The second uses:
If-None-Exist: identifier=http://lab|123&amp;patient.name=Smith

Both requests should logically resolve the same underlying resource. If they both happen at the same time they get different IDs and we now have duplicate resources with different IDs.

Everything from here is a workaround:

	Choose a piece of information to lock on (domain specific)
	Search -&gt; Persist -&gt; Search to ensure a duplicate wasn't created



I understand it leaves this to implementors, but this will create divergence in behavior.
