# FHIR-50614: Clarify how the type modifier should be used in SubscriptionTopic + Subscription

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/2/25
- **Description**: In both the allowed filters section in SubscriptionTopic (SubscriptionTopic.trigger.canFilterBy.modifier) and the in-use filters section in Subscription (Subscription.filterBy.modifier), we have the modifier `type` allowed. How would I use that modifier though/how would I specify the resource types I'd like to use in the modifier?

 

i.e. the type modifier typically constrains a reference to a single resource type, and is the resource type itself rather than the constant type (e.g. subject:Patient=ABC, rather than subject:type=ABC). Where would I specify the resource type that the filterParameter should be constrained to?
