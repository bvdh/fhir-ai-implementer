- **Assignee**: Unassigned
- **Comments**: 5/20/24
- **Created**: 5/6/24
- **Description**: Currently the `Group.duration` element is a choice with `Duration` and `Range` options.  Given the description of the element (Length of time in which the characteristic is met), I am not sure what can be usefully stated in a `Range` that is not possible to express in a `Duration`.

Removing `Range` would also simplify the name from `durationDuration` to `duration`.

### 4. BrianAlper - 8/13/24
Security: Public

The Group.characteristic.duration&#91;x&#93; element has nothing to do with Period concepts, start dates, or end dates.   Also nothing to do with timing&#91;x&#93; or RelativeTime.

 

The Group.characteristic.duration&#91;x&#93; element is for describing the duration for which the characteristic must be met.  Examples are &lt; 15 minutes, 3-6 hours, &gt; 3 days.   These durations can be expressed with a Quantity datatype (Duration datatype as limited to time) such as &lt; 15 minutes or &gt; 3 days, or with a Range datatype such as 3-6 hours.

No change is warranted.

### 7. BrianAlper - 8/14/24
Security: Public

Group.characteristic.duration&#91;x&#93; is an 0..1 so it is an optional modifier to Group.characteristic which has code 1..1 and value&#91;x&#93; 1..1 for the 'criteria' or 'characteristic definition'.   Group.characteristic.duration&#91;x&#93; is defined as "Length of time in which the characteristic is met".   If you have a characteristic of "had a headache in the past 3 weeks" you would not use Group.characteristic.duration&#91;x&#93;, but if you wanted to add "lasting &gt; 15 minutes" so you are describing "had a headache lasting &gt; 15 minutes in the past 3 weeks" you would than have Group.characteristic.durationDuration with a value of '&gt; 15 minutes' — in this case there is a minimum but no maximum.   If you expressed a singular Quantity without a comparator, you may need to use a Range instead to avoid wrong precision interpretation.

### 8. Lloyd McKenzie - 8/14/24

### 9. BrianAlper - 8/15/24
Security: Public

For common usage, the &gt; vs. &gt;= is important so Quantity will be needed.  However, the idea that durationDuration will be expressed with a comparator is a 'best guess', not a requirement.  If someone wants to seek characteristics present for an exact/pinpoint duration, then they would use it without a comparator.  I don't think we want to add constraints on a core Resource element until and unless we see it done consistently across implementers and there is a demand for the constraint.   This is especially important because the Group.characteristic structure is expected to be used for conceptual Groups instead of creating a Characteristic Resource.   Agreeing to that model was based on the assumption we would not need to add extensions to meet the need for many use cases that are not really about groups of patients.  We removed characteristic element from EvidenceVariable rather than make a Characteristic Datatype or Characteristic Resource.  So we have to keep the expressibility within Group.characteristic or revisit the big picture for how this is handled.
