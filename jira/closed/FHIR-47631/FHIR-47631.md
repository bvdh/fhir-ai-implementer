# FHIR-47631: Version - restful interface requirements

## Metadata

- **Assignee**: Unassigned
- **Created**: 9/13/24
- **Description**: The section on Forward and backward compatibility rules states:

Restful interface	Existing endpoints will not be renamed or removed, nor have their expected behavior changed in a manner that would cause reasonable systems designed against prior versions to be non-interoperable. Additional endpoints and interactions may be introduced.

What the definition of endpoint in this sentence? It sounds like that it relates to the URL of a server offering a FHIR server,

This section relates to rules that apply to the FHIR content that is normative. It can and should not place requirements on deployed systems. The deployment is not normative.

I do see that value of the remark. A better place to host this remark would be in a separate section in this page and possibly on the http.html page.
