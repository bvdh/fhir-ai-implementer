# FHIR-47272: Async-bundle - Accept header meaning

## Metadata

- **Assignee**: Rosemary Hofstede
- **Created**: 9/10/24
- **Description**: Does the Accept header in the initial request also suggest the format the result Bundle? Can the client override this by including an Accept-header in the status request?

My preferred solution would be to use the format indicated in http-message and, if absent, the one in the original request.
In the case any of the http-responses has FHIR data, it SHALL also include a Content-Type header stating the format of the data returned.
