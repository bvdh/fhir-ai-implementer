- **Description**: The ig-publisher generates the following warning for examples that use UCUM codes with curly brackets (e.g., {Z-Score} )

&gt; WARNING: Observation/dxa-hip-l-z-score-bmd: Observation.value.ofType(Quantity): UCUM Codes that contain human readable annotations like {Z-Score} can be misleading. Best Practice is not to use annotations in the UCUM code, and rather to make sure that Quantity.unit is correctly human readable

These rules were written  "years ago"  based on committee discussions.

However, they are undocumented.  

 

Proposal:  add a "best-practice" invariant on `Quantity'

Text "UCUM Codes that contain human readable annotations like {Z-Score} can be misleading. Best Practice is not to use annotations in the UCUM code, and rather to make sure that Quantity.unit is correctly human readable"

 

FHIRpath: (code.exists() and code not  "{}")

 `value.ofType(Quantity).code.exists() implies value.ofType(Quantity).code.matches('\\\{.*?}').not()`
