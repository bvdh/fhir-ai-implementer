# FHIR-47235: Active canonical resources and status fields

## Metadata

- **Assignee**: Unassigned
- **Created**: 9/9/24
- **Description**: For some of the CanonicalResources, the status field is used for two purposes: 
1 indicate the status of the resource (retired, draft, .._)
2 indicate that the resource is active and should be used

The active status value is/might sometimes be misused to signal that these resources are active on this FHIR server (PlanDefinition should be applied when event is triggered, SubscriptionTopic is available for use, Questionnaire can be used to request certain information).

This usage is not the intent of this field. 

We should:
1. clearly state that when status=Active this only states something related to the resource, not something related to the use of the resource.
2. add a mechanism (or point to an existing one) that states the PlanDefinition, Questionnaire, SubscriptionTopic that are active on the server; are acted upon/can be used. 

Point 2 could be implemented by defining adding a additional field for this purpose or state that this should be implemented using a List (similar to problem-list) that holds the active resources).
