# FHIR-40745: clarify comment for multiple discriminators

## Metadata

- **Assignee**: Unassigned
- **Created**: 4/1/23
- **Description**: see this chat for background: https://chat.fhir.org/#narrow/stream/179190-united-states/topic/us-core.20further.20profiling.20struggles

 

The current definition on multiple discriminators is unclear as written:
Designates which child elements are used to discriminate between the slices when processing an instance. If one or more discriminators are provided, the value of the child elements in the instance data SHALL completely distinguish which slice the element in the resource matches based on the allowed values for those elements in each of the slices.
propose moving the part on multiple discriminators to the the comments elements and clarify to say:

 

If one discriminator is provided, the value of the child elements in the instance data SHALL be completely distinguish by that discriminator.  If more than one discriminator is provided, the value of the child elements in the instance data SHALL be completely distinguish by applying each discriminator to it. (For example, discriminate using discriminator A. If that matches multiple slices, then further discriminate using discriminator B. etc)"

 

and that order is not important?
- **Issue Type**: Change Request
- **Key**: FHIR-40745
- **Related Artifact(s)**: StructureDefinition
- **Related URL**: http://hl7.org/fhir/elementdefinition-definitions.html#ElementDefinition.slicing.discriminator
- **Reporter**: Eric Haas
- **Resolution**: Not Persuasive with Modification
- **Resolution Description**: We think this language is definitional and an inline implementation note may not be helpful. But we will update "value" to "values" in:

> Designates which child elements are used to discriminate between the slices when processing an instance. {color:#172b4d}*If one or more discriminators are provided, the values of the child elements in the instance data SHALL completely distinguish which slice the element in the resource matches based on the allowed values for those elements in each of the slices.*{color}
