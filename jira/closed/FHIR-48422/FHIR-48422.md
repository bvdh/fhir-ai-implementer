# FHIR-48422: Basic resource should be implicitly allowed for all Reference and Resource uses

- **Description**: Given that Basic is intended to be available for custom resource definitions. There should be implicit acceptance that any Reference might be to a Basic resource.
I think that it is already understood that Bundle.element.resource can carry a Basic; however a profiled version of Bundle.element.resource should also be implicit allowed to be Basic.

This implicit acceptance of Basic seems the only way to allow for future custom resources, and follows Postel's Law. However I am willing to understand other methods of supporting Postel's Law while still enabling profiling to really mean NOT Basic.
