# FHIR-46242: Profiling transaction bundles

## Metadata

- **Assignee**: Corey Spears
- **Created**: 6/27/24
- **Description**: I would like to indicate the profile on a transaction bundle a system accepts. 

The CapabilityStatement does not allow this and the batch/transaction operation is not defined as an OperationDefinition I can override. In IHE the interaction.documentation field is used for this purpose (see https://profiles.ihe.net/ITI/MHD/index.html).

Documenting the allowed profile in the documentation fields feels less optional. 

I suggest we add an extension to CapabilityStatement allows stating the batch/transaction that are profiles accepted.

see also: https://chat.fhir.org/#narrow/stream/179280-fhir.2Finfrastructure-wg/topic/Profiling.20transaction.20endpoint
