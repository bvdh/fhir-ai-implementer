- **Description**: Suggest adding parameters to control the file size of NDJSON of the Response - Complete.

A client may not want the NDJSON of the same type of FHIR resource generated by the FHIR server to be too small or too large. For a large volume of particular FHIR resources, small file size of NDJSON lead to too many small files that a client needs to request remotely. Too large single NDJSON may fail very often when downloading and also difficult to process for some resource limited client. Therefore, minimaNDJSONFileSize and maximaNDJSONFileSize can be specified as part of the request parameters.
