- **Description**: Some examples in R5 have  an incorrect json representation of integer elements.
They should be represented without any quotes around the value (so represented as a json string instead of a json number).

We have encountered this in the following elements in the examples package:

SubscriptionStatus-example.json:
- eventsSinceSubscriptionStart

	notificationEvent.eventNumber



StructureDefinition-integer64.json:

	differential.element&#91;1&#93;.minValueInteger64
	differential.element&#91;1&#93;.maxValueInteger64
(also the same elements in Bundle-types.json).



DocumentReference-example.json

	content.attachment.size



DocumentManifest-654789.json

	contained&#91;2&#93;.content.attachment.size
