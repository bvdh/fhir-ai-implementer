# FHIR-41181: Show links to elements in detailed definitions page

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/3/23
- **Description**: Look at http://hl7.org/fhir/imagingstudy-definitions.html for example

In the underlying DOM, each FHIR element has an anchor tag like http://hl7.org/fhir/imagingstudy-definitions.html#ImagingStudy.started but there is no "copy link" button or other affordance for getting these links when you're on the details page. (They're just surfaced today in the summary trees, which provides links to the details page for each element.) The detailed definitions page should surface links for each element, so anybody browsing the details page can find / save / share links.

Note that the "search parameters" tabs of each resource already support this style of interaction &#8211; e.g., http://hl7.org/fhir/imagingstudy-search.html has a hyperlink in the page for every param.
