# FHIR-55422: Attachment hash should not force SHA1

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/26/26
- **Description**: When reviewing the Attachment datatype mapping to and from openEHR, we noticed that the hash element in FHIR is restricted to SHA-1.

Systems that use other hashing methods internally are then forced to either not include a hash or recalculate one.

Given the normative status of the datatype, I would recommend adding a new pair of elements (or a backbone with them).

openEHR names the equivalent elements integrity_check and integrity_check_algorithm.  In the case of FHIR, the algorithm should probably exclude SHA-1, with a note to use the existing hash element if SHA-1 is the algorithm.
