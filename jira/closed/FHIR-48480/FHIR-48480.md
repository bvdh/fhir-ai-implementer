# FHIR-48480: Add a warning on elements that declare a binding when there are multiple bindable types

## Metadata

- **Assignee**: Unassigned
- **Created**: 10/1/24
- **Description**: In the vast majority of circumstances, it's not the authors intention for the binding to apply to multiple types.

We should add a 'warning' invariant that says:

"This element has a binding and has multiple bindable types.  Please verify that the intention is for the binding to apply to all of the bindable types.  Otherwise, slice the element by type and apply the binding only to the desired type."

The rule would be

if the count of types where code = code, Coding, CodeableConcept, CodeableReference, uri, string, or Quantity is &gt; 1 then binding doesn't exist.
