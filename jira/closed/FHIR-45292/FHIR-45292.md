# FHIR-45292: Clarify that Coding.system is required if known

## Metadata

- **Assignee**: Unassigned
- **Created**: 4/17/24
- **Description**: This is requesting that some language be added to https://www.hl7.org/fhir/datatypes.html#Coding to clarify that Coding.system is required if there is a known value to put there.

See discussion at https://chat.fhir.org/#narrow/stream/179202-terminology/topic/Required.20Coding.20fields.20for.20a.20requied.20binding
- **Issue Type**: Change Request
- **Key**: FHIR-45292
- **Related Page(s)**: Datatypes
- **Related Section(s)**: 2.1.28.0.4
- **Related URL**: https://www.hl7.org/fhir/datatypes.html#Coding
- **Reporter**: Paul Lynch
- **Resolution**: Persuasive with Modification
- **Resolution Description**: On the [Datatypes:Coding|https://build.fhir.org/datatypes.html#Coding] page, we will change

{quote}
In some cases, the {{system}} might not be known - only the code is known. In this case, no useful processing of the code may be performed unless the system can be safely inferred by the context. This practice should be avoided where possible, as information sharing in a wider context is very likely to arise eventually, and codes cannot be used in the absence of a known system.
{quote}

to 

{quote}
In some cases, the {{system}} might not be known - only the {{code}} is known. In this case, no useful processing of the code may be performed unless the system is known implicitly through knowledge of the context (e.g., from a known device/source). This practice should be avoided where possible, as information sharing in a wider context is very likely to arise eventually, and codes cannot be used in the absence of a known system.  Thus, systems SHOULD populate the {{Coding.system}} whenever the system is known.
{quote}
