- **Resolution Description**: Will *not* do this, as the behavior of searches by codes and by references is quite different (different modifiers, etc.)

We will add an additional bullet saying "If the result of the path expression is a type that does not have a conversion to a search parameter type listed above, that is an error.  The FHIRPath should drill deeper to a type that has a conversion.  For example, if a path results in a CodeableReference, the path should be extended to refer to the .concept or .reference sub-elements (which do have conversions defined)."
