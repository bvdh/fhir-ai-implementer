- **Description**: Transactions where ID's are used that correspond to fullUrls is a more complex part of the FHIR specification. The specification would benefit from an example of such behavior. 
An example as given by SmileCDR (https://smilecdr.com/docs/fhir_standard/transactions.html#a-basic-transaction) (request and response) should be added to the specification in the Bundle-example and http-transaction section.

- **Related URL**: https://hl7.org/fhir/R4/bundle-examples.html
- **Reporter**: Bas van den Heuvel
- **Resolution**: Persuasive
- **Resolution Description**: Add one or more example transaction bundles that includes complex item resolution.  Ideally it should cover all formats of URLs (e.g., relative, full, urn, contained, etc.) and contain other entries that refer back to them.  Specifically, there should be at least one example that covers _creating_ a resource and then _referencing_ that resource later in the {{transaction}} (note that this is not allowed in {{batches}}).

At least one example should also contain a reference to resolve something outside of the bundle itself (e.g., {{Patient/example}}).
