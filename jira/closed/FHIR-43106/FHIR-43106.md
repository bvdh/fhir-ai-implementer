# FHIR-43106: Fix rules on Bundle.entry.fullUrl in search

## Metadata

- **Assignee**: Unassigned
- **Created**: 11/13/23
- **Description**: R5 and current spec says:

–

The fullUrl element SHALL have a value except when:

	invoking a create
	invoking or responding to an operation where the body is not a single identified resource
	invoking or returning the results of a search or history operation.



—

 

This change was applied in https://github.com/HL7/fhir/commit/a026c2f242db04aea462f660655cbff949f00026#diff-b5c52b63f2140d604a0c8a5119e92fb41cb6ec215b1564304d93f0eafb3af07fR373 based on https://jira.hl7.org/browse/FHIR-31354 but that resolution said the opposite, albeit confusingly (emphasis and parens added by me)

 

"The fullUrl element SHALL have a value (except for batch or transaction entries) when:

	invoking a create
	invoking or responding to an operation where the body is not a single identified resource; 
	invoking or returning the results of a search or history operation



–
- **Issue Type**: Change Request
- **Key**: FHIR-43106
- **Related Artifact(s)**: Bundle
- **Reporter**: Josh Mandel
- **Resolution**: Persuasive with Modification
- **Resolution Description**: Will change:

"The fullUrl element *SHALL have a value* (except for batch or transaction entries) {*}when{*}:
 * invoking a create
 * invoking or responding to an operation where the body is not a single identified resource; 
 * invoking or returning the results of a search or history operation

–

to

"The fullUrl element SHALL have a value unless:
 * the Bundle is a batch or transaction request or response AND
 * the entry is

 * 
 ** invoking a create,
 ** invoking or responding to an operation where the body is not a single identified resource, OR
 ** invoking or responding to a search or history operation

–
- **Status**: Applied
- **Summary**: Fix rules on Bundle.entry.fullUrl in search
- **Work Group**: FHIR Infrastructure

- **Jira**: https://jira.hl7.org/browse/FHIR-43106
