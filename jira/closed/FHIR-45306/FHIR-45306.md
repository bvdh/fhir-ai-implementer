# FHIR-45306: Explicit Target Resource Types Needed in Chaining Search Examples

- **Description**: According to 3.2.1.6.5 Chaining (chained parameters) section, refer to https://build.fhir.org/search.html#chaining, an example,


GET [base]/DiagnosticReport?subject.name=peter

is a request to return all the lab reports with a subject whose name is "peter".

The DiagnosticReport subject reference refers to more than one resource type as its target, including Patient | Group | Device | Location | Organization | Practitioner | Medication | Substance | BiologicallyDervedProduct. The patient, group, location, organization, and practitioner accept the same search parameters as 'name'. The DiagnosticReport can be one of these resource types. Therefore, it's necessary to limit the search to a particular type:


GET [base]/DiagnosticReport?subject:Patient.name=peter

This request returns all the diagnostic reports with a patient subject named "peter."

Further, according to Advanced Search Note, Where a chained parameter searches a resource reference that may have more than one type of resource as its target, the parameter chain may end up referring to search parameters with the same name on more than one kind of resource at once. Servers SHOULD reject a search where the logical id refers to multiple matching resources across different types. The client has to specify the type explicitly using the syntax in the second example above.

Based on the above specification,

1.Refer to 3.2.1.6.8 Advanced filtering section at https://build.fhir.org/search.html,

For example, "Find all the observations for patient with a name including "peter" that have a LOINC code 1234-5":


GET [base]/Observation?code=http://loinc.org|1234-5&amp;subject.name=peter




GET [base]/Observation?_filter=code eq http://loinc.org|1234-5 and subject.name co "peter"

2. Refer to 3.2.3.1 Introduction at https://build.fhir.org/search_filter.html,


Observation: subject.name co "pet" - all observations on a patient with the characters "pet" in a given or family name


Both examples are limited to a specific type of patient. So, I am requesting some clearer changes. 

	In 3.2.1.6.8 Advanced filtering



Change


GET [base]/Observation?code=http://loinc.org|1234-5&amp;subject.name=peter 

to


GET [base]/Observation?code=http://loinc.org|1234-5&amp;subject:Patient.name=peter  

Change


GET [base]/Observation?_filter=code eq http://loinc.org|1234-5 and subject.name co "peter" 

to


GET [base]/Observation?_filter=code eq http://loinc.org|1234-5 and subject:Patient.name co "peter" 


	In 3.2.3.1 Introduction



Change 


Observation: subject.name co "pet" - all observations on a patient with the characters "pet" in a given or family name 

to


Observation: subject:Patient.name co "pet" - all observations on a patient with the characters "pet" in a given or family name
