# FHIR-51130: Clarify conditional reference rules in transactions

## Metadata

- **Assignee**: Unassigned
- **Created**: 6/17/25
- **Description**: In Resolving references in Bundles, the following text could use additional clarity.

Resolving conditional references. If the reference occurs in a transaction Bundle and the reference.value is a conditional reference (e.g. "http://example.org/base/Patient?identifier=1234"):


	Perform the &#91;search&#93; specified in the conditional reference on the server.
	
		If the search succeeds with a single resource, the resolution succeeds (and ends) here.
		If the search succeeds with multiple matching resources, resolution fails (and ends) here. An error SHOULD be raised.
	
	
	Otherwise the resolution fails (and ends) here.



Per a discussion here on Zulip, we feel that there should be additional clarity around the resolution process.

Specifically, is conditional resolution done:

	against the server contents,
	within the bundle,
	both (bundle first), or
	both (server first)?



Please add clarification text and ensure an example covers this scenario (see ticket FHIR-40498 regarding adding an example).
