# FHIR-43899: Actor / Obligation / Capabilities possible inversion

- **Description**: I am concerned about the ActorDefinition.capabilities element and possible inversion of considerations.

My understanding is that Obligations (many) reference Actors (few).  By inserting the capabilities into the actor, authors will need to update an actor-specific capability statement based on changes to obligations.

Unless the CapabilityStatement resources are automatically generated during processing, there is a high chance of error (out-of-sync changes).
- **Issue Type**: Change Request
- **Key**: FHIR-43899
- **Related Artifact(s)**: ActorDefinition
- **Reporter**: Gino Canessa
- **Resolution**: Persuasive
- **Resolution Description**: We agree.  Will remove ActorDefinition.capabilities and instead add a core element on CapabilityStatement that allows indicating the ActorDefinitions that the CapabilityStatement asserts compliance with.

Rationale is that it's much more likely for new CapabilityStatements to want to point to existing ActorDefinitions (e.g. an implementation CapabilityStatement pointing to an IG-defined ActorDefinition).
