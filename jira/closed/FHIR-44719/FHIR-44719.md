# FHIR-44719: Clarify that empty() considers extensions

## Metadata

- **Assignee**: Unassigned
- **Created**: 2/9/24
- **Description**: See https://chat.fhir.org/#narrow/stream/179266-fhirpath/topic/empty.28.29.20for.20missing.20value.20with.20extensions.3F.  There was disagreement within my development team about whether Patient.active.empty() would be true or false if Patient.active's value was missing but extensions were present.  I suggest making it clear that it should be false (see Grahame's response to the chat.fhir.org link above) by adding an example amongst the other examples in 2.1.9.1.2 about extensions on FHIR primitive types.  For instance:

!Patient.active.hasValue() &amp;&amp; !Patient.active.empty()  // can be true, if there are extensions for Patient.active
