# FHIR-55973: (Dis)allow picking the same item multiple times

- **Description**: If a question has repeats=true, an answer list, and child items, one can conceive of cases where you would want to pick the same item more than once-- though one could also design the form differently to avoid those cases.  This is about clarifying whether it is the intent that UIs support the ability to pick the same item more than once in the list.  If we do support that, then the next question is whether there are cases where the user should not be able to pick the same item more than once, and whether there should be some setting to control that.

For an example questionnaire and discussion, see this thread:   https://chat.fhir.org/#narrow/channel/179255-questionnaire/topic/selecting.20the.20same.20answer.20more.20than.20once
- **Grouping**: [SDC-CORE]
- **Issue Type**: Change Request
- **Key**: FHIR-55973
- **Related Artifact(s)**: Questionnaire
- **Reporter**: Paul Lynch
- **Resolution**: Persuasive
- **Resolution Description**: 1. Will add an invariant to QuestionnaireResponse.item that says:

answer.value.isDistinct()

"The same answer cannot be specified more than once for the same question".

 

2. In Questionnaire, will update repeats comment from "If a question is marked as repeats=true, then multiple answers can be provided" to "If a question is marked as repeats=true, then multiple distinct answers can be provided"

and definition from "may include multiple answers" to "may include multiple distinct answers"

3. In the Questionnaire.repeats comments, will move "When repeats=true for a group, it'll be represented with multiple items with the same linkId in the QuestionnaireResponse. For a question, it'll be represented by a single item with that linkId with multiple answers." to be the first paragraph of the comment.
- **Status**: Applied
- **Summary**: (Dis)allow picking the same item multiple times
