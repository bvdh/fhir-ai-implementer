# FHIR-54755: Narrative - please explain more clearly how to refer from a narrative to a resource

- **Description**: The "Linking between Data and Narrative" section explains how to link between Narrative and Data. 

The server producing the narrative might want to include a reference to a resource that is the source of the information. In what way should this be done (textlink, IHE-IMR, ...)?

Also indicate what extensions should/could be used by the client rendering the narrative to add such reference.
- **Issue Type**: Change Request
- **Key**: FHIR-54755
- **Related Issues**: FHIR-54759: Markdown - how to refer to resources
- **Related Page(s)**: Narrative

- **Related URL**: https://build.fhir.org/narrative.html#linking
- **Reporter**: Bas van den Heuvel
- **Resolution**: Persuasive with Modification
- **Resolution Description**: 1. Will add a section above "Internal Id References" called "References to other resources" with the following content:

"A narrative may include a link to another resource by using an <a/> tag with the {{href}} being a relative or absolute reference to the target resource.  When navigating such a link, the system displaying the narrative SHOULD make available the target resource via its normal resource rendering mechanism.

E.g.

<p>Patient: <a href="Patient/example">(link)</a></p>"

2. Remove the "Internal Id References" section and move the "Linking between Data and Narrative" section up, keeping the anchor tags for both.
