- **Description**: There's a broken sentence in the package spec:
A dependency on a core package is required, and this dependency defines the basic
I assume this intends to say that this defines the FHIR version this package is based on (see the comment in fhirVersions)

 

Furthermore, the canonical parameter in the package spec has changed from optional (Confluence package spec) to required (Core package spec) when migrating this into the spec. The comment makes clear that this is not for all packages. Suggest changing this to optional (see comment) or similar.
