# FHIR-40957: Documentation of capabilityStatement.rest.interaction should use the word "interaction" in place of "operation

## Metadata

- **Assignee**: Unassigned
- **Created**: 4/26/23
- **Description**: In the documentation of capabilityStatement.rest.interaction and its subcomponents, the word "operation" is used repeatedly. Since "operation" is already used other places in the capabilityStatement, it is confusing to refer to the interactions as operations. 

I think just substituting the word "interaction" for operation in the documentation improves the readability of the text, even if the underlying concepts are still just as opaque.
- **Grouping**: [CapabilityStatement2]
- **Issue Type**: Change Request
- **Key**: FHIR-40957
- **Related Artifact(s)**: CapabilityStatement (Conformance)
- **Related URL**: http://hl7.org/fhir/R5/capabilitystatement-definitions.html
- **Reporter**: John Carter
- **Resolution**: Persuasive with Modification
- **Resolution Description**: We will change 'operation' to interaction in the documentation here, but also on the RESTful API page in the few places where we use the wrong term.
- **Status**: Applied
- **Summary**: Documentation of capabilityStatement.rest.interaction should use the word "interaction" in place of "operation
