- **Resolution**: Persuasive with Modification
- **Resolution Description**: Having a binding on {{Group.characteristic.value}} is potentially problematic since it would cover not just the {{CodeableConcept}} but any codeable value, such as a {{Quantity}}.  That said, while we would like to define an example set, it is not feasible since the values depend on the value of the code.  We will remove the binding from this element.

While looking at this, we did see that {{Group.characteristic.code}} has the same issue - a binding name without a binding.  For this element, it does appear to be reasonable to define a starter/example set of codes with common characteristics that could be used out of LOINC (e.g., age, etc.).  We will define a ValueSet with common characteristic codes for use and update the binding.
