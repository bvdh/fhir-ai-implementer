# FHIR-40270: Claim that as() is an alias for ofType() is not correct

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/18/23
- **Description**: The specification says that as() is an alias for ofType() but that's not true - it's a syntactical variation for the as operator, and that works differently - singleton input only
- **Issue Type**: Change Request
- **Key**: FHIR-40270
- **Related Page(s)**: FHIRPath (fluentpath)
- **Related URL**: https://build.fhir.org/fhirpath.html#functions
- **Reporter**: Grahame Grieve
- **Resolution**: Persuasive with Modification
- **Resolution Description**: 1. Remove mention of "as()" function from fhirpath.html in FHIR spec; it's defined in the base fhirpath spec already.

2. Update definition of "ofType()" in fhirpath.html in FHIR spec to:

a.  Change "(type : identifier)" to "(type: type specifier)", matching what's already in the base fhirpath spec.
b.  Change "This works the same..." to "This is as defined..." to make the relationship to the base fhirpath spec clearer.
- **Status**: Applied
- **Summary**: Claim that as() is an alias for ofType() is not correct
- **Work Group**: FHIR Infrastructure

- **Jira**: https://jira.hl7.org/browse/FHIR-40270
