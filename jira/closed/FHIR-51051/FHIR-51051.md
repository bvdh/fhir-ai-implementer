- **Description**: In the section on data types, we currently say:

"Except as described in the preceding paragraph, Datatypes will not be removed or changed except as allowed above for elements. New datatypes may be introduced. Types declared on existing elements will not be removed or changed, except for the special case that string may be changed to markdown."

The exception carved out for string-&gt;markdown is too narrow.  We believe the intent is that it should be possible to change simple types to other simple types provided that the content of instances will be valid against both types.  In particular, we would like to be able to change "code" to "uri".

Discussion point: There are extensions that have permitted context of 'code' that do not support a context of 'uri', which means that in principle, this could be a breaking change.  (The same goes for string -&gt; markdown.)
