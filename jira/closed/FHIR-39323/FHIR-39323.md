# FHIR-39323: Sampled data should support non-quantity samples

## Metadata

- **Assignee**: Unassigned
- **Created**: 10/24/22
- **Description**: We wish to record an analysis code at a fixed rate, multiple times per minute. (e.g. normal heart rate vs. arrhythmia, or patient normal breathing vs. aponia episode.)

The SampledData type only supports decimal values. This leaves us with the option to either use sampled data, but express the coded values as numeric, which requires the consumer to know the mapping; or to provide high frequency individual observations., which becomes voluminous. Neither is optimal.

My preference would be to either allow SampledData to support coded values, or to create a similar data type for coded values. Or, consider a standard extension on SampledData to describe a mapping of codes to numeric values so that consumers of the SampledData know how to reverse the mapping.
