# FHIR-44644: Call more attention to system and type requirements

## Metadata

- **Assignee**: Unassigned
- **Created**: 2/1/24
- **Description**: PA expressed concern that the rules around "you must use standard system URIs where they're defined" is too buried.  Suggest adding the following to the Identifier.system comment:

"The FHIR specification, HL7 affiliates, and others may define identifier systems that SHALL be used for certain types of identifiers.  Further information can be found &#91;here&#93;(identifier-registry.html)
- **Issue Type**: Change Request
- **Key**: FHIR-44644
- **Related Page(s)**: Datatypes
- **Reporter**: Lloyd McKenzie
- **Resolution**: Persuasive with Modification
- **Resolution Description**: Will add

"The FHIR specification, Terminology.hl7.org and HL7 affiliates (for identifiers specific to their jurisdiction) define identifier systems that SHALL be used for certain types of identifiers.  Specific rules can be found [here](identifier-registry.html)"
- **Status**: Applied
- **Summary**: Call more attention to system and type requirements
- **Work Group**: FHIR Infrastructure
