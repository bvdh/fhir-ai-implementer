# FHIR-46626: Harmonize 'min' and 'max' element from ParameterDefinition with ElementDefinition.min/max

## Metadata

- **Assignee**: Unassigned
- **Created**: 8/13/24
- **Description**: For ElementDefinition.min/.max there are some important constraints defined that ensure the consistency of the cardinality of an element:

	min is defined as an unsignedInteger to avoid min &lt; 0
	min needs to be equal or smaller to max (see eld-2)
	max needs to be a valid number of "*" (see eld-3)



The same constraints should be applied to ParameterDefinition.min/.max to avoid invalid combinations of min/max.
