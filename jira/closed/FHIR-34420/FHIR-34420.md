- **Description**: FHIR R4B contains an extension definition http://hl7.org/fhir/StructureDefinition/structuredefinition-standards-status

The extension has a code value with a required binding to http://hl7.org/fhir/ValueSet/standards-status
...however this valueset doesn't seem to exist in R4B.

I think its a case where the valueset has moved from the FHIR specification to terminology.hl7.org

Should this valueset be in R4B? Or should this definition in R4B get updated to use http://terminology.hl7.org/ValueSet/standards-status instead of http://hl7.org/fhir/ValueSet/standards-status ?


Note: in the latest R5 draft, the binding is to http://hl7.org/fhir/StructureDefinition/structuredefinition-standards-status but that one actually exists there:  https://build.fhir.org/valueset-standards-status.html
