- **Description**: In FHIR-21642 the value of a constant is defined as

The value can either be a constant or an expression using other constants.

The current definition of a constant states "Constants can refer to other constants in their expressions, though circular expressions will result in an error". This remark leaves open the use of other variables such as source variables. 

Is this intentional? Can I define constants who's value depends on the context when the map is executed? If so, is this done once or every time the map is executed?

If this is not the intent, please rephrase the sentence to:

"Constants can only refer to other constants in their expressions, though circular expressions will result in an error"
