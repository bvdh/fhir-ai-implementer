# FHIR-17107: Resolving references in Bundles

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/13/18
- **Description**: The instructions for "How to resolve a reference in a Bundle" are unclear in a few ways. 


	The text gives instructions to "append the id to the base URL", but it's unclear what the "base URL" is in the context of a bundle. Does a bundle necessarily have a base URL? How is it determined?




	The instructions describe a two-step process ("convert" and "look"). But the instructions under step 1 don't appear to feed into step 2; for example, the instruction to "try to resolve within the bundle as for a RESTful URL reference" seems to short-circuit step 2.




	The examples don't cover the case where "If the fullUrl starts with urn:uuid: or urn:oid:". All cases should be covered in the examples.
