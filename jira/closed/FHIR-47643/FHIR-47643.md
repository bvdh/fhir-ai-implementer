- **Description**: The section on Forward and backward compatibility rules states:




References
Where one conformance resource points to another (e.g. CapabilityStatement to profile, profile to profiles, profile to value set, etc.), the reference may change to point to a newer version of the conformance resource or to a distinct conformance resource so long as the content of the newly referenced resource adheres to the compatibility rules with respect to the previously referenced version. Additional resources or profiles can also be added to the list of available targets for a reference.




In the case of that a reference to a Draft resource is added to a normative field of a normative resource, does that mean that the draft resource can no longer be removed or have its name changed. I think that is what the spec says (backward parsability). I do not think this the behavior we want.
