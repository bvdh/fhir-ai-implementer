# FHIR-45005: Group characteristic comments are incorrect and combinationMethod defaults need to be defined

## Metadata

- **Assignee**: Unassigned
- **Created**: 3/19/24
- **Description**: Currently the Group.characteristic attribute contains the following comment: 



Comments
All the identified characteristics must be true for an entity to a member of the group.





I think this is a left over from previous versions as the combinationMethod attribute now declares how to interpret what/how many characteristics need to match to be a member of the group.  This comment should be changed to reflect that fact. 

 

The Group.combinationMethod attribute currently accepts the following possible values: 



all-of &#124; any-of &#124; at-least &#124; at-most &#124; except-subset





The attribute itself has a cardinality of 0..1.   There is some vague implicit notion that the default for the combinationMethod if not supplied should be all-of from the comment on the modifier.  



Is Modifier
true (Reason: Changing the combinationMethod from all-of to any other value changes the interpretation of multiple characteristics.)





 

This should either be made more explicit or the cardinality should be changed to 1..1 to make it explicit from the instance.
