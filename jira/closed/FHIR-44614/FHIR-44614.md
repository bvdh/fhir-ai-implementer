- **Description**: The existing specification says: "Implementations should always use the correct element for data when one exists"

 

That's my mistake - it's never appropriate to use a cross-version extension when there's an existing element that is correct to use, so the language should be changed to say:

 

"Implementations SHALL always use the equivalent element for data when one exists.  Cross-version extensions can be used for elements that don't exist, elements with types that do not exist, or elements that now repeat"
