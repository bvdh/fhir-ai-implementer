# FHIR-51428: Bundle paging use should not use next when no more pages are available

- **Description**: The best-practice for Bundle paging should encourage servers to not populate a `next` `link` when the server knows there are no more results available. 

Clients must always be robust to the next link returning no results and must be robust to a Bundle not being full. This might happen if data security/privacy has redacted some results, but may be for many reasons. 

By populating a next link, when the server knows there is no more results, will force the client to do one-more round-trip to request `next` only to find out that there are no more results. These round-trip interactions are unnecessary and waste time/resources.
