# FHIR-49666: CapabilityStatement.rest.resource.type required binding is too narrow

- **Description**: The binding for CapabilityStatement.rest.resource.type is required, and bound to a value set that only includes concrete resources. This is problem because CapabilityStatements are used across versions, and also there's reason to use it with logical models (= custom resources). 

It should work the same way as ElementDefinition.type.code (https://hl7.org/fhir/elementdefinition-definitions.html#ElementDefinition.type.code) so extensibility is possible.
