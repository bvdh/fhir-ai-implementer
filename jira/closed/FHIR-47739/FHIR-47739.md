# FHIR-47739: Clarify the difference between statement.parent and statement.derivedFrom

## Metadata

- **__EMPTY**: Karl Naden
- **__EMPTY_1**: Public

- **Assignee**: Unassigned

- **Created**: 9/13/24
- **Description**: The descriptions of the statement.parent and statement.derivedFrom elements are quite similar: the difference between "clarifies or restricts" and "refine and enable" is not clear. Please clarify the description and provide examples of when each would be used.
- **Issue Type**: Change Request
- **Key**: FHIR-47739
- **Related Artifact(s)**: Requirements
- **Reporter**: Karl Naden
- **Resolution**: Persuasive with Modification
- **Resolution Description**: Will rename "Requirements.statement.parent" to "partOf"

Will change the definition to "Identifies a higher-level requirement or statement which this referencing statement is a logical sub-requirement of.  I.e. This statement is a necessary step to achieving the referenced requirement/statement."

Will change the definition of "derivedFrom" to:

"Indicates that this statement is refining, tightening, or establishing more context for the referenced requirement/statement. 

comment: "With derivedFrom, the scope of what is being accomplished is the same (as opposed to partOf, where the scope of what is being accomplished by the part is narrower)."
- **Status**: Applied
- **Summary**: Clarify the difference between statement.parent and statement.derivedFrom
