# FHIR-57324: StructureMap.group.name type id conflicts w/ constraints

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/18/26
- **Description**: I think I found an issue in StructureMap. The datatype for group.name is id with regexp &#91;A-Za-z0-9\-\.&#93;{1,64} and there is a constraint on group.name with regexp &#91;A-Z&#93;(&#91;A-Za-z0-9_&#93;){1,254} introducing '_' which is not allowed in id causing a QA Error in the IG. Related issue the IG publisher removes the '-' in the rendering of the FML.

The datatype for StructureMap.group.name should be string.

See discussion here #IG creation &gt; StructureMap.group.name type regexp conflicts w/ constraints
