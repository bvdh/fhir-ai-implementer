# FHIR-51079: Add guidance around id uniqueness for inter-version conversion

## Metadata

- **Assignee**: Unassigned
- **Created**: 6/11/25
- **Description**: As per https://chat.fhir.org/#narrow/channel/179252-IG-creation/topic/Duplicate.20IDs.20in.20IGs/with/523613582

Add a dragon section to the resource.html#id section that says:

"If a system does not server-unique (as opposed to resource-unique) logical ids for resources and are using Basic to do &#91;cross-version representations&#93;, be aware that ID collisions are possible and problematic. Possible mitigations could include prefixing/suffixing ids, using a common id namespace for resources subject to being exposed as Basic, or moving to globally unique logical ids.  Note that ids SHOULD be consistent regardless of the FHIR version used to expose the data."

- **Resolution Description**: Will add:

If a system does not have server-unique (as opposed to resource-unique) logical ids for resources and are using Basic to do [cross-version representations], be aware that ID collisions are possible and problematic. Possible mitigations could include prefixing/suffixing ids, using a common id namespace for resources subject to being exposed as Basic, or moving to globally unique logical ids.  Note that ids SHOULD be consistent regardless of the FHIR version used to expose the data.
