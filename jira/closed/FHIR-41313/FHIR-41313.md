- **Description**: In Batch processing, is it allowed to have the same resource appear in multiple entries?  For example, issuing two separate PATCH requests against the same resource?

Paraphrasing our current rules:

	no interdependencies between the different entries in the Bundle that cause change on the server
	results of any one change CANNOT alter success/failure of another change
	cannot reference any resources created during a batch



Given the current rules, it is theoretically possible to PATCH the same resource twice in a Batch.  However, it is quite error prone and I believe outside the desired behavior for Batch processing.

I am requesting that we clarify in the spec whether this is allowed or not to prevent behavior mismatches between implementations.

As a note, I am in favor of saying this is not allowed.

Thanks!
