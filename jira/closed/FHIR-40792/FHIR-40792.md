# FHIR-40792: Request to define :contains for hierarchical searches

- **Description**: Today, we allow the `:above` and `:below` modifiers for searching within hierarchical trees of resources (e.g., when a Location contains sub-locations, etc.).

It would be good to allow for a `:contains` modifier on hierarchies to retrieve the entire tree based on a node.

For example, assume a set of tasks in a tree that: T1 is the root, T1A and T1B are sub-tasks, and T1A1 and T1A2 are sub-tasks of T1A.  If you are working on T1A, you currently need to perform searches for both `:below=T1A` and `:above=T1A` in order to get the entire context of the task you are on.  These cannot be combined in a single query, since you cannot 'or-join' anything with different modifiers.

I propose adding a `:contains` syntax to search hierarchies so that a single search of `:contains=T1A` returns both the tasks above and below the current task in a single request (e.g., from the example above: T1, T1A, T1A1, and T1A2).
