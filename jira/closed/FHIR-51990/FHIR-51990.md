# FHIR-51990: Binding name not globally unique to be useful in code generation

- **Description**: The binding name extension in structure definition has the following description:

"A name that can be used for code generation when generating named enumerations for the binding."

On the surface this seems reasonable and trivial. However, this can only work if the binding names are globally unique. In reality, there's no check done in the publisher, so you may have 2 bindings to the same valueset with different binding names, or worse, 2 bindings to different valuesets with the same binding name. See this comment for details: https://github.com/google/kotlin-fhir/pull/37#issuecomment-3196364105. More specifically:


	Structure definition AuditEvent specifies ValueSet with url http://hl7.org/fhir/ValueSet/security-label-examples with binding name SecurityLabels
	Structure definition Meta specifies ValueSet with url http://hl7.org/fhir/ValueSet/security-labels with binding name SecurityLabels
	Structure definition Consent specifies ValueSet with url http://hl7.org/fhir/ValueSet/security-label-examples with binding name SecurityLabels
	Structure definition DocumentReference specifies ValueSet with url http://hl7.org/fhir/ValueSet/security-label-examples with binding name SecurityLabels



This results in the extension not being useful at all. It's much better at the moment just to use the valueset name. Gino mentioned that he only uses it in very specific contexts &#8211; and I think if that is case the definition and description of this extension should reflect that.

The binding name extension is not currently used by Firely .NET SDK or HAPI (they did use it in R4 for enum names, but not in R5 as far as I can tell).

For more background please see discussion: https://chat.fhir.org/#narrow/channel/179166-implementers/topic/Kotlin.20FHIR.20model.20code.20generation.20questions

- **Resolution Description**: The current description is inaccurate.  We can't force this to be unique.  However, we should change the description.

Will change "A name that can be used for code generation when generating named enumerations for the binding"

to "A name used when rendering the binding and summarizing bindings in tables"

Will also add a note: "If the same binding name appears multiple times within the same StructureDefinition, it SHALL have the same description or value set.
