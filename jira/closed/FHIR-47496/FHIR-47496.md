# FHIR-47496: Group.member.involvement ValueSet Adaption

## Metadata

- **Assignee**: Unassigned
- **Created**: 9/12/24
- **Description**: The new element Group.member.involvement, that was created for representing families or household should reflect the relationship between a member and the Group itself, see also the description „Code that describes how user is part of the group". However the corresponding valueSet https://hl7.org/fhir/6.0.0-ballot2/valueset-group-involvement-set.html contains codes that are used for the relationship between members, like daughter, son, mother-in-law, etc.

PA's proposal is to:

	Remove the codes of the codeSystem http://terminology.hl7.org/CodeSystem/v3-RoleCode from the ValueSet http://hl7.org/fhir/ValueSet/group-involvement-set
	Add the code "family member" to the codeSystem http://hl7.org/fhir/group-involvement



For the relationship between the members of a group the new Resource PersonalRelationshop can potentially be used as soon as it in the spec. See also https://chat.fhir.org/#narrow/stream/179254-patient-administration-WG/topic/PersonalRelationship.20resource

related discussion about this topc on Zulip:
https://chat.fhir.org/#narrow/stream/179254-patient-administration-WG/topic/Family.20and.20Household.20Groups
- **Issue Type**: Change Request
- **Key**: FHIR-47496

- **Reporter**: Reinhard Egelkraut
- **Resolution**: Persuasive with Modification
- **Resolution Description**: We will remove the codes and add the code as proposed.  Will add a comment to the element that says:

"Some systems may track detailed family relationship codes (e.g. child, mother, grandfather) rather than merely saying "family member".  There are a few options to support conveying such information:
 * Instances of the PersonalRelationship resource can be used to indicate relationships between group members.  Such relationships can be leveraged independent of the participants' involvement of the group
 * More detailed family relationship codes (e.g. from the [Personal Relationship RoleType Valueset]([https://terminology.hl7.org/6.0.2/ValueSet-v3-PersonalRelationshipRoleType.html])) MAY be sent as additional Codings in this element.  These SHOULD also be accompanied by the 'family-member' code to support systems that may not understand more detailed codes.  If this is done, then the relationship codes are interpreted as being with respect to whichever group member is designated as "head of household"/"primary contact".  If there is no such group member explicitly designated, then the nature of the more detailed personal relationship codes cannot reliably be interpreted."

We will add a new code to the group-involvement code system called "primary contact" with a definition "This is the focal individual for groups based on a 'household'.  Sometimes also called 'head of household'."

We will add an example of a Group that shows how to do this properly (i..e. where there's a designated head-of-household and where there are some family members with family relationship codings specified along with 'family-member' in a single CodeableConcept).
