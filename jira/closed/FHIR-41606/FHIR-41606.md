# FHIR-41606: Broken naming convention for OperationDefinition.parameter

## Metadata

- **Assignee**: Unassigned
- **Created**: 7/19/23
- **Description**: https://build.fhir.org/operationdefinition-definitions.html#OperationDefinition.parameter.name says in a comment:

&gt; This name must be a token (start with a letter in a..z, and only contain letters, numerals, and underscore. Note that for search parameters (type = string, with a search type), the name may be altered by the search modifiers.

But many OperationDefinitions violate this convention by beginning param names with "_". I suggest removing this comment to reflect widespread usage.
- **Issue Type**: Change Request
- **Key**: FHIR-41606
- **Related Artifact(s)**: OperationDefinition
- **Related URL**: https://build.fhir.org/operationdefinition-definitions.html#OperationDefinition.parameter.name
- **Reporter**: Josh Mandel
- **Resolution**: Persuasive
- **Resolution Description**: Will update the comments from

 
|This name must be a token (start with a letter in a..z, and only contain letters, numerals, and underscore. Note that for search parameters (type = string, with a search type), the name may be altered by the search modifiers.|

 

to

 
|This name must be a token (start with a letter in a..z or underscore and only contain letters, numerals, and underscore). Note that for search parameters (type = string, with a search type), the name may be altered by the search modifiers.|
- **Status**: Applied
- **Summary**: Broken naming convention for OperationDefinition.parameter
- **Work Group**: FHIR Infrastructure

- **Jira**: https://jira.hl7.org/browse/FHIR-41606
