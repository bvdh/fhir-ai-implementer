# FHIR-39754: Clarify expectations on profiling recursive backbone elements

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/1/23
- **Description**: There are a number of resources that contain backbone elements that recurse either directly or indirectly.  (E.g. Questionnaire.item).  Typically, when a profile is created that constrains the content of such elements, the intention is that the constraint applies to all references to the backbone element, not merely its 'root' appearance.

However, we haven't declared anywhere that this is the case.  We should.  We can also explain that if the intention is to have constraints that apply only to the root occurrence, those can be achieved via invariants on the containing element.
