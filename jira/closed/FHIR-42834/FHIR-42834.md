# FHIR-42834: Clarify escaping examples

## Metadata

- **Assignee**: Gino Canessa
- **Created**: 9/27/23
- **Description**: Reading this section: https://build.fhir.org/search.html#escaping and wondering why the 1st and 3rd examples are the same:

Standard percent escaping still applies, such that these URLs have the same meaning:

	GET &#91;base&#93;/ValueSet?url=http://acme.org/fhir/ValueSet/123,http://acme.org/fhir/ValueSet/124,ValueSet/125
	GET &#91;base&#93;/ValueSet?url=http%3A%2F%2Facme.org%2Ffhir%2FValueSet%2F123%2Chttp%3A%2F%2Facme.org%2Ffhir%2FValueSet%2F124%2CValueSet%2F125
	GET &#91;base&#93;/ValueSet?url=http://acme.org/fhir/ValueSet/123,http://acme.org/fhir/ValueSet/124\,ValueSet/125



The 3rd one query has a escaped comma which should change the comma from the multiple-or connector to a character in the search value.

Suggestion:
Either remove the 3rd query, or write an explanation why the 3rd query having the same meaning as previous two.

- **Resolution Description**: Fix the third example to show a mixture of 'slash' escaping *and* 'percent' escaping.  E.g.: 
{{GET [base]/ValueSet?url=http://acme.org/fhir/ValueSet/123%2Chttp://acme.org/fhir/ValueSet/124\,ValueSet/125}}

Insert a description before an additional example to show that both escaped and non-escaped can appear in the same query.  E.g.: 
{{GET [base]/ValueSet?url=http://acme.org/fhir/ValueSet/123,http://acme.org/fhir/ValueSet/124\,ValueSet/125}}
