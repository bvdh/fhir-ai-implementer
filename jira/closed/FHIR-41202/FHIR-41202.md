# FHIR-41202: Define a way for an OperationDefinition to indicate that the operation follows the async pattern.

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/6/23
- **Description**: I can't seem to find any documentation in the async pattern pages that show how an OperationDefinition should indicate that the response is asynchronous. If there is a way to do this today, please add it to the async pattern documentation, and if there is no currently defined way to do this, please define it (new extension, etc.).
- **Issue Type**: Change Request
- **Key**: FHIR-41202
- **Related Artifact(s)**: OperationDefinition
- **Related Page(s)**: Asynchronous API (async)
- **Related URL**: http://hl7.org/fhir/async.html
- **Reporter**: Rick Geimer
- **Resolution**: Persuasive with Modification
- **Resolution Description**: Will define OperationDefinition.synchronicity code 0..1

short "synchronous | asynchronous"

definition: "Indicates that this operation must always be handled as synchronous or asynchronous"

meaningWhenMissing "It is up to server discretion whether to handle a particular invocation of the operation as synchronous or asynchronous"

 

Will also define an extension CapabilityStatement-synchronicity code 0..1 with context of CapabilityStatement.rest.interaction, ...rest.operation, ...rest.resource.operatoin, ...rest.resource.interaction

short "synchronous | asynchronous"

definition: "Indicates that this interaction or operation will always be handled by this server as synchronous or asynchronous"

meaningWhenMissing "It is up to server discretion whether to handle a particular invocation of the interaction or operation as synchronous or asynchronous"

 

We will update the asynch page to say:

Operations can declare as part of their definition whether they will be performed synchronously, asynchronously, or either.  (See [OperationDefinition.synchronicity].)  Systems can also declare whether specific operations or interactions will be handled synchronously, asynchronously, or a mixture of them with [synchronicity extensions] in their CapabilityStatements.)
