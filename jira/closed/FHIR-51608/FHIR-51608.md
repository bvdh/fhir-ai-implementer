# FHIR-51608: Clarify urls for resources retrieved from compartments

- **Created**: 7/31/25
- **Description**: When a resource is retrieved from a compartment, presumably the resource full url, location header, etc. all reflect the "native" location not the location within the compartment. Use of a compartment should not be treated as an alternate base url scoped to the definitional resource (you can't tell patient 1234 to point their FHIR client at http://example.com/Patient/1234 and expect everything to work). This has a number of implications:

	Clarify that If I retrieve http://example.com/Patient/1234/Observation/5678, all the places  (location header, id, etc.) that use an absolute url should use http://example.com/Observation/5678, all the places that use a relative url should use /Observation/5678.
	Clarify that references within a resource to other resources should not be modified to point within the compartment. If http://example.com/Observation/5678 has a reference to a Practitioner/2222, the reference when accessed through http://example.com/Patient/1234/Observation/5678 remains /Practitioner/2222 or http://example.com/Practitioner/2222. It is not modified to http://example.com/Patient/1234/Practitioner/2222.
	Clarify that, generally, there is no expectation that a non-absolute id will resolve within a compartment. If http://example.com/Patient/1234/Observation/5678 has a focus of Location/3333, the user shouldn't expect http://example.com/Patient/1234/Location/3333 to return the resource.
	Clarify that none of the operations on the FHIR base url "/" are supported on a compartment, including metadata, and bundle handling (maybe it's some limited set, perhaps only $everything). 
	Clarify the ids and fullUrls returned in search bundles within the compartment (http://example.com/Patient/1234/Observation?code=foo). Clarify the next/prev/self urls in the bundle. 
	Clarify if operations on a resource type or instance are permitted on the compartment view of that type or instance. Can I call http://example.com/Patient/1234/Observation/$lastn? Should an operation called within a compartment only operate on resource instances within the compartment?



These issues should be clarified broadly in the ComponentDefinition narrative, and specifically on each of the appropriate bundle, search, REST, identity etc. pages.
