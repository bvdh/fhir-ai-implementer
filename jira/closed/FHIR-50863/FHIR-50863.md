# FHIR-50863: What happens to the resourceDefinition url when an additional resource moves to core?

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/13/25
- **Description**: In theory, once something is core, it doesn't need an resourceDefinition anymore, because it's core.  Also, the presence of resourceDefinition signals that something isn't a core resource.  However, removing this element would be a breaking change.

We don't want to declare versions of things once resources are normative (because that encourages implementers to extract rent from upgrades).  Should we simply say that resourceDefinition should appear everywhere in R6+ and simply have it version-agnostic when core?
