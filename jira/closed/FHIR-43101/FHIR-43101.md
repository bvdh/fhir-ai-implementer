# FHIR-43101: Further clarification if rule around fullUrl

## Metadata

- **__EMPTY**: Elliot Silver
- **__EMPTY_1**: Public
- **__EMPTY_2**: Does this definition need to account for version-specific fullURLs?
- **Assignee**: Unassigned
- **Comments**: 11/14/23
- **Created**: 11/12/23
- **Description**: There's a rule on bundle that says:

&gt; if the fullUrl does look like a RESTful server URL (e.g. meets the regex, then the 'id' portion of the fullUrl SHALL end with the Resource.id.

we should (a) fix the typo (missing ')'), and (b) clarify the the stated type has to match as well 

 

(see wider discussion at relative reference resolves only if source or target has id)
- **Issue Type**: Change Request
- **Key**: FHIR-43101
- **Related Artifact(s)**: Bundle
- **Reporter**: Grahame Grieve
- **Resolution**: Persuasive
- **Resolution Description**: Will change "if the fullUrl does look like a RESTful server URL (e.g. meets the regex, then the 'id' portion of the fullUrl SHALL end with the [Resource.id|http://resource.id/]."

to

"if the fullUrl does look like a RESTful server URL (e.g. meets the regex), then the portion of the URL that, by FHIR syntax, corresponds to the resource type and id within the fullUrl SHALL match the resource type and id of the resource."
- **Status**: Applied
- **Summary**: Further clarification if rule around fullUrl
- **Work Group**: FHIR Infrastructure

- **Jira**: https://jira.hl7.org/browse/FHIR-43101
