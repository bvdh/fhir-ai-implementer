# FHIR-43364: Extensions with context Element apply everywhere but a resource is not an Element

## Metadata

- **Assignee**: Unassigned
- **Created**: 12/11/23
- **Description**: Extensions indicate the context in which an extension can be used.

Traditionally, the indication Element indicates that extension can be used anywhere. 

The issue is that a Resource is no longer an Element. This makes this confusing to new readers. The specification should be updated to correct this by either introducing a new expression to state anywhere or to add documentation stating the historic reason and use of Context: Element.

In the latter case, we should also discuss whether their is a need an capability to indicate the way an Extension has the Context "only on Resource root" _ or  "everywhere except on the resource root{_}".
- **Issue Type**: Change Request
- **Key**: FHIR-43364
- **Related Page(s)**: Extensibility

Extension Registry
- **Reporter**: Bas van den Heuvel
- **Resolution**: Persuasive with Modification
- **Resolution Description**: This would be difficult to change for all internal tooling and have implications on existing published content.

Propose additional documentation (the 'latter' option in the request) to note that defining a {{StructureDefinition.context.type}} of {{element}} can reference the root elements of resources and datatypes, which allows for extensions on the root of any structure that allows it (e.g., any {{DomainResource}}).

Notes for the second part of the question:
* For a context of 'only on resource root', a context with {{type=element}} and {{expression=DomainResource}} can be used for all resources, or a specific resource can be specified (e.g., {{type=element}} and {{expression=Patient}}).
* Using {{type=element}} and {{expression=element}} does represent _any_ element, including the 'root' element of a resource.
* There is currently no way to define 'all elements except the root' in a {{type=element}} context, if that functionality is required, authors can use {{type=fhirpath}} and set a FHIRPath {{expression}}.
