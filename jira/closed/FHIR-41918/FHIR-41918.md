# FHIR-41918: Clarify reference.reference constraint scope

## Metadata

- **Assignee**: Ward Weistra
- **Created**: 8/25/23
- **Description**: If a reference is constrained to specific resource types, it's valid to include a .display or .identifier without having to populate .reference pointing to a valid referenced resource. But it is argued ([here|https://chat.fhir.org/#narrow/stream/179190-united-states/topic/reference.2Eidentifier.20conformance/near/381589986) that 

"If a reference is constrained to a specific profile, it SHOULD contain a reference that resolves to a valid instance of that profile. It MAY contain an identifier that identifies a valid instance of that profile, somehow. It MUST contain one of those." (As I understood it.)

Include this guidance, or other guidance if incorrect.
