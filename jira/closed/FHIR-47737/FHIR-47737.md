# FHIR-47737: Support actor links at the statement level

## Metadata

- **__EMPTY**: Karl Naden
- **__EMPTY_1**: Public
- **__EMPTY_2**: meant to submit as a change request
- **Assignee**: Unassigned
- **Comments**: 9/16/24
- **Created**: 9/13/24
- **Description**: Consider a use case where someone wants to create a Requirements instance representing all of the discrete requirements in an IG. Typically IGs place requirements on multiple different interacting actors and the actor element in the Requirements Resource supports associating multiple actors with the requirement set. However, there is no way to indicate which statements apply to each actor, meaning that you won't know which actor of the set indicated at the top level a given requirement applies to. This seems like an important piece of information to represent that should be supported by the resource. Recommend adding a statement.actor element that mirrors the base actor element and is a (improper) subset of those defined at the top level
- **Issue Type**: Change Request
- **Key**: FHIR-47737
- **Related Artifact(s)**: Requirements
- **Reporter**: Karl Naden
- **Resolution**: Persuasive with Modification
- **Resolution Description**: 1. We will change actor from canonical to backbone element and will add two children:

key 1..1 id "Unique label for actor" "A unique string used to reference the actor from specific requirements"

reference 1..1 canonical "Actor referenced" "The canonical for an actor relevant to some of the requirements"

2. We will add Requirements.statement.actor 0..* id "Key of relevant actor" "A reference to the key of an actor listed in Requirements.actor that is relevant to this requirement statement."

comment: "If not specified, all actors listed on the Requirements is presumed to be relevant to the statement."

3. Will add an invariant that Requirements.statement.actor must match a Requirements.actor.key.

4. Will change "If more than one actor is specified, then it's up to the statements to describe in narrative if they don't apply to all actors." in the comment on actor to

"If more than one actor is specified, then it's up to the statements to explicitly identify the actors that are relevant if the statement doesn't apply to all actors."
- **Status**: Applied
- **Summary**: Support actor links at the statement level
- **Work Group**: FHIR Infrastructure
