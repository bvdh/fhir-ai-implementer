- **Description**: The section on searching with Composite Search Parameters uses this example: `GET &#91;base&#93;/Group?characteristic-value=gender$mixed`.

At the same time, there is a Group example resource that seems to align with this search.

However, both the code and the valueCodeableConcept components of this Group only have a text component, and no code / Coding.

Since you would need a `:text` modifier to match this, but modifiers are not allowed on composite search parameters, there is no way to match this group.

I suggest to alter the example resource to include proper code / Coding values so the example search interaction matches this Group.
