# FHIR-50579: Implementation Obligations should mention Vital Signs profile

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/1/25
- **Description**: The "Implementation Obligations" page states that

"

The base FHIR specification is intended to be independent of any particular implementation context, so no elements are flagged as mustSupport=true as part of the base specification, and no obligations are defined for any elements (except for an example profile).

"

This seems misleading given the existence of the Vital Signs profile, which has direct implementation requirements and uses mustSupport.
