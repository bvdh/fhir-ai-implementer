- **Description**: In the definition of the string type, there is mention of \r and \n; but there is no indication of if strings should/shall use just \n or \r\n. 

&gt; Strings SHOULD not contain Unicode character points below 32, except for u0009 (horizontal tab), u000D (carriage return) and u000A (line feed).

It is possible we mean what is said in GFM Markdown about this topic

&gt; A line ending is a newline (U+000A), a carriage return (U+000D) not followed by a newline, or a carriage return and a following newline.

 

The xhtml is less problematic as these characters never mean anything, but leaving it unsaid is potential trouble.

 

 

I suggest we just clarify in string (to be like markdown) make it clear that a recipient must expect both patterns.
