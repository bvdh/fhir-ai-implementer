- **Resolution**: Persuasive with Modification
- **Resolution Description**: We will:
 * define a standard element 0..* called "aliasCode" that allows one SearchParameter instance to be used using additional search strings.
 * Will add/update examples to show the use of this.
 * highlight the existence of the aliases in the SearchParameter registry page and in the table of search parameters for each resource.
 * define a formal naming guideline for all HL7-defined search parameters
 * identify what the new 'preferred' codes should be for all of the existing search parameters.  (Where a SearchParameter is 'normative', preferred will need to remain the normative code.)
 * update the search parameters to have those as the official codes while retaining the original codes as 'alias' codes that should also be supported.
