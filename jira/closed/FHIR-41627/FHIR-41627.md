# FHIR-41627: Declaration of SUBSETTED resources for language

## Metadata

- **Assignee**: Unassigned
- **Created**: 7/26/23
- **Description**: The R5 (and current) spec describes the use of HTTP headers to instruct servers to return content for a specified language.

If the server returns a resource with a specific language component, does that not infer that the whole resource is not returned and as such the resource should be marked as SUBSETTED.

In other parts of the standard where responses are SUBSETTED, the following text is used

Servers SHOULD mark the resources with the tag SUBSETTED  to ensure that the incomplete resource is not actually used to overwrite a complete resource.

There is no clear signposting as to how the use of that tag prevents updating. The provision of ETags for SUBSETTED resources is not described (or I could not find it).
