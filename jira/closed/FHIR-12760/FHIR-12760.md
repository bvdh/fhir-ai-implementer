# FHIR-12760: timing offset should allow for negative and positive numbers

- **Description**: The timing.offset in STU3 now has type 'unsignedInt'. However, in order to distinguish between an offset which is 'earlier' and an offset whicht is 'later' it makes sense to allow for both negative and positive offset.

         Discussed on Zulip with Lloyd: https://chat.fhir.org/#narrow/stream/implementers/subject/Diabetes.207-point.20profile.20self-monitoring.20and.20terminology          

         Please note that when implementing this, the current example using offset also needs to be amended:

          'BID, 30 mins before meal, for next 10 days' on "http://build.fhir.org/datatypes.html#Timing" would have to be corrected along with the offset Type. It now has an offset of '30', but if it is before it would have to be '-30'

         Please note that 'weird' use cases are made possible applying this change ("30 minutes before after meal"), it may be worth adding this in usage notes, see Zulip for further details.
