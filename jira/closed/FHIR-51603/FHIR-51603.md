- **Description**: It isn't clear from the narrative sections of CompartmentDefinition whether the inclusion criteria for a resource in the compartment is based on the element name, a search parameter, or a FHIRpath expression.

Digging deeper, I do fine that the CompartmentDefinition.resource.param element short description and definition mention search, so I can only presume compartments are defined based on search parameters, but it seems that something so fundamental should be more visible and explicitly defined.

Also, for compartment RelatedPerson, Observation is included in the compartment based on the {{subject }}search parameter even though {{subject }}can return non-RelatedPerson resources. There is no need for a FHIRpath ofType(), or a RelatedPerson-specific search parameter. It would be helpful to be explicit about that.
