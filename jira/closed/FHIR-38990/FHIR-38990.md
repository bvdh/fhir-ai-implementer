- **Description**: In addition to canonical URLs (e.g., ImplementationGuide, OperationDefinition, etc.), CapabilityStatement instances should provide package and (optionally) registry information.

For example, if a server supports http://hl7.org/fhir/us/core/StructureDefinition/us-core-patient, it would be nice to know that the clients should load hl7.fhir.us.core|5.0.1, and that it is available on http://packages.fhir.org.

This information could either be general to a CapabilityStatement (e.g., a root element that lists package name, version, and (optional) registry URL), or as extensions on canonical URLs.

) but it would be better if the user doesn't have to guess where they came from, since there can be multiple hits.
