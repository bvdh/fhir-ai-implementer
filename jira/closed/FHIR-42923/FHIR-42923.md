- **Description**: In order to help implementers with this problem, any element defined in any version of FHIR is automatically assigned an extension URL that uniquely identifies the element and can be used in the relevant FHIR version. The extension URL for an element can automatically be derived:
[http://hl7.org/fhir/[version]/StructureDefinition/extension-[Path]]
Clarify that "relevant FHIR version." I think this is this FHIR version of the resource instance you are trying to create, not the FHIR version of the resource type where this element is defined.

Conversely, clarify that `&#91;version&#93;` in the url is the FHIR version where the element is defined, not the FHIR version of the resource instance you are creating.

In the "Formal definitions for extensions" table at the bottom of the section, clarify what the rows and columns represent. I think this is meant to be read as "To get the R4 extension definitions for use in R3 resources, use the entry at the intersection of R4 Extensions and R3 Package which gives us hl7.fhir.extensions.r4:3.0.1."
