# FHIR-42765: Need to add value to .type ValueSet in Address DataType for Birth (Required binding)

## Metadata

- **Assignee**: Unassigned
- **Created**: 9/15/23
- **Description**: There is an extension for birth place but you cannot specify the type of address since "birth" is not a type option and the binding is Required.  Therefore need to add a value to the Valueset, http://hl7.org/fhir/R5/valueset-address-use.html
- **Issue Type**: Change Request
- **Key**: FHIR-42765
- **Related Page(s)**: Datatypes
- **Related Section(s)**: 2.1.28.0.15
- **Related URL**: http://hl7.org/fhir/R5/datatypes.html#Address
- **Reporter**: Linda Michaelsen
- **Resolution**: Not Persuasive with Modification
- **Resolution Description**: Suspect you mean a 'use' rather than a 'type'.  (The type of 'birth' address would almost always be physical.)  However, there's no need to declare 'use' in this situation.  If the use of the address is already defined by the element with the type of Address, you wouldn't specify the use.  'use' is really only relevant when there's a collection of addresses you need to differentiate.

If we were to add a 'use' then there'd be two different ways to convey birth address - in Patient.address with a 'use' of 'birth', or using the extension.  And we don't want that.

We will add a comment on 'use' that says "This element is relevant when there is potentially a collection of addresses and a need to differentiate the purpose of each.  When this type is used on an element or extension where the 'purpose' of the address is implicit in the extension definition, 'use' will typically be omitted."
