# FHIR-42989: Add "check for implicitRules" to safety checklist

## Metadata

- **Assignee**: Unassigned
- **Created**: 10/23/23
- **Description**: &gt; For each resource that my system handles, I've reviewed the Modifier elements

is hard to understand on its own &#8211; we should add an implicit check for "implicitRules" element on every resource.  https://build.fhir.org/resource-definitions.html#Resource.implicitRules
- **Issue Type**: Change Request
- **Key**: FHIR-42989
- **Related Page(s)**: Clinical Safety (safety)
- **Related URL**: https://build.fhir.org/safety.html#conformance
- **Reporter**: Josh Mandel
- **Resolution**: Persuasive with Modification
- **Resolution Description**: Update
{quote}
For each resource that my system handles, I've reviewed the Modifier elements
{quote}
to
{quote}
For each resource that my system handles, I have reviewed the [Modifier elements|https://hl7.org/fhir/conformance-rules.html#isModifier] defined for that resource and ensured that my system does not incorrectly process resources that use them.
{quote}

Update 
{quote}
My system checks for [modifierExtension elements|https://hl7.org/fhir/extensibility.html#modifierExtension]
{quote}
to:
{quote}
My system checks for [modifierExtension elements|https://hl7.org/fhir/extensibility.html#modifierExtension] for each resource it processes and behaves properly when encountering known or unknown extensions in them.
{quote}

Add:
{quote}
My system checks for [Resource.implicitRules values|https://hl7.org/fhir/resource-definitions.html#Resource.implicitRules] for each resource it processes and behaves properly when encountering unknown values in them.
{quote}
