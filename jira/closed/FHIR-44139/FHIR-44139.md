- **Description**: The Multi-Version Implementation Guides section starts with this sentence:
Most implementation guides target a single version - that is, they describe how to use a particular version, and all the profiles, value sets and examples they contain etc. are valid for that particular version. 
Since the IG resource describes its own version, its versions of FHIR, and its versions of dependencies, I think it is helpful to be clear which version we are talking about (especially since the word "FHIR" is not even used until the third paragraph). This is a simple fix:
Most implementation guides target a single version of FHIR - that is, they describe how to use a particular version, and all the profiles, value sets and examples they contain etc. are valid for that particular version.
