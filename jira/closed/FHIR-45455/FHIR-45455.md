# FHIR-45455: The slice function should be clearer on the parameters

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/3/24
- **Description**: The Additional functions section should include bookmarks to be able to navigate to the definitions of each function individually.

The slice function isn't clear what the parameters should be, and there should be examples for using each of the functions.

Further discussion here:

https://chat.fhir.org/#narrow/stream/179177-conformance/topic/co-occurrence.20of.20slices
- **Grouping**: [Applied]
- **Issue Type**: Change Request
- **Key**: FHIR-45455
- **Related Page(s)**: FHIRPath (fluentpath)
- **Related Section(s)**: Additional Functions - Slice
- **Related URL**: http://build.fhir.org/fhirpath.html
- **Reporter**: Brian Postlethwaite
- **Resolution**: Persuasive
- **Resolution Description**: # Will add bookmark anchors for each of the function to support navigation
 # Will add the following to the definition of 'slice':
"where 'structure' is the (possibly versioned) canonical of the profile in which the slice is defined, and 'name' is either the ElementDefinition.sliceName for the slice if that sliceName only appears once in the StructureDefinition snapshot, or the ElementDefinition.id of the element holding the sliceName if the sliceName is not unique in the profile."
 # Will include an example of use for all of the FHIR-specific functions.
