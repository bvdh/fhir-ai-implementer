# FHIR-49712: constraint: answerOption.value cannot be a valueless structure

## Metadata

- **Created**: 1/30/25
- **Description**: It does not make sense to have Questionnaire.item.answerOption.valueCoding where the Coding is present but has neither code nor display.

Likewise, it does not make sense to have Questionnaire.item.answerOption.valueQuantity if the Quantity does not have a 'value'.

Constraints should be added to prevent that, and a similar rule should probably be put in place for valueReference as well.
- **Grouping**: [SDC-IG]
- **Issue Type**: Change Request
- **Key**: FHIR-49712
- **Related Artifact(s)**: Questionnaire
- **Reporter**: Paul Lynch
- **Resolution**: Persuasive with Modification
- **Resolution Description**: In the base Questionnaire profile will add an invariant that says that:
 * answerOption.valueCoding must have a code or a display, and that if it has a code it must have a system
 * valueQuantity must have value
 * valueReference must have .reference, .identifier, or .display
