# FHIR-40607: Discrepancy in UCUM display values for duration

## Metadata

- **Assignee**: Unassigned
- **Created**: 3/14/23
- **Description**: The valueset http://hl7.org/fhir/ValueSet/duration-units is the minimal valueset for the Duration data type, and presents all display values in the plural, e.g. "days".

The valueset http://hl7.org/fhir/ValueSet/all-time-units is the maximal valueset for the Duration data type, and presents all display values as equivalent to the code (e.g. "d"). This valueset also seems incorrectly defined, as it doesn't actually limit values to time units.

The valueset https://build.fhir.org/valueset-ucum-common.html presents display values in the singular, e.g. "day".

Although we accept variation in display values, we should be internally consistent and use the same display throughout. Additionally, we should correct the definition of the all-time-units ValueSet.
