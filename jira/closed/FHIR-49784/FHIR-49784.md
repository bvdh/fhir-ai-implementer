# FHIR-49784: Definition of SearchParams on nested elements

## Metadata

- **Assignee**: Unassigned
- **Created**: 2/12/25
- **Description**: We have found a problem in the definition of SearchParameters defined on nested elements. If I have defined a CodeSystem with codes in the second or third level of a concept, the search parameter 'code' with the expression 'CodeSystem.concept.code' does not reach these codes.

This problem occurs in all versions including the current one.
