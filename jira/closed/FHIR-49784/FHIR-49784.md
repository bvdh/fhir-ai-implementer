# FHIR-49784: Definition of SearchParams on nested elements

## Metadata

- **Assignee**: Unassigned
- **Created**: 2/12/25
- **Description**: We have found a problem in the definition of SearchParameters defined on nested elements. If I have defined a CodeSystem with codes in the second or third level of a concept, the search parameter 'code' with the expression 'CodeSystem.concept.code' does not reach these codes.

This problem occurs in all versions including the current one.

The related Zulip discussion is https://chat.fhir.org/#narrow/channel/179177-conformance/topic/Definition.20of.20SearchParams.20on.20nested.20elements
- **Grouping**: [Applied]
- **Issue Type**: Change Request
- **Key**: FHIR-49784
- **Related Artifact(s)**: CodeSystem

Composition

Questionnaire
- **Related Section(s)**: SearchParams
- **Related URL**: https://hl7.org/fhir/codesystem.html#search
- **Reporter**: JonasSchön
- **Resolution**: Persuasive with Modification
- **Resolution Description**: Modify the requested FHIRPath expressions to include the {{repeat}} function to ensure that nested properties are searched when appropriate.

Perform a check to determine where else this may apply (sub-element of the same type as parent).  For resources that are under FHIR-I, fix them.  For resources under other WGs, file tickets asking for the change.
