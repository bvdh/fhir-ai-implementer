# FHIR-46620: Clarify or Correct StructureDefinition context of Resource

## Metadata

- **Assignee**: Unassigned
- **Created**: 8/13/24
- **Description**: In StructureDefinition.context when you have a type of element, a valid and common value for expression is Resource.

Unfortunately, Resource does not actually allow extensions - DomainResource does.

I assume the ship has sailed on forcing people to use DomainResource as the context, but as the specs do not clarify a filter on Resource, the context can be read in invalid ways (e.g., trying to put an extension on a Bundle). 

Do we:

	Document use of Resource as an allowed context with caveats (e.g., clarifying that Resource does not allow extensions and that using the value actually means any Resource that is a valid context - i.e., DomainResource)?
	Introduce a validator warning saying that they SHOULD be changed to DomainResource and fix them going forward?
	Both?
