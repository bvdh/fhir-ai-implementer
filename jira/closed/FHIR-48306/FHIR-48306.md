# FHIR-48306: Clarify StructureDefinition.experimental

## Metadata

- **Assignee**: Unassigned
- **Created**: 9/17/24
- **Description**: From Zulip thread: Terminology: Overuse of the Experimental flag.

It appears the intention of the experimental flag is to classify a structure, not identify a status on it (e.g., a structure is part of experimental work that will not 'move' to production, not that something is in progress and not yet ready for production).

We need to update the descriptions of the element to indicate this.  Suggest updating the short, definition, and comment of StructureDefinition.experimental to clarify use.

Specifically:

	short should identify the use as classification and not status
	comment should explain the use in the context of status and FMM levels.
- **Issue Type**: Change Request
- **Key**: FHIR-48306
- **Related Artifact(s)**: StructureDefinition
- **Reporter**: Gino Canessa
- **Resolution**: Persuasive with Modification
- **Resolution Description**: In CanonicalResource (and those that inherit from it), will make the following changes:

1. We will change "For testing purposes, not real usage" to  "For testing only - never for real usage"

2. We will change "A Boolean value to indicate that this \{{title}} is authored for testing purposes (or education/evaluation/marketing) and is not intended for genuine usage."

to

"A Boolean value to indicate that this \{{title}} is authored for testing purposes (or education/evaluation/marketing) and no version of this resource will ever be intended for genuine usage."

3. Will update comments to append:

"Experimental resources might include example instances in implementation guides, instances created solely for testing purposes, etc."

4. Will update comments to append:

"If experimental is 'true' for any version of a resource, it should be true for all versions of the resource.  (If experimental changes, then it is being misused or a resource that was never intended for real-world use has unexpectedly changed its purpose.). 

Experimental resources are not expected to be stable and may well have breaking changes without a corresponding change to the 'version' element."
- **Status**: Applied
