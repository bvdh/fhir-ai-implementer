# FHIR-54062: Rework startOffset/endOffset constraint

## Metadata

- **Assignee**: Bas van den Heuvel
- **Created**: 1/13/26
- **Description**: The current constraints for timing seem a little under-specified



  tim-11
Rule
Timing.repeat
Only use startOffset when frequency is present and &gt; 1
startOffset.exists() implies (frequency &gt; 1)


  tim-12
Rule
Timing.repeat
Only use endOffset when frequency is present and &gt; 1
endOffset.exists() implies (frequency &gt; 1)





The way endOffset is used in a dosage example
timing
        repeat
          endOffset = 7 d
          frequency: 21
          period: 28
          periodUnit: d
Is intended to represent 21 days of taking the medication followed by 7 days of not taking the medication over a total period of 28 days.

 

The implication is that both period and frequency are important (where the constraint only requires frequency be present)

 

But starting from nothing, the above representation could quickly become confusing if frequency were some other number (like 14) - implying that for unambiguous meaning, there probably should be some relationship between frequency, period, and the offsets that leaves the frequency divided into the non-offset period understandably.

(i.e. frequency = 42 could potentially be understood above - do it twice a day - but frequency = 11 would be confusing)

Best suggestion I can come up with right now is:

frequency / x = n 

OR

x / frequence = n

Where n is a whole number and x is period - startOffset - endOffset

but that feels super clunky.

- **Resolution**: Persuasive with Modification
- **Resolution Description**: # We will update the constraints to require that period also exists
 # We will update the comments for both elements to point to a section in the notes called "Periods with start or end offsets"
 # In that section we will say: "If a Timing.repeat has a startOffset and/or an endOffset, it must have a period that indicates the duration the offset is with respect to.  In cases without offsets, the frequency indicates the number of occurrences that happen across the entire period.  If offsets are present, the frequency indicates the number of occurrences that happen across the period less the offsets.  For example, if period is 28 days with no offsets, a frequency of 21 would equate to "3 repetitions every 4 days" (somewhat unusual).  However, if a stopOffset of 7 days were present, that same frequency would now equate to "1 repetition per day" because the 'effective' period would be "28 days - 7 stop offset days = 21 days"."
 # We will correct the short description from "Events within the repeat period step once" to "Events within the repeat period st{*}o{*}p once"
