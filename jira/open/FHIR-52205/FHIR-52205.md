Issue Summary

The $docref operation's start date parameter definition in FHIR (US Core, IPA, and FHIR R6) is ambiguous, leading to implementation disputes. The current definition mixes references to "care dates" (relating to the care provided) and "record currency dates" (relating to document creation or update). Additionally, the behavior when no start or end date is provided is unclear, causing confusion about whether the "most recent document" or "records relating to the most recent encounter" should be returned.
Proposed Updated Definition

 

start:

The start date and end date parameters define the clinical date range (e.g., encounter or service dates) of the care documented. If no start date is provided, all clinical documents with care dates before the end date are in scope. If neither a start date nor an end date is provided, the operation returns clinical documents associated with the most recent encounter or care event. Clients SHOULD provide date values precise to the second with a time offset (e.g., 2025-09-01T00:00:00+10:00).

end:

The start date and end date parameters define the clinical date range (e.g., encounter or service dates) of the care documented. If no end date is provided, all clinical documents with care dates after the start date are in scope. If neither a start date nor an end date is provided, the operation returns clinical documents associated with the most recent encounter or care event. Clients SHOULD provide date values precise to the second with a time offset (e.g., 2025-09-01T00:00:00+10:00).

Revise the $docref operation documentation in US Core, IPA, and FHIR R6 to reflect the proposed definition.
