# FHIR-54404: Several typos on the search and filter pages

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/15/26
- **Description**: “hierarchial” spelled wrong

not-in is only allowed on token data types, yet the table also includes “reference”


https://hl7.org/fhir/6.0.0-ballot4/search.html#modifiertexttoken  
Space missing in header:
text with referenceor token type search parameters

https://hl7.org/fhir/6.0.0-ballot4/search.html#modifiertextstring
The example is using the parameter section (which is a token) rather than section-text.
Also uses a capital “OR”
Should be:
GET &#91;base&#93;/Composition?section-text:text=(bone or liver) and metastases


https://hl7.org/fhir/6.0.0-ballot4/search.html#has
Description of the example does not have the right code. Should say 1234-5 instead of 1234


https://hl7.org/fhir/6.0.0-ballot4/search_filter.html#:~:text=The%20interpretation%20of%20the%20operation%20depends
Second table (under the _filter section has the operation “Co”, and it should be “co”


https://hl7.org/fhir/6.0.0-ballot4/search.html#_revinclude#:~:text=GET%20[base]/MedicationDispense?_include=Medication:prescription
Seems to have the wrong parameter value for _include in the following (should be MedicationDispense rather than Medication)
GET &#91;base&#93;/MedicationDispense?_include=Medication:prescription
      &amp;_include:iterate=MedicationRequest:requester&amp;criteria...
