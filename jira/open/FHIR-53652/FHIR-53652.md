# FHIR-53652: Improve Narrative Accessibility Scope

## Metadata

- **Assignee**: Unassigned
- **Created**: 12/17/25
- **Description**: WCAG 2.1 AA Compliance
Implementations SHOULD strive for WCAG 2.1 Level AA compliance (applicable to narrative div):

	Provide text alternatives for non-text content
	Ensure content is keyboard accessible
	Use clear, hierarchical heading structure



Add Aria attributes

Key Advantage: ARIA attributes work with HTML 4.0 (they're layered on top), so adding them doesn't require moving to HTML5.

Current Gap: FHIR narratives use semantic HTML, but lack the ability to programmatically expose certain relationships and purposes to assistive technologies.

Use Cases in Clinical Narratives:
&lt;!-- Critical allergy alert --&gt;
&lt;div role="alert" class="fhir-alert fhir-allergy"&gt;
  &lt;strong&gt;SEVERE ALLERGY:&lt;/strong&gt; Anaphylactic reaction to Penicillin
&lt;/div&gt;

&lt;!-- Status information (lab results summary) --&gt;
&lt;div role="status" class="fhir-summary"&gt;
  &lt;p&gt;Complete Blood Count: 4 abnormal values detected&lt;/p&gt;
&lt;/div&gt;

&lt;!-- Major content regions --&gt;
&lt;div role="region" aria-labelledby="demographics-heading" class="fhir-section"&gt;
  &lt;h3 id="demographics-heading"&gt;Patient Demographics&lt;/h3&gt;
  &lt;!-- content --&gt;
&lt;/div&gt;

&lt;!-- Data table (redundant but helpful for complex tables) --&gt;
&lt;table role="table" aria-label="Laboratory Results - Complete Blood Count"&gt;
  &lt;thead&gt;
    &lt;tr role="row"&gt;
      &lt;th role="columnheader"&gt;Test&lt;/th&gt;
      &lt;th role="columnheader"&gt;Result&lt;/th&gt;
      &lt;th role="columnheader"&gt;Reference Range&lt;/th&gt;
    &lt;/tr&gt;
  &lt;/thead&gt;
  &lt;tbody&gt;
    &lt;tr role="row"&gt;
      &lt;td role="cell"&gt;WBC&lt;/td&gt;
      &lt;td role="cell"&gt;12.5 K/µL&lt;/td&gt;
      &lt;td role="cell"&gt;4.5-11.0 K/µL&lt;/td&gt;
    &lt;/tr&gt;
  &lt;/tbody&gt;
&lt;/table&gt;
Proposed Attributes:

	role (high impact)
	aria-label (high impact)
	aria-labelledby (high impact)
	aria-describedby
	aria-hidden



Consider (some) HTML5 Elements
Benefits are incremental (ARIA solves most accessibility gaps)

Proposed Elements:

	&lt;figure&gt; and &lt;figcaption&gt; (medical imaging)
	&lt;mark&gt; (highlight abnormal values)
	&lt;section&gt;, &lt;article&gt;, &lt;aside&gt; (document structure)



No RTL Language Support

The dir attribute is not in the allowed attributes list, yet it is essential for right-to-left languages. "dir" attribute SHALL have one of these values: "ltr", "rtl", "auto".
(Suggestion) Add Accessibility Guidance Section

Narratives SHOULD use semantic HTML elements to improve accessibility:

Headings:

&lt;h3&gt;Patient Demographics&lt;/h3&gt;
&lt;!-- Use hierarchical heading levels (h1-h6) --&gt;
Definition Lists (Key-Value Pairs):

&lt;dl&gt;
  &lt;dt&gt;Blood Pressure&lt;/dt&gt;
  &lt;dd&gt;120/80 mmHg&lt;/dd&gt;
&lt;/dl&gt;

Tables:

&lt;table&gt;
  &lt;caption&gt;Laboratory Results&lt;/caption&gt;
  &lt;thead&gt;
    &lt;tr&gt;
      &lt;th scope="col"&gt;Test&lt;/th&gt;
      &lt;th scope="col"&gt;Value&lt;/th&gt;
    &lt;/tr&gt;
  &lt;/thead&gt;
  &lt;tbody&gt;
    &lt;tr&gt;
      &lt;td&gt;Glucose&lt;/td&gt;
      &lt;td&gt;95 mg/dL&lt;/td&gt;
    &lt;/tr&gt;
  &lt;/tbody&gt;
&lt;/table&gt;
