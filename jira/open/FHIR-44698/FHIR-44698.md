# FHIR-44698: Inconsistent linking of resource types

## Metadata

- **Assignee**: Unassigned
- **Created**: 2/8/24
- **Description**: There are no links to Resource, DomainResource, MetadataResource or CanonicalResource on the Types page, except in the diagrams. The "Resources" link goes to the full list of resources.

Section 2.1.6.1.1 (https://build.fhir.org/uml.html#interfaces) indicates that there are abstract "interface" types. However, section 1.2 (https://build.fhir.org/patterns.html#1.2) does not list any of the interface types in its discussion of interfaces. (I suppose there may be a distinction between interface types and interface patterns, but given that both are fairly esoteric areas the duplicative language isn't helpful)

The Patterns page (https://build.fhir.org/patterns.html) does not show up in the documentation index.
