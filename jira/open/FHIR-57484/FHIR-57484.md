# FHIR-57484: update .code cnl-0 description

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/29/26
- **Description**: update .code description, short and comments

The phrase "natural language name" directly contradicts the cnl-0 regex.  The Short is awkwardly worded.  The Comments duplicate and underspecify the constraint.

Proposed updates:

 
Change
 
Definition: A natural language name identifying the title. This name should be usable as an identifier for the resource by machine processing applications such as code generation.
 
to 
 
Definition: A machine-friendly identifier suitable for code generation. For example, "ClinicalDocument" or "Composition_v2". See the cnl-0 constraint for the naming rules.
 
&#8212;


Short: Name for this title (computer friendly)
 
to
 
Short: Computer-friendly name
 
&#8212;


Comments: The name is not expected to be globally unique. The name should be a simple alphanumeric type no-whitespace name to ensure that it is machine-processing friendly.

to
 
Comments: The name is not expected to be globally unique.
 
update  cnl-0 description

cnl-0 's current description, "Name should be usable as an identifier for the module by machine processing applications such as code generation."  tells the reader why the rule exists but not what it actually checks. A reader encountering this warning has no way to know from the human text and must read the FHIRPath expression and decode the regex themselves.

 

Propose updating the description to:

 
"Name should be a machine-friendly identifier suitable for code generation: it must start with an uppercase letter (A–Z), be 2 to 255 characters long, and contain only letters, digits, and underscores. Whitespace, hyphens, and other punctuation are not permitted."
