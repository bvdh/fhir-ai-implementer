# FHIR-50997: Incorrect use of "DELETE" as an HTTP status code in example response Bundle.

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/28/25
- **Description**: The example at Example Bundle/bundle-response (JSON) shows the following response entries for DELETE operations:



 


{
  "response": {
    "status": "DELETE"
  }
} 

 

 

The use of "DELETE" as a value for response.status is not compliant with the FHIR specification. According to the Bundle - FHIR v5.0.0 definition, the value must be a valid HTTP status code string.

Using "DELETE" in this context appears to be a mistake—possibly a misinterpretation of the HTTP method instead of the HTTP response status.
