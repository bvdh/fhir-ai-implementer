# FHIR-52355: Return value for the $add and $remove operation

## Metadata

- **Assignee**: Unassigned
- **Created**: 9/4/25
- **Description**: The return value is defined as 'Resource containing content added/removed.' but there are no examples / extra details - it could be interpreted as the entire Group/List resource after the content has been added/removed, or a 'stripped' resource (similar to the one provided in input) listing only the content that has been actually added/remove.

Note that the second interpretation is much better, the first one would force the server to load and send the entire (large) resource, that sort of defeat the purpose of having operations like $add and $remove that operate only by difference. 

Requested change: add language and examples that specify that the expected return is a resource with just the added or removed elements.
