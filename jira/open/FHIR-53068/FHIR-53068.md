# FHIR-53068: Clarify conformance language of Coding.display

## Metadata

- **Assignee**: Unassigned
- **Created**: 10/6/25
- **Description**: The Coding data type has this conformance language regarding the value of the display element:

```
If the 'display' element is populated, the string used in display SHALL be one of the display strings defined for that code by the code system.
```

This language does not specify the behavior of the display element with respect to versions of the code system.

We should clarify the relationship of display to the code system version:

1. If no version is specified in the Coding, does this mean that the display is allowed to be from any version of the CodeSystem (as opposed to only the latest, or the version as specified in context (such as a parameter to the $validate-code operation))
2. If the version is specified in the Coding, does this mean that the display SHALL be from that version of the CodeSystem?

We should also clarify the interaction of this conformance language with the lenient-display parameter of the $validate-code operation.
