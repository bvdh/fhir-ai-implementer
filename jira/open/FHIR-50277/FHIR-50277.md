# FHIR-50277: Reinstate "unit" code in Questionnaire Item UI Control Codes System

- **Description**: "unit" was deprecated with advice to use the extension, [http://hl7.org/fhir/StructureDefinition/questionnaire-unit|http://hl7.org/fhir/extensions/StructureDefinition-questionnaire-unit.html.] This does not provide equivalent behaviour and request "unit" code is reinstated please.

The issue is that by using the display value of the valueCoding of the extension it doesn't not permit customisable user interface displays for units. It mandates the use of correct display terms for the concept as defined in a code system. E.g. ucum code "mm&#91;Hg&#93;" has display "millimeter of mercury", however, it would be preferrable to use a UI display of "mm Hg". The only way I know to be able to use this preferrable display is by defining a codesystem supplement. Seems quite onerous to do that when many ucum displays are not good for a UI. Thanks.

- **Resolution**: Persuasive
- **Resolution Description**: We will undeprecate the code.  We will change the description from:

"Display item is rendered adjacent (horizontally or vertically) to the answer space for the parent question, typically to indicate a unit of measure. *Deprecation Note* - The display element of the [http://hl7.org/fhir/StructureDefinition/questionnaire-unit|https://hl7.org/fhir/extensions/StructureDefinition-questionnaire-unit.html] should be used instead."

to

"Display item is rendered adjacent (horizontally or vertically) to the answer space for the parent question, typically to indicate a unit of measure.  In most circumstances, the display for the unit can be taken from the [http://hl7.org/fhir/StructureDefinition/questionnaire-unit|https://hl7.org/fhir/extensions/StructureDefinition-questionnaire-unit.html] and this display type will not be needed.  However, if there's a reason why the 'display' for the unit Coding is not the desired label to render when displaying the form, this display item can be used instead."
