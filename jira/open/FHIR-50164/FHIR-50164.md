# FHIR-50164: Missing invariant on Identifier

## Metadata

- **__EMPTY**: Jens Villadsen
- **__EMPTY_1**: Public
- **__EMPTY_2**: FWIW I am strongly against having the value on Identifier to dictate the system of the Identifier. I dont see what is gained by imposing this constraint on the system - e.g. apparently the following is not allowed:

{{
{"system": "http://abc.com","value": "urn:uuid:6eec04cd-ac0c-472b-a6f0-2ff177728b68"}
}}

I see no logical reason why it should not be allowed. If a system identify its entities by UUID then that system should be stated as the system in the Identifier - BUT - if that genuinely is considered wrong then a constraint should be in place.
