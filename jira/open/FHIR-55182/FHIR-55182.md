# FHIR-55182: Questionnaire.item.enableWhen.operator has no escape valve

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/20/26
- **Description**: The element Questionnaire.item.enableWhen.operator is a mandatory element with a required binding, but appears to have neither an 'unknown' or 'other' concept, which can make the containing element impossible to populate in some cases. Please add 'unknown', 'other', or some equivalent escape valve concept to the value set.

(Comment 148 - imported by: Lloyd McKenzie)

- **Issue Type**: Change Request
- **Key**: FHIR-55182

- **Related URL**: http://build.fhir.org/questionnaire.html
- **Reporter**: Lloyd McKenzie
- **Resolution**: Not Persuasive with Modification
- **Resolution Description**: We already have a 'standard' mechanism to handle more complex expressions.  We will add the following to the comments for enableWhen:

"If there is a need to express conditional rules more complex than can be expressed with the enableWhen structure, systems should use the [enableWhenExpression]() extension.  For example, logic that involves a mixture of and and or logic, checking for membership in a value set or within a range, etc."
