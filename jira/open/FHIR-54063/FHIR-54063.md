# FHIR-54063: Bundle.signature Usage in Finnish Kanta Specs

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/13/26
- **Description**: In reviewing the R6 Signatures spec, I figured I'd also write this up as a comment:

There's regulation in Finland that requires data be sent over FHIR, and part of that paradigm has involved making sure the data is signed - using Bundle.signature.

https://www.kanta.fi/en/system-developers/fhir-specification-for-electronic-signature

As far as I can tell, the new plan to have Provenances embedded in a Bundle (and not be a part of the data that is incorporated into the signature) should be able to work for them if they moved to R6, but it might be worth trying to start a dialogue with the relevant regulatory body directly. 

If Bundle.signature were to not be deprecated, it would probably be worth adding some canonicalization updates to call out how much of the Bundle.signature (if any) were incoprated into the signature.
