# FHIR-55488: Order of repeatable elements - Improve clarity

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/28/26
- **Description**: The section on cardinality says 

"For elements that have cardinality &gt; 1, the order in which they appear may have meaning. Unless the element definition (either in this specification or the extension) defines a meaning to the order explicitly (using ElementDefinition.orderMeaning), the meaning of the order is not defined, and implementations are allowed to reorder the elements. "

The last segment specifically has unclear meaning/impact. 
During a discussion on Dosage.steps and whether we need a sequence number (Integer), we looked at where does FHIR state that array order is preserved, and this paragraph mostly helps, but that last sentence caused doubts again. 

Proposal: this should be reworded like: 
"and applications should be aware that reordering the order of the elements change the meaning and should look at the element's orderMeaning."

Additionally - is it useful to indicate somehow when a 0..* element has orderMeaning set? Either not invasive like a hover on the "0..*" (currently has no tooltip), or an explicit icon next to the element?
