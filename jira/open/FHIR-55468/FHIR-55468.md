# FHIR-55468: We should force that all required and extensible THO bindings are fixed version

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/27/26
- **Description**: There's an issue of conformance for both required and extensible bindings if the value set isn't stable.  With required, you're not allowed to send anything not in the value set, so if the content changes, that breaks stuff.  With extensible you're required to map and not send local codes or free text in place of the bound codes unless none exist - if what exists is dynamic, that's impossible for implementers to cope with.
