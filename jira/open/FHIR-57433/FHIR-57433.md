# FHIR-57433: Clarify global inheritance expectations

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/23/26
- **Description**: We need to make clear what the expected behavior is for 'global' profile declarations.  I believe they must be inherited by derived IGs, and if re-specified for the same type in a derived IG, they must be the same or a proper constraint on the parent profile.  However, this isn't made clear anywhere.  It could appear on the profiling page, but I think the best place is probably a comment on the 'global' element itself.
