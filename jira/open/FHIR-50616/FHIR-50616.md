# FHIR-50616: Clarification Questions for SubscriptionTopic.trigger.notificationShape.relatedQuery

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/2/25
- **Description**: The SubscriptionTopic resource defines a relatedQuery element and has associated narrative in section 2.14.6.8.3 (and is similarly defined in SubscriptionStatus), but I'm unclear on how this element should be used:

 

1) RelatedQuery is defined on SubscriptionTopic and SubscriptionStatus, but not Subscription. Are RelatedQueries hardcoded by the server, and not something that each subscriber can ask for/pick between a set of options?

 

2) Are there standard tokens to use in the query itself, for referring to the focal resource? (e.g. requesting all conditions for a patient notification via Condition?patient={focusResourceFHIRID}, like {context.patientId} in CDS Hooks)

 

3) Is there an example binding for relatedQuery.queryType? What type of codings are expected there?
