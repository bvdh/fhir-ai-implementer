# FHIR-55975: Add best practice invariants that warn about missing code (and system?) on codings

## Metadata

- **Assignee**: Unassigned
- **Created**: 2/26/26
- **Description**: Sometimes authors use Coding when they should be using string.  While this isn't prohibited, we should nudge them towards better behavior.
- **Issue Type**: Change Request
- **Key**: FHIR-55975
- **Related Artifact(s)**: Questionnaire
- **Reporter**: Lloyd McKenzie
- **Resolution**: Persuasive
- **Resolution Description**: 1. On answerOption.valueCoding, enableWhen.answerCoding and initialValue.valueCoding, will add a "best practice" invariant if the type is 'Coding' that says "When using codings, both code and system should be present.  See FHIR specification Questionnaire 'Using Coded Answers' usage notes for guidance."

2. Will add a section in the usage notes for Questionnaire titled "Using Coded Answers" with the following content:

It's not uncommon for legacy questionnaires to have "coded" questions that have a code but no system or sometimes even just a display value.  While this is not prohibited, it's not good practice.

A Coding with no code cannot be meaningfully computed on, and a Coding with no system has no semantic meaning.  If only a display is available, consider using the 'string' type.  If the codes are 'made up', simply define a CodeSystem url based on the Questionnaire url and itemId or some other unique string."

3. Will add pointers to this from answerOption.value, enableWhen.value, and initialValue.value in the comments saying "See [here]() for additional guidance on using Coding in Questionnaires."
