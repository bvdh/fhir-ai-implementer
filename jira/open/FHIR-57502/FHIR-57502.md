# FHIR-57502: Replace Request.replaces with Request.releatesTo (instead of the changes in FHIR-54876)

## Metadata

- **Assignee**: Unassigned
- **Created**: 6/3/26
- **Description**: https://jira.hl7.org/browse/FHIR-54876 changes the meaning of Request.replaces in ways that are backwards incompatible, confusing, and likely dangerous.

Given the historically entrneched use of document relationships, where "replaces" always invalidates the original document, it is better to use the same pattern as DocumentReference.relatesTo.
