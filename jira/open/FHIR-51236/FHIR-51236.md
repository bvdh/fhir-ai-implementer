# FHIR-51236: Handling of Attachment.hash

## Metadata

- **Assignee**: Unassigned
- **Created**: 6/25/25
- **Description**: The encoding of Attachment.hash has been clarified on this Zulip thread: Attachment.hash and hash representation. The raw bytes of the hash SHALL be base64-encoded, not the hexadecimal representation or any other conversion.

That could be clarified in the description of the hash property.

In the resource examples using an Attachment.hash, the following values are providing invalid examples:
