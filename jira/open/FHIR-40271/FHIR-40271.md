# FHIR-40271: FHIR is() example use non-singleton examples

- **Assignee**: Unassigned
- **Comments**: 1/18/23
- **Created**: 1/18/23
- **Description**: The is() examples on the FHIRPath page are misleading because they use non-singleton inputs, and so they are likely to blow up in practice. The examples should use singleton inputs (family in this case)
- **Grouping**: [Fit-For-Teleconference, block]
- **Issue Type**: Change Request
- **Key**: FHIR-40271
- **Related Page(s)**: FHIRPath (fluentpath)
- **Reporter**: Grahame Grieve
- **Resolution**: Not Persuasive with Modification
- **Resolution Description**: While this request is correct, the *is()* function is present in FHIRPath for legacy support, we should be encouraging the use of the *ofTye()* function, which is the plugin replacement for is() that DOES handle collections.

This should be changed for all uses of the is() function on the page 
(including Patient.is(FHIR.Patient) at the very start of the section)

The examples that are using is() to perform a boolean type check, not a cast (or example of using the type style parameter) should be corrected to be inside a select so that errors don't come from having collections - don't show examples of bad practice
- **Status**: Triaged
- **Summary**: FHIR is() example use non-singleton examples
