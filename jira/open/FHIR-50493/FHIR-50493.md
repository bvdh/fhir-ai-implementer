# FHIR-50493: "reference" search parameter type no longer points to Resource

## Metadata

- **Assignee**: Unassigned
- **Created**: 4/30/25
- **Description**: The addition of the resource search parameter type makes this line about the reference search parameter type no longer true:

"

The reference search parameter type is mostly used for resource elements of type Reference or canonical. However, it is also be used to search resource elements of type Resource - i.e. where one resource is directly nested within another - see the Bundle search parameters 'message' and 'composition' as an example of this.

"

So, this line should be removed - both these examples now use the resource parameter type.
