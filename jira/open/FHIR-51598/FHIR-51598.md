# FHIR-51598: Imprecise sentence - No chaining AFTER :identifier modifier

## Metadata

- **Assignee**: Gino Canessa
- **Created**: 7/30/25
- **Description**: Within the specification, I found the following sentence confusing:
Chaining is not supported when using the :identifier modifier, nor are chaining, includes or reverse includes supported for reference elements that do not have a reference element.
(source)

As written, it sounds as if, in general, chaining and the :identifier modifier should not be combined.
Therefore, a query like the following should be forbidden—although this is not the case:
GET &#91;base&#93;/DocumentReference?encounter.account:identifier=7567867

We have also received similar questions and comments from other implementers.

Thus, we would like to propose changing the sentence as follows:

"When using the :identifier modifier, chaining after the modifier is not supported. Additionally, chaining, includes, or reverse includes are not supported for Reference type elements that do not have a reference element."

A thread with a corresponding discussion was opened her: https://chat.fhir.org/#narrow/channel/179166-implementers/topic/No.20chaining.20AFTER.20.3Aidentifier.20modifier/with/527619873

- **Resolution**: Persuasive with Modification
- **Resolution Description**: Will change:

"Chaining is not supported when using the :identifier modifier, nor are chaining, includes or reverse includes supported for reference elements that do not have a {{reference}} element."

to

"Chaining is not expected to be supported when using the :identifier modifier and is expressly prohibited following a :identifier modifier.  E.g. {{DocumentReference?encounter.account:identifier.account-holder=abc}} is not permitted.   As well, chaining, includes or reverse includes are not supported for reference elements that do not have a {{reference}} element."
