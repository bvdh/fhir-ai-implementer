# FHIR-56402: Clarify what's allowed for implicitRules

## Metadata

- **Assignee**: Unassigned
- **Created**: 4/20/26
- **Description**: We need to be clear whether implicitRules is allowed to break syntax rules
- **Issue Type**: Change Request
- **Key**: FHIR-56402
- **Related Artifact(s)**: BaseResource
- **Reporter**: Lloyd McKenzie
- **Resolution**: Persuasive with Modification
- **Resolution Description**: We will add the following to the comment for implicitRules:

"An implicitRule *{*}MAY{*}* break syntax validity of an instance.  However, this is only done in extremely limited circumstances.  An implicitRule *{*}SHALL{*}* only break syntax validity if it is defined by HL7 international and is rooted in "http://hl7.org".

We will make the same change in the additionalImplicitRules extension.
- **Status**: Resolved - change required
- **Summary**: Clarify what's allowed for implicitRules
