# FHIR-57533: Modify Composition to function as a Bundle wrapper for a DiagnosticReport

## Metadata

- **Assignee**: Unassigned
- **Created**: 6/4/26
- **Description**: We want to minimize the overhead cost of introducing a Composition wrapper for document Bundles of DiagnosticReport. To that end, an extension has been created (refer to Bas JIRA) to enable a reference from a Composition directly to a DiagnosticReport that avoids having to reference from a .section.entry which would be misleading for recipients and otherwise has no meaning.

As this will be a common requirement (if JIRA One is not accepted) we recommend this be added as an attribute with the same definition, and add guidance/constraints that when used the variances identified in the attached spreadsheet are addressed to ensure the values for the respective attributes in the Composition and DiagnosticReport cannot be different.

For a Composition wrapper that duplicates DiagnosticReport content to facilitate using Composition Search for a few DiagnosticReport Searches (full functionality search would still use DiagnosticReport Search)

	Modify the definition of Composition.date to say that it is a date representative of the content referenced by the Composition rather than the date of the Composition itself, and remove references to when the Composition resource was edited, authored, or logically changed.
	Add element Composition.effective with similar definition to DiagnosticReport.effective
	Duplicate DiagnosticReport1.text into Composition1.text
	Record Composition1.derivedFrom = DiagnosticReport1
	Duplicate DiagnosticReport1.category into Composition1.category
	Duplicate DiagnosticReport1.basedOn into Composition1.basedOn
	Duplicate DiagnosticReport1.subject into Composition1.subject
	Populate Composition1.author with the device/software that generated the Composition during bundling.
	Populate Composition1.title from the display name of DiagnosticReport1.code
	Duplicate DiagnosticReport1.note into Composition1.note
	Populate Composition1.attester.party from DiagnosticReport1.resultsInterpreter
	Populate Composition1.attester.mode = professional
	Populate Composition1.attester.time from DiagnosticReport1.issued
	Duplicate DiagnosticReport1.relatesTo into Composition1.relatesTo
	Duplicate DiagnosticReport1.encounter into Composition1.encounter
	Duplicate DiagnosticReport1.effective into Composition1.event.period.start
	Populate Composition1.event.period.end with an estimate of the end time of the imaging acquisition.



For a minimal valid Composition wrapper just used to enable document Bundle while continuing to prohibit DiagnosticReport as the first entry:

	Populate Composition1.status by mapping from equivalent DiagnosticReport1.status
	Duplicate DiagnosticReport1.code into Composition1.type
	Composition.diagnosticReport = Reference (DiagnosticReport)
