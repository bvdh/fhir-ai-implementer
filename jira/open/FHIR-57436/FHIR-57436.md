# FHIR-57436: Introduce an `otherwise` clause to source rules in FML

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/25/26
- **Description**: The source has the ability to filter content using where expressions or type filters, but no way to easily access the remaining values.

This is particularly useful when manipulating some content, but permitting the remaining content to pass through.

Currently to achieve this you need to craft a where clause that captures the negative of all the where clauses.

This is what

src.a where  -&gt; tgt.a;
src.a where  -&gt; tgt.b;
src.a : Quantity -&gt; tgt.q;
src.a where (.not() and .not() and ofType(Quantity).not()) -&gt; tgt.c; // this is the otherwise rule, very wordy and sensitive to updates
src.a otherwise -&gt; tgt.c; // this would be the new way to express it
