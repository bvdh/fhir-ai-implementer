# FHIR-57407: Revert the changes to RelatedArtifact

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/20/26
- **Description**: Reviewing changes to RelatedArtifact, we see that this is not a sensible set of changes - we're deprecated the useable fields, added another one which doesn't work as well (it only allows one at a time).

We propose reverting the changes - specifically, removing artifact&#91;x&#93;, add resourceReference per R5, and don't deprecate the existing fields.

This does leave url and document.url as potentially duplicative, but Attachment.url is a reference to a fixed set of bytes, where as url is not

- **Key**: FHIR-57407

 # We will un-deprecate what's marked as deprecated in R5 (citation, resource, resourceReference, document)
 # We will add url back.
 # We will add guidance covering the apparent overlap between url and Attachment.url
