# FHIR-46200: Note in ElementDefinition.comment is incorrect

## Metadata

- **Description**: The definition of ElementDefinition.comment says: "Explanatory notes and implementation guidance about the data element, including notes about how to use the data properly, exceptions to proper use, etc. (Note: The text you are reading is specified in ElementDefinition.comment)."

However, that particular text is in fact specified in the ElementDefintion.definition:



    &lt;element id="ElementDefinition.comment"&gt;
[..]
      &lt;path value="ElementDefinition.comment"/&gt; 
      &lt;short value="Comments about the use of this element"/&gt; 
      &lt;definition value="Explanatory notes and implementation guidance about the data element, including
       notes about how to use the data properly, exceptions to proper use, etc. (Note:
       The text you are reading is specified in ElementDefinition.comment)."/&gt; 
      &lt;comment value="If it is possible to capture usage rules using constraints, that mechanism should
       be used in preference to this element."/&gt;
