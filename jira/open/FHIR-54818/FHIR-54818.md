# FHIR-54818: remove reference and link to Confluence interversion compatibility discussion

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/19/26
- **Description**: The following statement and link to a non-normative Confluence page from 2019 SHOULD NOT be in a normative part of this specification. There is no guarantee that the content of that Confluence page will continue to exist or change and thereby change/break the inter-version rules set forth in this normative specification.

"Additional discussion on inter-versioning issues can be found here: https://confluence.hl7.org/display/FHIR/Interversion+Compatibility  ."
- **Issue Type**: Change Request
- **Key**: FHIR-54818
- **Related Page(s)**: Change Management & Versioning (versions/compatibility)
- **Related URL**: https://hl7.org/fhir/6.0.0-ballot4/versions.html#deprecation
- **Reporter**: Chris Melo
- **Resolution**: Persuasive with Modification
- **Resolution Description**: Will remove the sentence, though for a different reason (the referenced content is old & out-of-date).  It's fine for this specification to point to external documents for "additional considerations".  Such content has no impact on conformance.  However, there are places we have external references that are intended to have conformance impact and we're not always super clear in differentiating.

We will define a style guide that ensures that external references clearly disambiguate whether the reference is informative or intended to have conformance weight and will review the external references in the core spec to ensure they comply with this guidance.
- **Status**: Resolved - change required
- **Summary**: remove reference and link to Confluence interversion compatibility discussion
