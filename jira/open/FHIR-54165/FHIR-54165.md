# FHIR-54165: Constraint docRef-2 Logic Error

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/13/26
- **Description**: Constraint docRef-2 Logic Error

The constraint docRef-2 states:
"practiceSetting SHALL only be present if context is not present"
But the FHIRPath expression is:
practiceSetting.empty() or context.where(resolve() is Encounter).empty()
The description says "if context is not present" but the expression says "if context is not an Encounter". These are different conditions:

	"not present" means context.empty()
	"not an Encounter" means context.where(resolve() is Encounter).empty()



The constraint description doesn't match the FHIRPath expression. The FHIRPath allows practiceSetting when context references an Appointment or EpisodeOfCare, but the description implies practiceSetting should only be present when context is completely absent.

 

Propose changing the description to "practiceSetting SHALL only be present if context not an encounter"
