# FHIR-53764: spd-2: SearchParameter.chain should be allowed for "resource" param type

## Metadata

- **Assignee**: Unassigned
- **Created**: 12/31/25
- **Description**: For SearchParameter, invariant spd-2 requires:

chain.empty() or type = 'reference'

This should also include the "resource" data type (for example, Bundle's "composition" search parameter), which are described as using chaining in their definition (https://build.fhir.org/search.html#resource):

chain.empty() or (type in ('reference' | 'resource'))

The long Definition for SearchParameter.chain would also need to be updated accordingly.

For consistency with invariant spd-3, you could also consider adding 'special' as a valid type to this list, although I personally am not sure if there's a use case for that.
