# FHIR-46453: dateTime RegEx allows to specify a date & time without timezone

## Metadata

- **Assignee**: Unassigned
- **Created**: 7/24/24
- **Description**: The dateTime RegEx seems to have changed in R5 and, unlike in R4 and earlier, now validates the following dateTime, which should be invalid because it doesn't have a timezone:

2024-07-23T12:27:16.321661

This was fixed between DSTU1 and DSTU2 with https://jira.hl7.org/browse/FHIR-8889 and seems to have regressed in R5.
