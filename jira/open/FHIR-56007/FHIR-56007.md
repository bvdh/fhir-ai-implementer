- **Description**: It is unclear, how an Operation without specified out Parameters is supposed to behave or whether having no out parameters is even legal.

See discussion: #conformance &gt; OperationDefinition without out-Parameter
- **Issue Type**: Change Request
- **Key**: FHIR-56007

- **Reporter**: Simone Heckmann
- **Resolution**: Persuasive with Modification
- **Resolution Description**: After reviewing https://build.fhir.org/operations.html#executing and https://build.fhir.org/operations.html#response , it is clear that there are two options for an operation that does not return an {{out}} parameter when successfully completing:
* return an empty body
* return an empty {{Parameters}} resource

We will update both of the above sections to clarify the allowed behaviors.


Note that there is a TC that should be applied as well
> containing ne or more output parameters
to 
> containing one or more output parameters
