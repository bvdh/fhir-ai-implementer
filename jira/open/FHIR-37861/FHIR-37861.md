# FHIR-37861: FHIRPath is and as samples use improper case for System datatypes

## Metadata

- **Assignee**: Unassigned
- **Created**: 8/5/22
- **Description**: https://www.hl7.org/fhir/fhirpath.html (and https://www.hl7.org/fhir/fhirpath.html) provide the follow example expressions:
Patient.name.given.is(System.string).not();
Patient.name.given.getValue().is(System.string);
Patient.name.given.as(System.string);
However, I believe that FHIRPath defines the System data types using a capital letter (both in the table just above this and also at http://hl7.org/fhirpath/N1/#primitive-types).

 

Based on http://hl7.org/fhirpath/N1/#case-sensitivity I believe FHIRPath types are case-sensitive.

Therefor, I believe these should be updated to the following:
Patient.name.given.is(System.String).not();
Patient.name.given.getValue().is(System.String);
Patient.name.given.as(System.String);

- **Resolution Description**: Update the examples as following:
{{Patient.name.given.is(FHIR.string) // true}}
{{Patient.name.given.is(System.String).not() // true}}
{{Patient.name.given.getValue().is(System.String) // true}}

And also

{{Patient.name.given.ofType(FHIR.string) // "given name value"}}
{{Patient.name.given.ofType(System.String) // {} (empty)}}



And also annotate this example, as that's why it has been added:
Patient.id is System.String   // true ; Although the Resource.id property appears to be is marked as a FHIR id, it is actually a simple string with no extensions, and is an attribute in the XML representation. It is constrained by the rules of the id datatype.
Patient.id is FHIR.id         // false ;
