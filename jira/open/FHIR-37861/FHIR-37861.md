# FHIR-37861: FHIRPath is and as samples use improper case for System datatypes

## Metadata

- **Assignee**: Unassigned
- **Created**: 8/5/22
- **Description**: https://www.hl7.org/fhir/fhirpath.html (and https://www.hl7.org/fhir/fhirpath.html) provide the follow example expressions:
Patient.name.given.is(System.string).not();
Patient.name.given.getValue().is(System.string);
Patient.name.given.as(System.string);
However, I believe that FHIRPath defines the System data types using a capital letter (both in the table just above this and also at http://hl7.org/fhirpath/N1/#primitive-types).

 

Based on http://hl7.org/fhirpath/N1/#case-sensitivity I believe FHIRPath types are case-sensitive.

Therefor, I believe these should be updated to the following:
Patient.name.given.is(System.String).not();
Patient.name.given.getValue().is(System.String);
Patient.name.given.as(System.String);
