- **Resolution Description**: 1. We will define a new extension called: "version-resolution-method" 0..1 code with a context of canonical

Definition: "If a canonical reference does not declare a specific version, indicates the algorithm to use to determine which version should be used."

Comment: "See discussion on canonical version regulation [here]()"

Allowed codes:

"package" "package-based" "The canonical is determined based on the version of the resource as defined in the package hierarchy in which the referencing artifact is defined, where the 'nearest' definition is the version used.  See further details [here]().  This is the default when the referenced artifact is defined in a package, though it will only come into play if the referenced artifact has any versions declared in the package hierarchy of the referencing artifact."

"latest" "latest" "Regardless of packaging of the referencing artifact, resolution is to the version that is determined to be the highest value according to the version algorithm for the resource.  (Note that this is *not* necessarily the one with the most recent publication date.)"

"manifest" "manifest-based" "The determination of the version is explicitly declared by a manifest made available in the scope of the resolution.  If a manifest is not available, or the referenced resource does not have a manifest entry, then resolution is not possible."
