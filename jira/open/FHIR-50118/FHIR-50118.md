# FHIR-50118: bdl-3b text doesn't align with fhirpath

## Metadata

- **Assignee**: Unassigned
- **Created**: 3/26/25
- **Description**: bdl-3b says For collections of type history, all entries must contain request or response elements, and resources if the method is POST, PUT or PATCH, while the fhirpath constraint is:

type = 'history' implies entry.all(request.exists() and response.exists() and ((request.method in ('POST' | 'PATCH' | 'PUT')) = resource.exists()))

There is a discrepancy between the two.
