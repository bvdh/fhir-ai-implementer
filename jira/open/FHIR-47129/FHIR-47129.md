# FHIR-47129: Duration description and expression conflict

## Metadata

- **Assignee**: Bas van den Heuvel
- **Created**: 9/6/24
- **Description**: The following description/expression of the "drt-1" constraint for Duration data type don't match at: https://www.hl7.org/fhir/datatypes.html#QuantityVariations

Description: "There SHALL be a code if there is a value and it SHALL be an expression of time. If system is present, it SHALL be UCUM."
Expression: "code.exists() implies ((system = %ucum) and value.exists())"
