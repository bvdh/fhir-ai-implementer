# FHIR-56636: confusing constraint on SearchParameter.name

## Metadata

- **Assignee**: Unassigned
- **Created**: 4/29/26
- **Description**: SearchParameter.name is constrained by : 

	spd-0 and inv-0 in R4,
	spd-0 in R4B
	cnl-0 in R5 en continuous build



with slight differences between invariant but overall, name SHOULD :

	starts with upper case letter
	contains only upper/lower case letter, number and '_'
	contains 255 caracters max



However, all the existing SearchParameter.name start with lower case ! (https://hl7.org/fhir/R4/search-parameters.json)

In R6, the naming convention (5.6.6) states that : Use lowercase letters with hyphens for multi-word parameters (kebab-case), in contradiction with invariant. __ 

 

I don't know what is the best way to resolve the inconsistency, may be discarding the invariants, but I think something need to be done.
