# FHIR-55501: Issues in mapping transforms table

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/29/26
- **Description**: Resolution of https://jira.hl7.org/browse/FHIR-42980 misses an issue brought up in the comments.

Several of the entries in the table of transform parameters (section https://build.fhir.org/mapping-language.html#7.4.0.8.2) include parameters with a question mark. The question mark's meaning is unclear: does this indicate uncertainty about whether the item is a parameter, or something else (optionality perhaps)?

If the question marks have a meaning it should be explained adjacent to the table. If the question marks indicate uncertainty or further editorial work is needed, then that work should be done.

 

Also, there appear to be errors in the html for the FHIRpath entries of cc, c, qty, id, cp, causing the entry to be rendered as, ,e.g., 



%factory.ContactPoint()"&gt;





I suspect the closing quote, and greater-than sign should not be rendered.
