# FHIR-53729: Improve seemingly inconsistent requirement in List resource Narrative Content Section

## Metadata

- **Assignee**: Unassigned
- **Created**: 12/29/25
- **Description**: Current Wording:
Each List.item should appear in the narrative for the resource; i.e. it SHALL NOT be necessary to retrieve the list items in order to have a human-readable rendering of the content.
This is confusing. It uses a lower case "should" to say what is in the narrative, but then uses a rephrase "i.e." to say something "SHALL NOT", which is upper case confirmative language. It is unclear how a system/implementer may ignore the recommendation of including each List.item in the narrative (therefore not include one or more List items in the narrative), yet not violate the conformance statement (it seems it would be necessary to retrieve the list items in order to create a human readable rendering).
