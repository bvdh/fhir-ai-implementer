# FHIR-55573: Clarify signature base64 encoding note proposed in JIRA

## Metadata

- **Assignee**: Unassigned
- **Created**: 2/2/26
- **Description**: The resolution to FHIR-54064 Clarify base64 encoding in Json Signature Requirements - Jira says that the JWS is base-64 encoded. 

 

Can we change the note from:

NOTE: Because JWS is already base-64 encoded, this means that the signature will be double-encoded.

To:

NOTE: Because JWS is already base64url encoded, this means that the signature will be double-encoded.

 

Emphasis not needed.

 

Primary reason is that base64url and base64 are meaningfully different from my understanding. And even though the note adds the technical clarification needed (the data will be double encoded), I could see the current language leading to folks not wanting to double encode because it says JWS is base-64 encoded and the data needs to be base64 encoded - which leads to the natural question of why does it need to be double encoded if they're already encoded the same way.
