# FHIR-54962: Type wrangling and default groups - clarifications

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/20/26
- **Description**: Please expand on the text in the the type wrangling section to elaborate on why string representation is (or could be) relevant to type conversions.

What happens if the types can't be mapped, is that an error? Should the engine log and skip the property, or throw and fail the entire map. Is this a configuration option for an engine?

This section should also cover that the type wrangling could also consider groups that map between the 2 types - groups that have the stereotype &lt;&lt;types&gt;&gt; - refer forward to the default mapping groups.

Move the Default mapping groups to be a child of the groups section (then the above can refer back rather than forward) - as this appears to be where the difference between types and type+ is defined

This section and the default mapping groups should be referenced in the target transform section to further elaborate on how data is copied from source to target.
