# FHIR-55601: Add an "otherCode" extension

## Metadata

- **Assignee**: Unassigned
- **Created**: 2/4/26
- **Description**: When a required binding has an "other" code, an extension to be used to provide the "other" code is suggested. However, there is not a general extension for that purpose. Consider adding "otherCode" as an extension that can be used routinely to specify the "other" code for required bindings.
- **Issue Type**: Change Request
- **Key**: FHIR-55601
- **Related Page(s)**: Datatypes Extensions
- **Related URL**: https://hl7.org/fhir/extensions/extension-registry.html
- **Reporter**: Bryn Rhodes
- **Resolution**: Persuasive
- **Resolution Description**: Define an otherCode extension that can be used when an "other" code appears in a required binding to specify what the actual code is:

otherCode 0..1 CodeableConcept, usable on code
Short: Indicates the actual code for an "other"
Definition: Used on code elements with a required binding that includes an "other" value to provide the actual coded concept.
- **Status**: Triaged
- **Summary**: Add an "otherCode" extension
- **Work Group**: FHIR Infrastructure
