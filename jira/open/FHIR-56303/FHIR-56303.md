- **Description**: It would be nice if the validator didn't error or warn on the fact that this extension contains content that looks like HTML (because that's the point).  However, before we could safely suppress such messages, we'd first need to enforce that the HTML within this extension is actually safe.  We should add a constraint on the 'value&#91;x&#93;' as follows:

```

      &lt;constraint&gt;
        &lt;key value="xhtml-ext-1"/&gt;
        &lt;severity value="error"/&gt;
        &lt;human value="The XHTML contained here SHALL abide by the same constraints as narrative.  I.e. contain only the basic html formatting elements and attributes described in chapters 7-11 (except section 4 of chapter 9) and 15 of the HTML 4.0 standard, &lt;a&gt; elements (either name or href), images and internally contained style attributes"/&gt;
        &lt;expression value="htmlChecks()"/&gt;
      &lt;/constraint&gt;

```
