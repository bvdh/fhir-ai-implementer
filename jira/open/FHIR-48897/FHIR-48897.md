# FHIR-48897: Add examples showing polling vs. subscription

## Metadata

- **Assignee**: Vassil Peytchev
- **Created**: 11/25/24
- **Description**: Receiving updates about Tasks (either there's a new one or someone's made an update to an existing one) is a foundational part of all of the Task-based workflows.  We should include some examples of what that process looks like when polling and when using subscription.  On the subscription side, we should show two variations - one with the full workflow including discovering subscription topics and creating the subscription might happen out-of-band, as well as a 'minimal' workflow where the initial steps of discovery and subscription creation happen out-of-band.  These examples should live in the Examples tab alongside the other examples we plan to create.
