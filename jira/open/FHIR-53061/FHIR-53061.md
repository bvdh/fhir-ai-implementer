# FHIR-53061: List.source needs to be 0..*

## Metadata

- **Assignee**: Unassigned
- **Created**: 10/6/25
- **Description**: The List.source element now supports multiple agent types, but does not support multiple entries. Thus when a Practitioner and Organization are known, this can't be saved. Yes the PractitionerRole is available, but this only works where this resource exists. There are also other cases where multiples are needed. 

Please allow .source to be 0..*
