# FHIR-53267: Extend context of use for package-source

## Metadata

- **Assignee**: Unassigned
- **Created**: 10/28/25
- **Description**: The package source extension allows the source package of an artifact to be declared on that artifact. However, there are use cases where we do not have control over the source of the artifact and need to capture package source as part of a dependency walk, rather than relying on it being present in the resource.

Expand the context of use to allow package-source to be specified on a RelatedArtifact so that we can capture package-source during a dependency walk. Note that we specifically need it to appear on RelatedArtifact, rather than say, canonical, because a dependency may be Reference, a canonical, or a non-canonical artifact specified with a uri (via the artifactUriReference extension)
