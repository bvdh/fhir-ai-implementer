# FHIR-56014: _summary=count overlaps with _count=0&_total=accurate

## Metadata

- **Assignee**: Unassigned
- **Created**: 3/5/26
- **Description**: There are 2 ways to express that you only want the number of results in a search:

	`_summary=count`
	`_count=0&amp;_total=accurate`



Although they can't really contradict each other, it is cleaner to have a single way of expressing this.

Additionally, `count` is a value for the _summary parameter that is only valid for search, whereas other values are also valid for other interactions (e.g., read).

Consider removing `count` as a value for `_summary`.
