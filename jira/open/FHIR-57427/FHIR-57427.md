# FHIR-57427: authoredOn is named confusingly

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/22/26
- **Description**: "authoredOn" seems to be named confusingly given that is defined in ServiceRequest as the date the request was "signed" (not written or authored). This gets even more confusing now that the O&amp;O working group has decided to create a separate technical 'creationDate'. I would suggest renaming "authoredOn" to 'authorizedOn' to clarify the difference between these two elements as well as the intention of this element on its own.
