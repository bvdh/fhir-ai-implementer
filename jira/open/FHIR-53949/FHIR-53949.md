# FHIR-53949: fhirpath page refers to functions as operations

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/12/26
- **Description**: The fhirpath page refers to functions as operations, this is not the terminology the fhirpath language uses. (search for "operation" on the page)
Such as: * "the parameter to the is() operation"

	"The primary exception is the is() operation"
	 all FHIR primitives have the operation getValue()



Each of these should be referred to as a function, not an operation.
(However don't change the instances where is explicitly referencing a fhir api operation, such as referring to the terminology service operations by the defined fhirpath function)
