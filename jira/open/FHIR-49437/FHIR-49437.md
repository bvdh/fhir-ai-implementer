# FHIR-49437: Common search parameters for pattern elements

- **Description**: Several events and request have different element names and the search parameters are correspondingly different across resources. 

While the meaning may slightly vary across resources, current situation prevents from doing searches across resource types e.g. ?_type=NutritionOrder,MedicationRequest,CarePlan&amp;recorder=DrWho

Propose to add common search parameters to the patterns which apply to all resources that follow those patterns.
- **Grouping**: [Workflow]
- **Issue Type**: Change Request
- **Key**: FHIR-49437
- **Related Artifact(s)**: Event Pattern

Request Pattern
- **Reporter**: Jose Costa-Teixeira
- **Resolution**: Persuasive with Modification
- **Resolution Description**: We will ensure that there are common search parameters for all pattern elements that are commonly searched.  Specifically:

Request - identifier, basedOn, replaces, groupIdentifier, status, intent, priority, doNotPerform, category, code, product, subject, encounter, occurrence, authoredOn, requester, performer, performerType

Event - identifier, basedOn, partOf, researchStudy, status, category, code, product, subject, encounter, occurrence, performer, performerType, location

Definition: (use common canonical/metadataresource search parameters) + performerType, code, product)

Will use the resource-pattern mappings to determine what element(s) to use for the search parameters for each resource and will use existing core extensions where there aren't core elements but there are extensions.
