# FHIR-38631: Which FHIR types are available from FHIRPath?

## Metadata

- **Assignee**: Unassigned
- **Created**: 9/14/22
- **Description**: http://hl7.org/fhir/fhirpath.html#types defines the types ("primitive data types, data types, resources") in the FHIR namespace for FHIRPath.

However, it doesn't actually list the types here, which makes me wonder

	are profiled datatypes like SimpleQuantity and MoneyQuantity included?
	are patterns/interface names included?



 

One place this matter is in the FHIRPath `is` and `as` operators.

For example, for `is`, its defined as follows:

&gt; If the left operand is a collection with a single item and the second operand is a type identifier, this operator returns true if the type of the left operand is the type specified in the second operand, or a subclass thereof.

 

It would be good for this core spec page to describe what "subclass" means here in the context of FHIR types (especially datatypes), because most of the current implementations get this wrong.

 

Based on the discussion at https://chat.fhir.org/#narrow/stream/179266-fhirpath/topic/FHIRPath.20.60is.60.20and.20.60as.60.20with.20FHIR.20types this is a place where the conformance tests should be improved.
