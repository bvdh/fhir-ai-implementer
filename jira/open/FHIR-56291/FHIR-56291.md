# FHIR-56291: Unspecified separator and escape character for searching

## Metadata

- **Assignee**: Unassigned
- **Created**: 4/7/26
- **Description**: This ticket was raised as a result of tickets https://jira.hl7.org/browse/FHIR-54778 and https://jira.hl7.org/browse/FHIR-54806. Here, it was noted that in the applicable search sections (ImagingStudy - FHIR v6.0.0-ballot4 and ImagingSelection - FHIR v6.0.0-ballot4) every Multiples row of a search parameter contains the same issues:

	The second bullet specifies how the multiples are to be separated, while it is undefined in the first bullet. &#91;This is actually not an issue, as the parameter (including value!) may repeat, being separated the HTTP-way, namely with an ampersand.&#93;
	It is not clear how the separator (supposedly a comma) is to be escaped when needed in a value.



While looking to resolve this issue, a reference to https://build.fhir.org/search.html#multivalue was made. However, that text has issues:

	It says that OR-values can be separated by commas, but not that they shall be.
	It does not tell how escaping the (comma) separator is to be done when a comma is required in a value. &#91;I suppose that the &amp;-escaping required for AND-searching is already described somewhere.&#93;



Please correct both issues.

Note that the original comments probably apply to all searching, and hence the tools to generate each resource are also to be adapted according to corrections required for this ticket.
