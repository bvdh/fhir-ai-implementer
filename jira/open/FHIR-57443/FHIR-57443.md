# FHIR-57443: Confusion about how Provenance.target of Bundle/id should work

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/25/26
- **Description**: The signatures page says "The Provenance.target with a reference value of exactly Bundle/&#91;id&#93; where id is the id of the Bundle".

However, the Bundle 'id' will be different on every server the Bundle is copied to.  While Provenances aren't part of the signatures by default, this would still mean that when a document Bundle is stored on a server, it would be necessary to update the Provenance.target of all entries to point to the new 'id'.  That seems 'unpleasant', and also isn't called out anywhere.  As well, in some cases the Provenances themselves will be signed, and in this case, changing the id would break the signature.
