# FHIR-54694: Add reference to OperationDefinition to ExampleScenario

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/18/26
- **Description**: Similar to how the ExampleScenario.actor.definition references the canonical of an ActorDefinition it should be possible to reference an existing OperationDefinition within the ExampleScenario.

Proposed solution: Add ExampleScenario.process.step.operation.definition - Canonical(OperationDefinition)
- **Issue Type**: Change Request
- **Key**: FHIR-54694
- **Related Artifact(s)**: ExampleScenario
- **Related URL**: https://hl7.org/fhir/6.0.0-ballot4/examplescenario.html
- **Reporter**: AnjaSchwab
- **Resolution**: Persuasive with Modification
- **Resolution Description**: Will add the element as proposed. 0..1 canonical(OperationDefinition)
short: "Operation invoked"
definition: "If this operation represents the invocation of a FHIR operation, points to the canonical definition of that operation

Will add a constraint that says "definition is only permitted if type.code = 'operation'"

Will add a 'rationale' to type that says "This binding is extensible to provide support for non-FHIR interactions as well."
- **Status**: Resolved - change required
- **Summary**: Add reference to OperationDefinition to ExampleScenario
- **Work Group**: FHIR Infrastructure
