# FHIR-53960: Additional Functions - Inconsistent error handling patterns

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/12/26
- **Description**: Inconsistent error handling patterns - some functions return empty, others throw errors: * Return empty on error/invalid input: hasValue, getValue, toString, memberOf, subsumes, subsumedBy, htmlChecks, conformsTo, elementDefinition, slice, resolve, ofType, and all %server functions (read, create, update, delete, search, patch, validate, transform, everything, apply)

	Throw errors: checkModifiers (throws if modifiers found), %factory functions (noted as "or an error" for primitives)



Note: FHIRPath core specification says singleton-only functions should throw when given multiple values, but FHIR's implementation returns empty instead - this deviation should be explicitly documented and justified
 
Also the weight function doesn't declare if it's a single cardinality input function. Expect that it should be.
