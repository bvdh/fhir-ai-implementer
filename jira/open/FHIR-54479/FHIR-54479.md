# FHIR-54479: Dependent Group parameter variable restrictions

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/16/26
- **Description**: In the Dependent Rules section this text:

"The parameters provided must match the parameters required by the dependent group, in order. In addition, the mode of the variable must match - inputs that are targets must be target variables. Note, though, that target variables can be treated as source for a group."

Doesn't state how literal parameters should be handled, nor is there an example of doing so. The grammar does support doing this.

Are the source or target variables, or can they be used as either?

If a fhirpath expression is used to populate the variable, be clear that it cannot produce a collection and pass that into the group, or state that it can be a collection, as groups process those in their rules normally.
