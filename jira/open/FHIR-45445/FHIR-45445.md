- **Resolution Description**: We will update the ig-guidance IG to declare the following:

 

SearchParameters SHOULD follow these naming rules for the 'code' element, in order of importance:
 * All lower-case
 * Do not use underscore, except as a prefix for shared search parameters that span all resources or base resources.
 * Separate words with dashes
 * Align names with the corresponding resource element name (converting camelCase to dash-case) when the search parameter corresponds to a single element
 * Be consistent with similar search parameters in other resources

Operation parameter names should follow these naming rules for the parameter.name:
 * Be expressed in lowerCamelCase
 * Do not use underscore or dashes
 * Align names with the corresponding resource element name when the operation parameter corresponds to a single element
 * Be consistent with similar parameter names in other operations

We will also create a ticket for the validator to spit out a best practice message when these rules aren't followed.
