- **Description**: FHIRPath defines a set of calendar based units (https://hl7.org/fhirpath/#datetime-arithmetic) so that when adding 1 month, you get the same day of the next month, regardless of the number of days.  In many cases this is what systems would prefer over working with the UCUM unit 'mo', which is a specified as 1/12 of the UCUM 'a_j' unit (a Julian year).  The proposal here is:

1) Create a CodeSystem for these units defined in FHIRPath
2) Change ValueSets like http://build.fhir.org/valueset-units-of-time.html and http://hl7.org/fhir/valueset-age-units.html to use the new calendar based units (either adding them to the existing lists or replacing as appropriate for the ValueSet's use cases).
3) Replace wording introduced by FHIR-24857 advocating treating 'mo' as a calendar month, with wording recommending the use of the new FHIRPath month unit when calendar months are desired.  (Known locations are:  http://build.fhir.org/valueset-units-of-time.html, http://build.fhir.org/datatypes-definitions.html#Timing.repeat.durationUnit, and http://build.fhir.org/datatypes-definitions.html#Timing.repeat.periodUnit).

For additional discussion, see:  https://chat.fhir.org/#narrow/stream/179266-fhirpath/topic/FHIR-24858/near/296428394
