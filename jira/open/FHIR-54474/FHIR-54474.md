# FHIR-54474: Source should be able to be a fhirpath expression

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/16/26
- **Description**: Currently there is no way to take a single property and split it into multiple properties/values.

To achieve this we could update the source to be able to be a fhirpath expression (so that you can split an input field into multiple values)

e.g. (src.codes.split(',')) as scv -&gt; tgt.coding as c, c.code = scv
// example that shows taking an input field that is comma seperated, and populate that into a set of codings
