- **Description**: In https://build.fhir.org/fhirpath.html#functions:

 
hasValue() : Boolean

Returns true if the input collection contains a single value which is a FHIR primitive, and it has a primitive value (e.g. as opposed to not having a value and just having extensions). Otherwise, the return value is empty.
 

See the red underline section. Most implementations return `false` (and not empty). Since the signature is Boolean, shouldn't the return be false in this case?

- **Reporter**: Brian Kaney
- **Resolution**: Persuasive
- **Resolution Description**: Looks like this issue has already been updated for R6:

_If the input collection contains a single value which is a FHIR primitive, then will return true if the single value is a primitive value (e.g., as opposed to not having a value and just having extensions) \{*}and false if not{*}. If the input collection isn't a single FHIR primitive, the return value is empty._

Here you can see the inclusion of the bold text.

*However this could be more clear in the empty side of things that the property not existing also falls into this category.*
