# FHIR-56793: Define a standard extension to indicate available translations

## Metadata

- **Assignee**: Unassigned
- **Created**: 4/30/26
- **Description**: We have a translation extension that allows translations to be embedded on various strings, markdown, and codes in a resource, as well as a means to inject translations into narrative.  However, it's inefficient to search the entire resource to figure out what languages are available, and it's certainly not feasible to have a search parameter that's based on such a search.

Proposal:

1. Define a new extension called availableTranslation with type 0..* code, context Resource, binding - regular language value set

Definition: Indicates a language code for which at least some translatable elements in the resource have &#91;translations&#93;() for that language.

Comment: availableTranslation does not necessarily indicate that translations are available for the specified language for all translatable elements.

2. Define a standard search parameter that goes against this extension called available-translation

3. Define a standard search parameter that goes against this extension + Resource.language called available-language

4. Update the description of the 'translation' extension to note that resources with this extension present SHOULD also populate the availableTranslation extension.
