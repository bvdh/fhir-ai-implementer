# FHIR-53952: FHIR specific variables - clarifications and missing documentation on %profile

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/12/26
- **Description**: The section introduces 2 variables, then defines 3.
Move the %profile into the set of bullet points for the other purpose specific usages (as it's not available in all FHIR contexts).

The code block makes more sense as a table - change to a table.

Add another block that IS an actual set of examples of how you'd use the variable in an expression - with descriptions of what the example is demonstrating, and the context you'd use it in.

Add the variable names that are mentioned in the guidance for:
subs topics (%previous, %current)
questionnaires (%questionnaire. %qItem)
resource invariants (%profile)
(https://hl7.org/fhir/6.0.0-ballot4/elementdefinition-definitions.html#ElementDefinition.constraint.expression)

The documentation on the %profile should be moved to the element definition page, or somewhere else more appropriate where this validation functionality is described.

Also include a note that other FHIR implementation guides may define their own variables for their own context usage.
