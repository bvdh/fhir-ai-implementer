# FHIR-57524: FML grammar doesn't support `evaluate` fhirpath parameter

## Metadata

- **Assignee**: Unassigned
- **Created**: 6/4/26
- **Description**: The spec indicates that for the second argument in the transform should be a fhirpath expression.

 
    v -&gt; tn.text = evaluate(v, 'given.join(\' \')') "r2"; // this is just a string?
    v -&gt; tn.text = evaluate(v, given.join(' ')) "r2";     // this is what the HAPI parser and engine support
 
 
Suggest updating the grammar: (and add a note that the expression is only valid for the evaluate transform)
 
groupParam
  : literal
  | ID
  | fpExpression   // this is to support the `evaluate` transform, parser validation/visitor should check that this is only used there.
  ;

And also clarify in the evaluate transform that this is the type of the parmeter (it's not a string, it's an expression, one of the few places they are included, and the only transform function that IS an expression)
