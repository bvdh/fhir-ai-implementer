Handling Remote Data
Some functions in the FHIR usage of fhirpath involve functionality that could refer to external systems and thus can be subject to variations in system performance, timeouts, availability, caching, security and others.
 
The primary features impacted are:

	resolve (where the resource is not a contained resource)
	terminology related functions (memberOf, subsumes, subsumedBy, weight, and the %terminology factory)
	profile related functions (may need to be handled by a conformance server/resolve structure definitions) conformsTo, slice, elementDefinition
	%server factory functions
 
&#8212;
We should also be indicating what happens when content is unavailable, I think it's meant to just return an empty result rather than throw an exception and abort processing. - is there a way we could add a traceOnRemoteError() or something like that? - maybe more appropriate at the fhirpath level.
