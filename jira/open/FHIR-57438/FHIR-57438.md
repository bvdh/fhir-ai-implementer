- **Description**: The hl7.fhir.r4.core#4.0.1 package contains some terminology (CodeSystems, ValueSets, others) with resource versions of 4.0.1 that currently conflict with the latest versions of that terminology resource in THO. 

For example - ProvenanceParticipantType CodeSystem has a version of 4.0.1 in the fhir.r4 package but the latest version on THO that we're trying to consume through the hl7.terminology.r4#7.1.0 package has a version of 2.0.0 for this resource. 

This is resulting in issues with implementing terminology servers utilizing older versions of the resource (since the version in the r4.core package is higher) - ultimately resulting in errors being generated when implementers use newer codes that are valid in the terminology 7.1.0 package. 

I understand there are manual workarounds (removing the r4.core mis-versioned codeSystem, creating a 5.0.0 version of the same resource on our server with higher version) - however every time we reload either package from the feed it will overwrite the workaround or risk us missing code changes from the terminology package.

My highest priority is getting Provenance-participant-type codeSystem fixed however here are some additional CodeSystems I came across that have a similar issue of a later version in FHIR R4 core package than the terminology package (I'd recommend a thorough analysis):

	action-type
	adverse-event-category




	appointment-cancellation-reason




	catalogType




	can-push-updates




	plan-definition-type




	program




	allergy-intolerance-clinical
	allergy-intolerance-verification
	condition-clinical
	condition-ver-status
	data-absent-reason
	operation-outcome
	research-study-status
