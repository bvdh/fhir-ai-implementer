# FHIR-56412: Add new "relevant-actor" extension

## Metadata

- **Assignee**: Unassigned
- **Created**: 4/22/26
- **Description**: When inheriting from a parent profile, there may be obligations and additional bindings that are inherited that aren't relevant in the context of the current implementation guide.  There can't be a mechanism to cause obligations to be removed when deriving, however, it should be possible to at least not render obligations that aren't relevant.

Context: ImplementationGuide, StructureDefinition

0..* canonical(ActorDefinition)

Definition: "If present, indicates that only obligations and bindings that are globally applicable or are associated with one of the listed ActorDefinitions should be rendered when displaying the profile - as only the specified ActorDefinitions are relevant in the context of the use-case for the profile."

Comment: "If defined at the ImplementationGuide level, the repetitions of this extension listed there should be cascaded to all profiles defined within that ImplementationGuide, with any additional actors listed on those profiles treated as 'additional' relevant actors.

This extension should be flagged as non-inheritable
