- **Description**: If the # includes the whole Bundle, no additional cannolization is needed, 
if this not the case we need to discuss the way cannolications works, e.g. adding canonical rules as suggested below:

To sign the document together with the signing information in the current provenance is very common use case. The current approach of having 2 provance.target elements seems a little complex. Instead, I would suggest to simply define these in canonical rules of document. E.g., to update http://hl7.org/fhir/canonicalization/{fmt}#document to cover this, and to have another canonical rule such http://hl7.org/fhir/canonicalization/{fmt}#documentNoSignInfo to cover the other case which personally I believe is less useful.
