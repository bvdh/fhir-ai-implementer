# FHIR-48493: UseageContext should be sliced to match valueCodeableConcept domains to code

## Metadata

- **Assignee**: Unassigned
- **Created**: 10/3/24
- **Description**: There should be bindings on UsageContext.valueCodeableConcept to pair the values to UsageContext.code. For example, if UsageContext.code =http://terminology.hl7.org/CodeSystem/usage-context-type#gender, then valueCodeableConcept should be bound to http://hl7.org/fhir/ValueSet/administrative-gender. The usage-context-type concept definitions list appropriate value sets for most of the defined concepts. (Jurisdiction has a defined value set but it is not listed.)

The actual type of binding (required, extensible) will vary depending on concept.

I believe this can be accomplished with additional bindings, and setting the usage of each value set to the associated use code.
