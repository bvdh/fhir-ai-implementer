# FHIR-47635: Versions - withdrawn is not define by HTML 4.0

## Metadata

- **Assignee**: Unassigned
- **Created**: 9/13/24
- **Description**: I think the word used in HTLM 4 is "Obsolete" and not "withdrawn.

We should decide on what word to use. I suggest that we use "withdrawn" where HTTP 4 uses obsolete (prevents having to scan and update the complete specification...).

This is an important feature and should be very clear. I suggest that we copy the definitions from HTML4 to this section, change obsolete to withdrawn and refer to HTML 4 as the source of the info.
