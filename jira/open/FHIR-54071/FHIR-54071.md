# FHIR-54071: Range: description says arbitrarily high precision, but constraint uses precision boundaries

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/13/26
- **Description**: Constraint rng-2 uses lowBoundary() and highBoundary(). This means, for example, the following Range passes this constraint:

{{{}}
    "low": { "value": 2.4},
    "high": {"value": 2}
}

(because{{ high.highBoundary() }}is 2.5 )

This could arguably be correct given how numeric precision works. However, the description for Range also says:
The low and the high values are inclusive and are assumed to have arbitrarily high precision; e.g., the range 1.5 to 2.5 includes 1.50, and 2.50 but not 1.49 or 2.51. 
These seem at odds. Constraint rng-2 assumes a specific precision based off the value, but the description implies that low and high values should be taken as having infinite precision.
