# FHIR-54601: Handling of difference in range of FHIR.decimal and System.Decimal

## Metadata

- **__EMPTY**: Bryn Rhodes
- **__EMPTY_1**: Public
- **__EMPTY_2**: The 18 digits comes from the minimum decimal representation in XML (https://www.w3.org/TR/xmlschema-2/#decimal)

The 28,8 in FHIRPath comes from bcd decimal representation ranges common in database management systems, 28 total digits, with 8 digits of scale.

Note that while both the XML and FHIRPath decimal representations characterize the limits as minimums that systems can provide more support for, FHIR explicitly characterizes the 18 digits as a maximum. Agree that the FHIRPath page in FHIR should be clear about what happens when a FHIRPath decimal value exceeds this range, but should it be an error? A warning?
- **Assignee**: Unassigned
- **Comments**: 1/20/26
- **Created**: 1/16/26
- **Description**: There is a large difference in the range of decimal between FHIR (1e18) and FHIRPath (1e28).  Somewhere in the FHIRPath specification there should be a statement about what do with FHIRPath decimals that are outside of the range of FHIR decimals when placing a value into a FHIR resource.
