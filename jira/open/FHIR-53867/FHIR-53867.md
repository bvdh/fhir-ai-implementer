# FHIR-53867: Clarify handling of If-None-Exist for servers that don't support Conditional create

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/8/26
- **Description**: If a server does not support the Conditional Create interaction, but does support the Create interaction, and receives a Create request with an If-None-Exist header, should the server ignore the header and process the request as if it were a normal Create, or should the server reject the request?

While clients should be able to determine whether Conditional create is supported for a particular server via the CapabilityStatement, throwing an error might provide an extra layer of protection, particularly for use cases in which duplicate checking has clinical implications (e.g. Medications, Patient matching).
