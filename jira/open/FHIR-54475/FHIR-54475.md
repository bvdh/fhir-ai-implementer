# FHIR-54475: Prefixes section presents three representations of same concepts

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/16/26
- **Description**: The recent updates to this section now present four different definitions for the meaning of each prefix. It isn't clear which one is authoritative, when one might apply or another, or how the three relate.

 

For example, definition 1:
The range of the parameter value fully contains the range of the resource value.
Definition 2:
Boundary interpretation: The lower boundary of the resource value is greater than or equal to the lower boundary of the parameter value AND the upper boundary of the resource value is less than or equal to the upper boundary of the parameter value.
Definition 3:
Resource interval tests:
 resource-lower: &#91;parameter-lower, parameter-upper&#93;
 resource-upper: &#91;parameter-lower, parameter-upper&#93;
Definition 4 is the true/false table of prefixes and value bars.

Less useful/harder to understand definitions should be removed, and the relationship/use of the various definitions clarified.
