# FHIR-57495: Derived OperationDefinition parameter.type issue

## Metadata

- **Assignee**: Unassigned
- **Created**: 6/1/26
- **Description**: In the "Derived OperationDefinitions" section, we have the following rule:


if the original parameter.type is valued, SHOULD have the same type as the original parameter

However, I believe this is slightly off in some cases.  E.g., if an original operation defines a return parameter of type Resource, a derived operation may want to restrict to a specific resource type (e.g., Parameters, OperationOutcome, etc.).
