# FHIR-54059: Usage of Subscription.filterBy.event is unclear

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/13/26
- **Description**: Two parts of Subscription.filterBy.event are confusing to me:

 

1) If I want to filter by just an event (not also a parameter), it's unclear how to model this, because filterBy.filterParameter and filterBy.value are both required. 

 

2) The long description for this element says:
An event filter to be applied to the topic - e.g., if a topic defined multiple event triggers, this can be used to specify a single one. Multiple values are or-joined, multiple codings within a value are and-joined.
This is confusing to me; CodeableConcept.coding should provide translations of a given singular concept, but "and-joining" implies that multiple different events would be supplied.

(without this description, I'd have expected implementers to just find all recognized codings within the CodeableConcept, and match upon at least one of those)
