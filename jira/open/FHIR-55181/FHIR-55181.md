# FHIR-55181: Questionnaire.item.type has no escape valve

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/20/26
- **Description**: The element Questionnaire.item.type is a mandatory element with a required binding, but appears to have neither an 'unknown' or 'other' concept, which can make the containing element impossible to populate in some cases. Please add 'unknown', 'other', or some equivalent escape valve concept to the value set.

(Comment 147 - imported by: Lloyd McKenzie)

- **Resolution Description**: Because the Questionnaire.item.type is so intrinsically tied to the allowed data types of much of the item metadata as well as the QuestionnaireResponse answer data type, there's no practical way to allow 'other' types here.

Instead, we'll add a new section to Questionnaire notes called "Handling alternative question types" that says:

"In most cases, if there's a need to capture an answer type not in the list of allowed types, the preferred solution is to use a group with child questions that define the needed data.  However, if there's a genuine need to convey an unsupported type (e.g. SampledData), the recommended solution is as follows:
 * Set the item.type to 'string'
 * Place a modifier extension on the item that indicates that the type isn't actually a string, but something else
 * Communicate the answer (and if necessary initial values, min values, max values, etc. with an extension with a value of the desired data type)"

 

Will update the Questionnaire.type to reference that note section.
