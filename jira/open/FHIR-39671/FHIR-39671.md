The submitter is right.  the security-labels value set should point to http://terminology.hl7.org/ValueSet/v3-Confidentiality which is the new name and canonical for the value set since it was moved into THO instead of https://terminology.hl7.org/ValueSet/v3-ConfidentialityClassification which doesn't exist.  I checked and the old FHIR representation as the same OID and also the exact same definition and include so from a non-FHIR perspective they are the same value set and from a FHIR perspective they're functionally the same value set.
