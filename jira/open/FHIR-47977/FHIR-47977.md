# FHIR-47977: Data Absence Reason exception value set extensions

## Metadata

- **Assignee**: Unassigned
- **Created**: 9/16/24
- **Description**: Data absence should include additional reasons such as absence due to cross version extensions.
The list of errors should be extended with the error: not-a-datetime

Summarizing: please add the following specializations to the data absence reason valueset.

	not-applicable -&gt; cross-version
	error -&gt; not a datetime
- **Issue Type**: Change Request
- **Key**: FHIR-47977
- **Related Artifact(s)**: Data Absent Reason
- **Related URL**: https://hl7.org/fhir/extensions/5.2.0-ballot/StructureDefinition-data-absent-reason.html
- **Reporter**: Bas van den Heuvel
- **Resolution**: Persuasive with Modification
- **Resolution Description**: Add two new codes to the [DataAbsentReason|http://terminology.hl7.org/CodeSystem/data-absent-reason] Code System:
* Under {{error}} category, add {{not-a-date-time}} with the definition: "The value is undefined or unrepresentable as a date or time-based datatype (e.g., Date, Time, DateTime, Instant)."
* Under {{unsupported}}, add {{replaced-by-cross-version}} with the definition: "The value is unrepresentable in the element as defined in FHIR version of this resource and is instead present as a cross-version extension value."
- **Status**: Resolved - change required
- **Summary**: Data Absence Reason exception value set extensions
