# FHIR-53965: Environment Variables - clarifications

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/12/26
- **Description**: Make the following clarifications to the Environment Variables section:

	Include a note that the environment variables don't change between uses where the other extended variables defined in parts of fhir (such as %resource) are specific to an expressions specific evaluation. - This is the primary difference between env vars and fhir specific vars.
	%resource is already documented as a FHIR Specific variable, it should not be defined in the environment variables section.
	Include an implementer note that the use of the &#91;name&#93; in the environment variables requires that the extension profiles and valuesets referred to have the same id as was published with - which may have issues where there are multiple possible versions in scope.
