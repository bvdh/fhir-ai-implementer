# FHIR-57259: Clarify "complex types" in discussion of equality for structured values

## Metadata

- **Assignee**: Unassigned
- **Created**: 5/16/26
- **Description**: The discussion on changes to operators in FHIRPath says:

&gt; Equivalence works in exactly the same manner, but with the addition that for complex types, equality requires all child properties to be equal, except for "id" elements.

This is confusing in that it says "Equivalence" to start, but then changes to equality at the end. This should be clarified that it applies to equivalence, not to equality.

In addition, it should be clarified that it applies to "complex types" as well as "resource" types.
