# FHIR-54094: Reference.type condition not clear

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/13/26
- **Description**: https://hl7.org/fhir/6.0.0-ballot4/references.html

and https://hl7.org/fhir/6.0.0-ballot4/references-definitions.html#Reference.type 

In the Short display, it is mentioned "must be a resource in resources". Do we mean "must be a resource from &#91;Resource Types&#93;(https://hl7.org/fhir/6.0.0-ballot4/valueset-resource-types.html)" ? In that case, the binding of Reference.type to Resource Types value set should not be extensible, but required, which can be restrictive. Maybe change the short description to "should be a resource from &#91;Resource Types&#93;(https://hl7.org/fhir/6.0.0-ballot4/valueset-resource-types.html)"
