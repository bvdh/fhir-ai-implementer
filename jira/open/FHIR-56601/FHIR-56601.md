# FHIR-56601: OperationDefinition incorrectly classified as Definition pattern

## Metadata

- **Assignee**: Unassigned
- **Created**: 4/29/26
- **Description**: OperationDefinition follows the workflow Definition pattern despite not being related to workflow any more than a SearchParameter or DeviceDefinition is.

It is my understanding that only definitional resource types that define an Event follow the Definition pattern.

If Operation Definition is not a Definition, then is shouldn't claim that or be listed as one.

If my understanding is incorrect and OperationDefinition is a Definition, the other definitional-type resources (StructureDefinition; SearchParameter; MessageDefinition; CompartmentDefinition; StructureMap; ConceptMap; DeviceDefinition; and on, and on, and on – there are about 20 of them) should be checked for the claim of implementing the pattern, and updated.
