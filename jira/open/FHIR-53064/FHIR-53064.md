# FHIR-53064: Restrict permitted characters in data content

## Metadata

- **Assignee**: Unassigned
- **Created**: 10/6/25
- **Description**: IETF RFC 9839 discusses particular sets of Unicode characters that should be excluded from standards based on JSON (and XML) mostly because they aren't expected/permitted in well-formed unicode. A summary of the (not actually very long) RFC is available on the RFC author's blog. Previous work in this space includes RFC 8264.

To reflect this, we should do one or more of:

	update our data type definitions to prohibit appropriate unicode code points
	update our XML and JSON data representations so to prohibit appropriate unicode code points
	update our security guidance to recommend filtering certain unicode code points, and/or point to these documents.
	update our best practices to recommend filtering certain unicode code points, and/or point to these documents.
