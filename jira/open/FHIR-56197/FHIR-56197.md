- **Description**: In StructureDefinition.context we allow three different types. Request additional documentation for each of the types and what the expression means / how it is evaluated in each case.

For the extension and element types, we should clarify where in the extension the new extension is allowed to appear. For element and extension, the new definition only targets that element/extension (root element).

Request a new extension on StructureDefinition.context.type: 

	allowsDescendants
	boolean
	0..1
	Whether the context expression includes all descendants of the nominated expression
	This is only valid when the type is element or extension - when using fhirpath, the expression should be created to correctly identify either the individual element or the element and its descendants.
