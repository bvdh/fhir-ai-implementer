# FHIR-54079: Add examples showing the comparison of the terminology subsumes factory vs function usage

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/13/26
- **Description**: The subsumes() and subsumedBy() functions mention using a terminology service, and show an example of calling using the %terminolgy.subsumes() function, but not using the actual function created.
This below example should be clearer to show the difference between using the fluent or factory approach.
```
Procedure.category.subsumes(Procedure.code) // via fluent invocation
%terminologies.subsumes(Procedure.category, Procedure.code) = 'subsumes' // as a boolean test using the %terminology factory
```
And also include a link to the function that is also being used, and repeat the example down in the %terminology section too so people are aware of the 2 ways to call the functionality and be able to decide which is more appropriate for them.
