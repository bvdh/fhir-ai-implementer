# FHIR-53927: Clarify future support of mode parameter in relation to potential R6 normativity

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/10/26
- **Description**: The capabilities section of the HTTP page says that:

"Servers MAY ignore the mode parameter and return a CapabilityStatement resource. Note: servers might be required to support this parameter in further versions of this specification".
