# FHIR-52999: Clarify that discriminator paths should be relative

## Metadata

- **Assignee**: Unassigned
- **Created**: 9/29/25
- **Description**: Please clarify that the FHIRPath expression in the 


ElemenentDefinition.slicing.discriminator.path

 element should be a path relative to the sliced element and not start at any other node (such as the resource root).

The specification should state that expressions of pattern


Path.to.sliced.element.path.to.discriminatingValue

are not allowed while


$this.path.to.discriminatingValue

or


path.to.discriminatingValue

are. Or rather that the provided expression is evaluated from the sliced element.
