# FHIR-44621: For backported elements, how do extension contexts work?

- **Description**: If an element is brought into a different version via cross-version extensions, how are extensions managed?

For example, if there's an extension defined with a context of CodeableReference, will the 'extension' that represents the CodeableReference when used in an R4 instance be considered a valid element for use of that extension even though it doesn't explicitly have a context of 'extension'?
- **Issue Type**: Change Request
- **Key**: FHIR-44621
- **Related Page(s)**: Managing Multiple Versions
- **Reporter**: Lloyd McKenzie
- **Resolution**: Persuasive
- **Resolution Description**: Will add the following language:

"In some situations, there will be a need to send an extension on an element that is being communicated using an inter-version extension.  The extension to be communicated will have a 'context' of the element defined in that foreign version and will often NOT have an explicit context of the inter-version extension - whether a 'standard' extension that was explicitly defined for the purpose or one using the special syntax defined here that is implicitly defined.  In either case, validators SHALL behave as though the context includes the relevant cross-version extension."
