# FHIR-55477: Possible need for invariant on Bundle.identifier

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/28/26
- **Description**: When reviewing FHIR-55246, InM identified a potential need for an invariant on Bundle.identifier. The Messaging documentation (https://build.fhir.org/messaging.html) indicates that " Each time a new message is created, it SHALL be assigned an identifier (Bundle.identifier) that is unique within that message stream.". This would seem to indicate that the cardinality of Bundle.identifier should be 1..1 when Bundle.type is "message" (note that other types, such as document, may warrant always having a Bundle.identifier value).

Requiring a Bundle identifier is in line with the requirements of MessageHeader,response where the identifier of the original message is required (1..1).

The creation of an invariant to enforce this SHALL requirement should be considered.
