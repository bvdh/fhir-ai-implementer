# FHIR-43565: Provide JSON files in base JSON schema file (not with openapi bundle)

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/8/24
- **Description**: Please provide the JSON schema files in base JSON schema space (https://json-schema.org/specification) as opposed to including a bundling technology like openAPI and combining the specifications.  Each file should be distributed individually (within a zip) but should not be bundled if the spec doesn't provide a mechanism for it. If you are planning on use the openAPI Schema version, then please reference that as a download link

Example: discriminator/mapping are not listed within the scheme spec.
  "$schema": "http://json-schema.org/draft-06/schema#",
  "id": "http://hl7.org/fhir/json-schema/5.0",
  "description": "see http://hl7.org/fhir/json.html#schema for information about the FHIR Json Schemas",
  "discriminator": {
    "propertyName": "resourceType",
    "mapping": {
