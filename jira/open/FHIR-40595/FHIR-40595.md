# FHIR-40595: Clarify whether secondary resources not mentioned in _elements are omitted

## Metadata

- **Assignee**: Unassigned
- **Created**: 3/13/23
- **Description**: The text for _elements indicates that if the resource type is included in the element name, then those are the only elements returned for secondary resources in the search. However, it isn't clear what to do with those secondary resources if the resource name isn't included in the element name, or if resources match that aren't mentioned.

Clarify whether entire secondary resources should be filtered out, whether only the mandatory content of secondary resource should be returned, or whether the full secondary resource should be returned if they aren't mentioned in an _elements filter.
- **Grouping**: [R6]
- **Issue Type**: Change Request
- **Key**: FHIR-40595
- **Related Page(s)**: Search
- **Reporter**: Elliot Silver
- **Resolution**: Persuasive
- **Resolution Description**: Suggest adding clarification that resources not specified in the {{_elements}} parameter are returned in full.

Other default behaviors do not allow the full resource to be returned without specifying every single element in the resource.
- **Status**: Triaged
- **Summary**: Clarify whether secondary resources not mentioned in _elements are omitted
