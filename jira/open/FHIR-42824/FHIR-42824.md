# FHIR-42824: Definition and comment on CapabilityStatement.imports are contradictory

## Metadata

- **Assignee**: Corey Spears
- **Created**: 9/26/23
- **Description**: The definition of CapabilityStatement.imports states that the server may repeat the same resources, interactions and operations to add additional details to them.

On the other hand the comment on the same element states that the contents of any directly or indirectly imported CapabilityStatements SHALL NOT overlap, i.e. they cannot refer to the same rest/resource, operations/name, searchparam/name, interaction/code, messaging/endpoint, document/mode pair.

How is it possible to add additional details to e.g. a certain resource in a CapabilityStatement that imports another CapabilityStatement, if it's not allowed to refer to the same resource within both CapabilityStatements?

It seems like the comment on .imports should be clarified, mainly to make clear what 'overlap' means. For instance it makes sense to not allow the inclusion of a certain resource in the second CapabilityStatement if it adds nothing new, or includes a sub element that contradicts the imported CapabilityStatement.
