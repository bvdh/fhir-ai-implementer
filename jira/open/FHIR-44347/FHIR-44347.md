# FHIR-44347: Expand the extension.value data type options to allow for R4 implementations/tooling that don't recognize R5 resources (e.g., ActorDefinition)

## Metadata

- **Assignee**: Unassigned
- **Created**: 1/22/24
- **Description**: Since this is intended to be a cross version extension, I am asking for expansion of the value data types to allow for actor to be conveyed as a CodeableConcept (or string) for R4 implementations. When I attempt to define the ActorDefinition (R5 resource) it isn't accepted by tooling that is version specific (e.g., won't render in a Simplifier project and produces errors). I believe this will will also present challenges for users who are only implementing against the R4 package. If I leave it blank, it incurs meaning that I don't intend (i.e., applies to all actos). If there is an alternative way to pre-adopt this resource, the description of such in the extension would be a suitable resolution.

- **Resolution Description**: We will adjust the documentation on inter-version extensions to indicate that when you have a canonical that points to a resource not defined in the current release, you use Basic to represent the target resource and use the 'alternate-canonical' extension in the canonical to point to that.  We'll include an example.

(Note that there's a technical correction to be made to the extension definition, so you need to wait until an updated extensions-pack release gets out and some validator changes get made before this will work.)
