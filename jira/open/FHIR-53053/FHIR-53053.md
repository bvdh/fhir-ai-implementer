# FHIR-53053: Bundle.entry.response.outcome description update

## Metadata

- **Assignee**: Unassigned
- **Created**: 10/3/25
- **Description**: In R4, Bundle.entry.response.outcome is typed as Resource. The base spec doesn’t constrain it to OperationOutcome, yet the description says “OperationOutcome with hints and warnings (for batch/transaction).” If the type cannot be changed, please clarify in the spec that OperationOutcome is the intended/typical resource, add a non-normative SHOULD for servers to return OperationOutcome when providing processing details/errors.
