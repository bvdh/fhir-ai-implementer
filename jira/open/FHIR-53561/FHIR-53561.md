# FHIR-53561: Coding/CodeableConcept validation - improve wording

## Metadata

- **Assignee**: Unassigned
- **Created**: 12/2/25
- **Description**: It seems the wording on this page could be improved: https://build.fhir.org/validation

 

Specifically, it would help to clarify that Coding/CodeableConcept validation should include not only checking bindings, but also validating the Coding/CodeableConcept elements themselves. The current text focuses primarily on binding validation, which may unintentionally suggest that the validator only checks the binding and not the internal structure or correctness of the Coding/CodeableConcept.

 

Updating the wording to reflect both aspects of validation explicitly would make the intent clearer.
