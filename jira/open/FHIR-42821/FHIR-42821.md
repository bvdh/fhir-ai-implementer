# FHIR-42821: Obligations change the scope of FHIR.  Should make this more explicit

## Metadata

- **Assignee**: Grahame Grieve
- **Created**: 9/25/23
- **Description**: MustSupport/Obligations cause FHIR to move beyond merely being focused on data exchange, but we haven't made that super clear when we describe what FHIR is.  In the various introductory materials (including the architecture overview) as well as on the Obligations page, we should make clear that, while the core FHIR specification is focused entirely on exchange, the FHIR implementation guide framework - mainly through obligations - has the ability to define expectations on system behavior above and beyond the exchange layer.

We should also talk about the fact that the 'actors' we refer to in obligations are logical entities and the obligations may actually be satisfied by a collection of separate physical systems.
