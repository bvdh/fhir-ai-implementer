- **Description**: TestReport is not (easily) usable outside reporting on execution of TestScripts - i.e. detailed tests with the structure of the FHIR TestScript resource

Interoperability testing requires different tests - manual tests, tests expressed in other ways (narrative, Gherkin, Postman). 

The TestPlan resource is also a way to express test cases and for which there should be a Test Report.

After some attempts to use TestReport, we realize this is not usable and we need a resource for a Test Report.

 

The approach to the new / revamped resource should be similar to what was used in TestPlan - assume that there may be different types of tests, using different languages, with different criteria, with data that may or may not be available or may or not be valid FHIR resources... 

Request: 

Redesign the TestReport resource to support TestPlans and other ways of expressing tests.
