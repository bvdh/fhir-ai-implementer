# FHIR-54477: Simple Identity transform - no FML grammar support, and heading improvement

- **__EMPTY_2**: Change the mapRule in the grammar from:

  : qualifiedIdentifier '-&gt;' qualifiedIdentifier ruleName? ';'  #mapSimpleCopy
  | mapTransformationRule ';'                                 #mapFhirMarkup

to:

  : qualifiedIdentifier '-&gt;' qualifiedIdentifier ':' identityFieldList ruleName? ';'  #mapSimpleBatchIdentity
  | qualifiedIdentifier '-&gt;' qualifiedIdentifier ruleName? ';'                         #mapSimpleCopy
  | mapTransformationRule ';'                                                          #mapFhirMarkup

and include:

identityFieldList
  : identifier (',' identifier)*
  ;

- **Description**: Simple Form: Identity transform

	grammar does not support the super simple identity form:
`src -&gt; tgt: type, subtype, action, recorded;`
where it just lists out the properties to copy from the source to the target.
	this super simple form should have it's own sub-heading so it can be numbered and easily referenced
 
	better describe how the identity transforms do the creation, and that this is how all properties are actually implicitly created.




	Just referencing the property as a target is enough to create it
	doesn't need to explicitly create, unless the type is ambiguous, such as for choice type properties
	how is this represented in the StructureMap?
	Simple Form(s) should be moved earlier in the documentation
